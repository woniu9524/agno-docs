[Model Context Protocol (MCP)](https://modelcontextprotocol.io) は、エージェントが標準化されたインターフェイスを通じて外部システムと対話できるようにします。
Agno の MCP 統合を使用して、任意の MCP サーバーにエージェントを接続できます。

## 使用方法

<Steps>
    <Step title="使用したい MCP サーバーを見つける">
    機能する任意の MCP サーバーを使用できます。例を確認するには、MCP のメンテナー自身による [この GitHub リポジトリ](https://github.com/modelcontextprotocol/servers) をチェックしてください。
    </Step>

    <Step title="MCP 統合を初期化する">
    `MCPTools` クラスをコンテキストマネージャーとして初期化します。MCP サーバーを定義する推奨される方法は、`command` または `url` パラメータを使用することです。`command` を使用すると、使用したい MCP サーバーを実行するために使用されるコマンドを渡すことができます。`url` を使用すると、使用したい実行中の MCP サーバーの URL を渡すことができます。


    たとえば、"[mcp-server-git](https://github.com/modelcontextprotocol/servers/tree/main/src/git)" サーバーを使用するには、次のようにします。


    ```python
    from agno.tools.mcp import MCPTools

    async with MCPTools(command=f"uvx mcp-server-git") as mcp_tools:
        ...
    ```
    </Step>

    <Step title="MCPTools をエージェントに提供する">
    エージェントを初期化する際に、`tools` パラメータに `MCPTools` クラスを渡します。


    これで、エージェントは MCP サーバーを使用する準備が整いました。


    ```python
    from agno.agent import Agent
    from agno.models.openai import OpenAIChat
    from agno.tools.mcp import MCPTools

    async with MCPTools(command=f"uvx mcp-server-git") as mcp_tools:
        # エージェントのセットアップと実行
        agent = Agent(model=OpenAIChat(id="gpt-4o"), tools=[mcp_tools])
        await agent.aprint_response("What is the license for this project?", stream=True)
    ```
    </Step>

</Steps>


### 基本的な例: ファイルシステムエージェント

これは、[Filesystem MCP サーバー](https://github.com/modelcontextprotocol/servers/tree/main/src/filesystem) を使用してファイルを探索および分析するファイルシステムエージェントです。

```python filesystem_agent.py
import asyncio
from pathlib import Path
from textwrap import dedent

from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.tools.mcp import MCPTools
from mcp import StdioServerParameters


async def run_agent(message: str) -> None:
    """指定されたメッセージでファイルシステムエージェントを実行します。"""

    file_path = str(Path(__file__).parent.parent.parent.parent)

    # ファイルシステムにアクセスするための MCP サーバー (`npx` 経由)
    async with MCPTools(f"npx -y @modelcontextprotocol/server-filesystem {file_path}") as mcp_tools:
        agent = Agent(
            model=OpenAIChat(id="gpt-4o"),
            tools=[mcp_tools],
            instructions=dedent("""\
                あなたはファイルシステムの助手です。ユーザーがファイルやディレクトリを探索するのを手伝ってください。

                - ファイルシステムを移動して質問に答える
                - list_allowed_directories ツールを使用してアクセスできるディレクトリを見つける
                - 検査するファイルに関する明確なコンテキストを提供する
                - 見出しを使用して応答を整理する
                - 簡潔にし、関連情報に焦点を当てる\
            """),
            markdown=True,
            show_tool_calls=True,
        )

        # エージェントを実行する
        await agent.aprint_response(message, stream=True)


# 使用例
if __name__ == "__main__":
    # 基本的な例 - プロジェクトライセンスの探索
    asyncio.run(run_agent("What is the license for this project?"))
```


## Agno Playground で MCP を使用する

Agno Playground では MCP サーバーを実行することもできます。Playground は、エージェントと対話するための Web インターフェイスを提供します。以下は、Playground で実行されている GitHub エージェントの例です。

```python github_playground.py
import asyncio
from os import getenv
from textwrap import dedent

import nest_asyncio
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.playground import Playground
from agno.storage.agent.sqlite import SqliteAgentStorage
from agno.tools.mcp import MCPTools

# ネストされたイベントループを許可する
nest_asyncio.apply()

agent_storage_file: str = "tmp/agents.db"


async def run_server() -> None:
    """GitHub エージェントサーバーを実行します。"""
    github_token = getenv("GITHUB_TOKEN") or getenv("GITHUB_ACCESS_TOKEN")
    if not github_token:
        raise ValueError("GITHUB_TOKEN 環境変数が必須です")

    # MCP サーバーに接続するためのクライアントセッションを作成する
    async with MCPTools("npx -y @modelcontextprotocol/server-github") as mcp_tools:
        agent = Agent(
            name="MCP GitHub Agent",
            tools=[mcp_tools],
            instructions=dedent("""\
                あなたは GitHub のアシスタントです。ユーザーがリポジトリとそのアクティビティを探索するのを手伝ってください。

                - 見出しを使用して応答を整理する
                - 簡潔にし、関連情報に焦点を当てる\
            """),
            model=OpenAIChat(id="gpt-4o"),
            storage=SqliteAgentStorage(
                table_name="basic_agent",
                db_file=agent_storage_file,
                auto_upgrade_schema=True,
            ),
            add_history_to_messages=True,
            num_history_responses=3,
            add_datetime_to_instructions=True,
            markdown=True,
        )

        playground = Playground(agents=[agent])
        app = playground.get_app()

        # MCPTools コンテキストマネージャーをアクティブに保ちながらアプリを提供する
        playground.serve(app)


if __name__ == "__main__":
    asyncio.run(run_server())
```


## ベストプラクティス

1. **エラーハンドリング**: MCP サーバーの接続と操作に対して、常に適切なエラーハンドリングを含めてください。

2. **リソースクリーンアップ**: `MCPTools` または `MultiMCPTools` を非同期コンテキストマネージャーとして使用して、リソースの適切なクリーンアップを保証します。
```python
async with MCPTools(command) as mcp_tools:
    # ここにエージェントコードを記述
```

3. **明確な指示**: エージェントに明確で具体的な指示を提供します。

```python
instructions = """
あなたはファイルシステムの助手です。ユーザーがファイルやディレクトリを探索するのを手伝ってください。
- ファイルシステムを移動して質問に答える
- list_allowed_directories ツールを使用してアクセスできるディレクトリを見つける
- 検査するファイルに関する明確なコンテキストを提供する
- 簡潔にし、関連情報に焦点を当てる
"""
```
