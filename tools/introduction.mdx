Toolsは、Web検索、SQLの実行、メールの送信、APIの呼び出しなどの外部システムとの対話を可能にすることで、エージェントを「エージェントらしく」します。

Agnoには80以上の事前構築済みツールキットが付属していますが、ほとんどの場合、独自のツールを作成することになります。一般的な構文は次のとおりです。

```python
import random

from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.tools import tool


@tool(show_result=True, stop_after_tool_call=True)
def get_weather(city: str) -> str:
    """都市の天気情報を取得します。"""
    # 実際の実装では、これは天気APIを呼び出します
    weather_conditions = ["sunny", "cloudy", "rainy", "snowy", "windy"]
    random_weather = random.choice(weather_conditions)

    return f"The weather in {city} is {random_weather}."


agent = Agent(
    model=OpenAIChat(model="gpt-4o-mini"),
    tools=[get_weather],
    markdown=True,
)
agent.print_response("What is the weather in San Francisco?", stream=True)
```

<Tip>

上記の例では、`get_weather`関数がツールです。この関数が呼び出されると、`show_result=True`を設定したため、ツールの結果が出力に表示されます。

その後、`stop_after_tool_call=True`を設定したため、エージェントはツール呼び出し後に停止します。

</Tip>

### Toolkitクラスの使用

`Toolkit`クラスは、追加の実行制御とともに複数のツールを管理する方法を提供します。実行後にエージェントを停止すべきツールと、その結果を表示すべきツールを指定できます。

```python
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.tools.googlesearch import GoogleSearchTools

agent = Agent(
    model=OpenAIChat(id="gpt-4.5-preview"),
    tools=[
        GoogleSearchTools(
            stop_after_tool_call_tools=["google_search"],
            show_result_tools=["google_search"],
        )
    ],
    show_tool_calls=True,
)

agent.print_response("What's the latest about gpt 4.5?", markdown=True)
```

この例では、`GoogleSearchTools`ツールキットは、`google_search`関数を実行した後にエージェントを停止し、この関数の結果を表示するように構成されています。
