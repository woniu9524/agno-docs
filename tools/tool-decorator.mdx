ほとんどの本番環境では、独自のツールを作成する必要があります。そのため、Agno では最高のツール利用体験を提供することに注力しています。

ルールは簡単です。

- 任意の Python 関数をエージェントのツールとして使用できます。
- `@tool` デコレータを使用して、このツールが呼び出される前後に何が起こるかを変更します。

## 任意の Python 関数をツールとして使用する

たとえば、`get_top_hackernews_stories` 関数をツールとして使用する方法は次のとおりです。

```python hn_agent.py
import json
import httpx

from agno.agent import Agent

def get_top_hackernews_stories(num_stories: int = 10) -> str:
    """
    この関数を使用して、Hacker News のトップストーリーを取得します。

    Args:
        num_stories (int): 返されるストーリーの数。デフォルトは 10 です。

    Returns:
        str: トップストーリーの JSON 文字列。
    """

    # トップストーリー ID を取得
    response = httpx.get('https://hacker-news.firebaseio.com/v0/topstories.json')
    story_ids = response.json()

    # ストーリーの詳細を取得
    stories = []
    for story_id in story_ids[:num_stories]:
        story_response = httpx.get(f'https://hacker-news.firebaseio.com/v0/item/{story_id}.json')
        story = story_response.json()
        if "text" in story:
            story.pop("text", None)
        stories.append(story)
    return json.dumps(stories)

agent = Agent(tools=[get_top_hackernews_stories], show_tool_calls=True, markdown=True)
agent.print_response("Summarize the top 5 stories on hackernews?", stream=True)
```

## `@tool` デコレータのマジック

ツールの動作を変更するには、`@tool` デコレータを使用します。主な機能は次のとおりです。

- `requires_confirmation=True`: 実行前にユーザーの確認が必要です。
- `requires_user_input=True`: 実行前にユーザーの入力が必要です。`user_input_fields` を使用して、どのフィールドにユーザー入力が必要かを指定します。
- `external_execution=True`: ツールはエージェントの制御外で実行されます。
- `show_result=True`: エージェントの応答でツール呼び出しの結果を表示します。このフラグがない場合、ツール呼び出しの結果はモデルに送信され、さらに処理されます。
- `stop_after_tool_call=True`: ツール呼び出し後にエージェントの実行を停止します。
- `tool_hooks`: このツール呼び出しの前後にカスタムロジックを実行します。
- `cache_results=True`: 同じ呼び出しを繰り返さないように、ツール結果をキャッシュします。`cache_dir` と `cache_ttl` を使用してキャッシュを設定します。

ここでは、`@tool` デコレータで可能な多くのパラメータを使用した例を示します。

```python advanced_tool.py
import httpx
from agno.agent import Agent
from agno.tools import tool
from typing import Any, Callable, Dict

def logger_hook(function_name: str, function_call: Callable, arguments: Dict[str, Any]):
    """ツール実行をラップするフック関数"""
    print(f"About to call {function_name} with arguments: {arguments}")
    result = function_call(**arguments)
    print(f"Function call completed with result: {result}")
    return result

@tool(
    name="fetch_hackernews_stories",                # ツールのカスタム名 (指定しない場合は関数名が使用されます)
    description="Get top stories from Hacker News",  # カスタム説明 (指定しない場合は関数の docstring が使用されます)
    show_result=True,                               # 関数呼び出し後に結果を表示
    stop_after_tool_call=True,                      # ツール呼び出し後にすぐに結果を返し、エージェントを停止
    tool_hooks=[logger_hook],                       # 実行前後に実行するフック
    requires_confirmation=True,                     # 実行前にユーザーの確認が必要
    cache_results=True,                             # 結果のキャッシュを有効化
    cache_dir="/tmp/agno_cache",                    # カスタムキャッシュディレクトリ
    cache_ttl=3600                                  # キャッシュ TTL (秒単位、1時間)
)
def get_top_hackernews_stories(num_stories: int = 5) -> str:
    """
    Hacker News のトップストーリーを取得します。

    Args:
        num_stories: 取得するストーリーの数 (デフォルト: 5)

    Returns:
        str: トップストーリーをテキスト形式で返します
    """
    # トップストーリー ID を取得
    response = httpx.get("https://hacker-news.firebaseio.com/v0/topstories.json")
    story_ids = response.json()

    # ストーリーの詳細を取得
    stories = []
    for story_id in story_ids[:num_stories]:
        story_response = httpx.get(f"https://hacker-news.firebaseio.com/v0/item/{story_id}.json")
        story = story_response.json()
        stories.append(f"{story.get('title')} - {story.get('url', 'No URL')}")

    return "\n".join(stories)

agent = Agent(tools=[get_top_hackernews_stories])
agent.print_response("Show me the top news from Hacker News")
```

### `@tool` パラメータのリファレンス

| パラメータ | タイプ | 説明 |
|-----------|------|-------------|
| `name` | `str` | 関数名のオーバーライド |
| `description` | `str` | 関数説明のオーバーライド |
| `show_result` | `bool` | True の場合、関数呼び出し後に結果を表示します |
| `stop_after_tool_call` | `bool` | True の場合、エージェントは関数呼び出し後に停止します |
| `tool_hooks` | `list[Callable]` | 関数実行をラップするフックのリスト |
| `pre_hook` | `Callable` | 関数実行前に実行するフック |
| `post_hook` | `Callable` | 関数実行後に実行するフック |
| `requires_confirmation` | `bool` | True の場合、実行前にユーザーの確認が必要です |
| `requires_user_input` | `bool` | True の場合、実行前にユーザーの入力が必要です |
| `user_input_fields` | `list[str]` | ユーザー入力が必要なフィールドのリスト |
| `external_execution` | `bool` | True の場合、ツールはエージェントの制御外で実行されます |
| `cache_results` | `bool` | True の場合、関数結果のキャッシュを有効にします |
| `cache_dir` | `str` | キャッシュファイルを格納するディレクトリ |
| `cache_ttl` | `int` | キャッシュ結果の有効期間 (秒単位、デフォルト: 3600) |
