## 工具钩子
您可以使用工具钩子在调用工具之前或之后执行验证、日志记录或任何其他逻辑。

工具钩子是一个函数，它接收函数名、函数调用和参数。在工具钩子内部，您必须调用函数调用并返回结果。

例如：

```python
def logger_hook(
    function_name: str, function_call: Callable, arguments: Dict[str, Any]
):
    """记录函数调用的持续时间"""
    start_time = time.time()

    # 调用函数
    result = function_call(**arguments)
    
    end_time = time.time()
    duration = end_time - start_time
    
    logger.info(f"函数 {function_name} 执行耗时 {duration:.2f} 秒")

    # 返回结果
    return result
```

或者

```python
def confirmation_hook(
    function_name: str, function_call: Callable, arguments: Dict[str, Any]
):
    """确认函数调用"""
    if function_name != "get_top_hackernews_stories":
        raise ValueError("不允许调用此工具")
    return function_call(**arguments)
```

您可以将工具钩子分配给代理和团队。工具钩子将应用于代理或团队中的所有工具。

例如：

```python
agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[DuckDuckGoTools()],
    tool_hooks=[logger_hook],
)
```

您也可以一次分配多个工具钩子。它们将按照分配的顺序应用。

```python
agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[DuckDuckGoTools()],
    tool_hooks=[logger_hook, confirmation_hook],  # logger_hook 将在外层运行，confirmation_hook 将在内层运行
)
```

您还可以将工具钩子分配给特定的自定义工具。

```python
@tool(tool_hooks=[logger_hook, confirmation_hook])
def get_top_hackernews_stories(num_stories: int) -> Iterator[str]:
    """从 Hacker News 获取热门故事。

    Args:
        num_stories (int): 要检索的故事数量
    """
    # 获取热门故事 ID
    response = httpx.get("https://hacker-news.firebaseio.com/v0/topstories.json")
    story_ids = response.json()

    # Yield 故事详情
    final_stories = []
    for story_id in story_ids[:num_stories]:
        story_response = httpx.get(
            f"https://hacker-news.firebaseio.com/v0/item/{story_id}.json"
        )
        story = story_response.json()
        if "text" in story:
            story.pop("text", None)
        final_stories.append(story)

    return json.dumps(final_stories)

agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[get_top_hackernews_stories],
)
```

## 前置和后置钩子

前置和后置钩子允许您修改在调用工具之前和之后发生的事情。它是工具钩子的替代方案。

设置 `@tool` 装饰器中的 `pre_hook` 以在调用工具之前运行函数。

设置 `@tool` 装饰器中的 `post_hook` 以在调用工具之后运行函数。

这是一个使用 `pre_hook`、`post_hook` 和 Agent Context 的演示示例。

```python pre_and_post_hooks.py
import json
from typing import Iterator

import httpx
from agno.agent import Agent
from agno.tools import FunctionCall, tool


def pre_hook(fc: FunctionCall):
    print(f"前置钩子：{fc.function.name}")
    print(f"参数：{fc.arguments}")
    print(f"结果：{fc.result}")


def post_hook(fc: FunctionCall):
    print(f"后置钩子：{fc.function.name}")
    print(f"参数：{fc.arguments}")
    print(f"结果：{fc.result}")


@tool(pre_hook=pre_hook, post_hook=post_hook)
def get_top_hackernews_stories(agent: Agent) -> Iterator[str]:
    num_stories = agent.context.get("num_stories", 5) if agent.context else 5

    # 获取热门故事 ID
    response = httpx.get("https://hacker-news.firebaseio.com/v0/topstories.json")
    story_ids = response.json()

    # Yield 故事详情
    for story_id in story_ids[:num_stories]:
        story_response = httpx.get(
            f"https://hacker-news.firebaseio.com/v0/item/{story_id}.json"
        )
        story = story_response.json()
        if "text" in story:
            story.pop("text", None)
        yield json.dumps(story)


agent = Agent(
    context={
        "num_stories": 2,
    },
    tools=[get_top_hackernews_stories],
    markdown=True,
    show_tool_calls=True,
)
agent.print_response("What are the top hackernews stories?", stream=True)
