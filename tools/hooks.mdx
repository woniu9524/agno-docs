## ツールフック
ツールフックを使用すると、ツールの呼び出し前後にバリデーション、ロギング、その他のロジックを実行できます。

ツールフックは、関数名、関数呼び出し、および引数を受け取る関数です。ツールフック内では、関数呼び出しを実行し、その結果を返す必要があります。

例：

```python
def logger_hook(
    function_name: str, function_call: Callable, arguments: Dict[str, Any]
):
    """関数呼び出しの時間を記録します"""
    start_time = time.time()

    # 関数を呼び出す
    result = function_call(**arguments)
    
    end_time = time.time()
    duration = end_time - start_time
    
    logger.info(f"Function {function_name} took {duration:.2f} seconds to execute")

    # 結果を返す
    return result
```

または

```python
def confirmation_hook(
    function_name: str, function_call: Callable, arguments: Dict[str, Any]
):
    """関数呼び出しを確認します"""
    if function_name != "get_top_hackernews_stories":
        raise ValueError("This tool is not allowed to be called")
    return function_call(**arguments)
```

エージェントやチームにツールフックを割り当てることができます。ツールフックは、エージェントまたはチーム内のすべてのツールに適用されます。

例：

```python
agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[DuckDuckGoTools()],
    tool_hooks=[logger_hook],
)
```

複数のツールフックを一度に割り当てることもできます。それらは割り当てられた順序で適用されます。

```python
agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[DuckDuckGoTools()],
    tool_hooks=[logger_hook, confirmation_hook],  # logger_hook は外側のレイヤーで実行され、confirmation_hook は内側のレイヤーで実行されます
)
```

特定のカスタムツールにツールフックを割り当てることもできます。

```python
@tool(tool_hooks=[logger_hook, confirmation_hook])
def get_top_hackernews_stories(num_stories: int) -> Iterator[str]:
    """Hacker News からトップストーリーを取得します。

    Args:
        num_stories (int): 取得するストーリーの数
    """
    # トップストーリーのIDを取得
    response = httpx.get("https://hacker-news.firebaseio.com/v0/topstories.json")
    story_ids = response.json()

    # ストーリーの詳細をyieldする
    final_stories = []
    for story_id in story_ids[:num_stories]:
        story_response = httpx.get(
            f"https://hacker-news.firebaseio.com/v0/item/{story_id}.json"
        )
        story = story_response.json()
        if "text" in story:
            story.pop("text", None)
        final_stories.append(story)

    return json.dumps(final_stories)

agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[get_top_hackernews_stories],
)
```

## 事前フックと事後フック

事前フックと事後フックを使用すると、ツールの呼び出し前後に何が起こるかを変更できます。これはツールフックの代替手段です。

ツールの呼び出し前に関数を実行するには、`@tool` デコレータで `pre_hook` を設定します。

ツールの呼び出し後に関数を実行するには、`@tool` デコレータで `post_hook` を設定します。

ここでは、`pre_hook`、`post_hook`、および Agent Context を使用するデモ例を示します。

```python pre_and_post_hooks.py
import json
from typing import Iterator

import httpx
from agno.agent import Agent
from agno.tools import FunctionCall, tool


def pre_hook(fc: FunctionCall):
    print(f"Pre-hook: {fc.function.name}")
    print(f"Arguments: {fc.arguments}")
    print(f"Result: {fc.result}")


def post_hook(fc: FunctionCall):
    print(f"Post-hook: {fc.function.name}")
    print(f"Arguments: {fc.arguments}")
    print(f"Result: {fc.result}")


@tool(pre_hook=pre_hook, post_hook=post_hook)
def get_top_hackernews_stories(agent: Agent) -> Iterator[str]:
    num_stories = agent.context.get("num_stories", 5) if agent.context else 5

    # トップストーリーのIDを取得
    response = httpx.get("https://hacker-news.firebaseio.com/v0/topstories.json")
    story_ids = response.json()

    # ストーリーの詳細をyieldする
    for story_id in story_ids[:num_stories]:
        story_response = httpx.get(
            f"https://hacker-news.firebaseio.com/v0/item/{story_id}.json"
        )
        story = story_response.json()
        if "text" in story:
            story.pop("text", None)
        yield json.dumps(story)


agent = Agent(
    context={
        "num_stories": 2,
    },
    tools=[get_top_hackernews_stories],
    markdown=True,
    show_tool_calls=True,
)
agent.print_response("What are the top hackernews stories?", stream=True)
