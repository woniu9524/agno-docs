启用一个代理来使用 Google Calendar 查看和安排会议。

## 先决条件

### 安装依赖项

```shell
pip install tzlocal
```

### 设置 Google 项目和 OAuth

参考：https://developers.google.com/calendar/api/quickstart/python

1. 启用 Google Calendar API

   - 前往 [Google Cloud Console](https://console.cloud.google.com/apis/enableflow?apiid=calendar-json.googleapis.com)。
   - 选择项目并启用。

2. 前往 API 和服务 -> OAuth 同意屏幕

3. 选择用户类型

   - 如果您是 Google Workspace 用户，请选择内部。
   - 否则，请选择外部。

4. 填写应用详情（应用名称、徽标、支持电子邮件等）。

5. 选择范围

   - 点击添加或移除范围。
   - 搜索 Google Calendar API（确保您已启用 Google Calendar API，否则范围将不可见）。
   - 根据需要选择范围
     - 在下拉列表中勾选 `/auth/calendar` 范围
   - 保存并继续。

6. 添加测试用户

   - 点击添加用户并输入您希望在测试期间允许的用户电子邮件地址。
   - 注意：当应用处于“测试”模式时，只有这些用户可以访问应用的 OAuth 功能。
     任何其他用户都会收到访问被拒绝的错误。
   - 要使应用可供所有用户使用，您需要将应用的状态更改为“生产中”。
     在此之前，请确保应用已通过 Google 的全面验证，特别是当它使用敏感或受限范围时。
   - 点击返回控制台。

7. 生成 OAuth 2.0 客户端 ID

   - 前往凭据。
   - 点击创建凭据 -> OAuth 客户端 ID
   - 选择应用程序类型为桌面应用。
   - 下载 JSON。

8. 使用 Google Calendar 工具
   - 将下载的凭据路径作为 `credentials_path` 传递给 Google Calendar 工具。
   - 可选：设置 `token_path` 参数以指定工具应创建 `token.json` 文件的位置。
   - `token.json` 文件用于存储用户的访问和刷新令牌，并在授权流程中自动创建（如果尚不存在）。
   - 如果未明确提供 `token_path`，则该文件将在默认位置创建，即您当前的目录。
   - 如果您选择指定 `token_path`，请确保您提供的目录具有写入权限，因为应用程序需要在身份验证过程中创建或更新此文件。

## 示例

以下代理将使用 GoogleCalendarTools 来查找今天的事件。

```python cookbook/tools/googlecalendar_tools.py
from agno.agent import Agent
from agno.tools.googlecalendar import GoogleCalendarTools
import datetime
import os
from tzlocal import get_localzone_name

agent = Agent(
    tools=[GoogleCalendarTools(credentials_path="<PATH_TO_YOUR_CREDENTIALS_FILE>")],
    show_tool_calls=True,
    instructions=[
        f"""
        您是日程安排助手。今天是 {datetime.datetime.now()}，用户的时区是 {get_localzone_name()}。
        您应该帮助用户在他们的 Google 日历中执行以下操作：
            - 根据提供的详细信息创建事件
            - 获取从特定日期和时间开始的已安排事件
        """
    ],
    add_datetime_to_instructions=True,
)

agent.print_response("Give me the list of todays events", markdown=True)
```

## 工具包参数

| 参数          | 类型  | 默认值 | 描述                                                              |
| ------------- | ----- | ------ | ----------------------------------------------------------------- |
| `credentials_path` | `str` | `None` | 包含 OAuth 2.0 客户端 ID 的 credentials.json 文件的路径。 |
| `token_path`       | `str` | `None` | 存储用户访问和刷新令牌的 token.json 文件的路径。        |

## 工具包函数

| 函数       | 描述                                     |
| ---------- | ---------------------------------------- |
| `list_events`  | 列出用户主日历中的事件。                   |
| `create_event` | 在用户主日历中创建新事件。                 |
