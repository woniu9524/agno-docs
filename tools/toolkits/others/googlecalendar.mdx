Google Calendar を操作して会議を表示およびスケジュール設定できるエージェントを有効にします。

## 前提条件

### 依存関係のインストール

```shell
pip install tzlocal
```

### Google プロジェクトと OAuth のセットアップ

参照: https://developers.google.com/calendar/api/quickstart/python

1. Google Calendar API を有効にする

   - [Google Cloud Console](https://console.cloud.google.com/apis/enableflow?apiid=calendar-json.googleapis.com) にアクセスします。
   - プロジェクトを選択して有効にします。

2. API とサービス -> OAuth 同意画面 に移動します。

3. ユーザーの種類を選択する

   - Google Workspace ユーザーの場合は、「内部」を選択します。
   - それ以外の場合は、「外部」を選択します。

4. アプリの詳細（アプリ名、ロゴ、サポートメールなど）を入力します。

5. スコープを選択する

   - 「スコープの追加または削除」をクリックします。
   - Google Calendar API を検索します（Google Calendar API を有効にしたことを確認してください。そうしないと、スコープが表示されません）。
   - スコープを適切に選択します。
     - ドロップダウンから `/auth/calendar` スコープにチェックを入れます。
   - 保存して続行します。

6. テストユーザーの追加

   - 「ユーザーを追加」をクリックし、テスト中に許可するユーザーのメールアドレスを入力します。
   - 注意: アプリが「テスト」モードの場合、これらのユーザーのみがアプリの OAuth 機能にアクセスできます。
     他のユーザーはアクセス拒否エラーを受け取ります。
   - アプリをすべてのユーザーが利用できるようにするには、アプリのステータスを「本番稼働中」に移行する必要があります。
     移行する前に、アプリが機密性の高いスコープまたは制限されたスコープを使用している場合は、Google による完全な検証が完了していることを確認してください。
   - 「ダッシュボードに戻る」をクリックします。

7. OAuth 2.0 クライアント ID を生成する

   - 「認証情報」に移動します。
   - 「認証情報を作成」->「OAuth クライアント ID」をクリックします。
   - アプリケーションの種類として「デスクトップ アプリ」を選択します。
   - JSON をダウンロードします。

8. Google Calendar ツールの使用
   - ダウンロードした認証情報へのパスを Google Calendar ツールに `credentials_path` として渡します。
   - オプション: `token_path` パラメータを設定して、ツールが `token.json` ファイルを作成する場所を指定します。
   - `token.json` ファイルは、ユーザーのアクセス トークンと更新トークンを保存するために使用され、存在しない場合は承認フロー中に自動的に作成されます。
   - `token_path` が明示的に指定されていない場合、ファイルはデフォルトの場所（現在の作業ディレクトリ）に作成されます。
   - `token_path` を指定する場合は、アプリケーションが認証プロセス中にこのファイルを作成または更新する必要があるため、指定したディレクトリに書き込みアクセス権があることを確認してください。

## 例

以下のエージェントは、GoogleCalendarTools を使用して今日のイベントを検索します。

```python cookbook/tools/googlecalendar_tools.py
from agno.agent import Agent
from agno.tools.googlecalendar import GoogleCalendarTools
import datetime
import os
from tzlocal import get_localzone_name

agent = Agent(
    tools=[GoogleCalendarTools(credentials_path="<PATH_TO_YOUR_CREDENTIALS_FILE>")],
    show_tool_calls=True,
    instructions=[
        f"""
        あなたはスケジュールアシスタントです。今日は {datetime.datetime.now()} で、ユーザーのタイムゾーンは {get_localzone_name()} です。
        Google カレンダーで次の操作を実行するのを支援する必要があります。
            - 特定の日時からの予定を取得する
            - 指定された詳細に基づいてイベントを作成する
        """
    ],
    add_datetime_to_instructions=True,
)

agent.print_response("Give me the list of todays events", markdown=True)
```

## Toolkit パラメータ

| パラメータ          | タイプ  | デフォルト | 説明                                                                    |
| ------------------ | ----- | ------- | ------------------------------------------------------------------------------ |
| `credentials_path` | `str` | `None`  | OAuth 2.0 クライアント ID を含む `credentials.json` ファイルへのパス。     |
| `token_path`       | `str` | `None`  | ユーザーのアクセス トークンと更新トークンを保存する `token.json` ファイルへのパス。 |

## Toolkit 関数

| 関数       | 説明                                        |
| -------------- | ------------------------------------------- |
| `list_events`  | ユーザーのプライマリ カレンダーからイベントを一覧表示します。 |
| `create_event` | ユーザーのプライマリ カレンダーに新しいイベントを作成します。 |
