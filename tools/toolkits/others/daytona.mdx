**Daytona** 为运行您 AI 生成的代码提供了安全且弹性的基础设施。在 Agno，我们与 Daytona 集成，使您的 Agents 和 Teams 能够在您的 Daytona 沙箱中运行代码。

## 先决条件

Daytona 工具需要 `daytona_sdk` Python 包：

```shell
pip install daytona_sdk
```

您还需要一个 Daytona API 密钥。您可以从您的 [Daytona 账户](https://app.daytona.io/account) 获取：

```shell
export DAYTONA_API_KEY=your_api_key
```

## 示例

以下示例演示了如何创建一个可以在 Daytona 沙箱中运行 Python 代码的 Agent：

```python cookbook/tools/daytona_tools.py
from agno.agent import Agent
from agno.models.anthropic import Claude
from agno.tools.daytona import DaytonaTools

daytona_tools = DaytonaTools()

# 设置一个专注于编码任务的 Agent，并允许访问 Daytona 工具
agent = Agent(
    name="Coding Agent with Daytona tools",
    agent_id="coding-agent",
    model=Claude(id="claude-sonnet-4-20250514"),
    tools=[daytona_tools],
    markdown=True,
    show_tool_calls=True,
    instructions=[
        "您是编写和验证 Python 代码的专家。您可以访问远程、安全的 Daytona 沙箱。",
        "您的主要目的是：",
        "1. 根据用户请求编写清晰、高效的 Python 代码",
        "2. 在 Daytona 沙箱中执行和验证代码",
        "3. 与用户共享完整的代码，因为这是主要用例",
        "4. 提供有关代码如何工作的详细解释",
        "您可以使用 run_python_code 工具在 Daytona 沙箱中运行 Python 代码。",
        "指南：",
        "- 始终与用户共享完整的代码，并正确格式化在代码块中",
        "- 在共享之前通过在沙箱中执行来验证代码功能",
        "- 根据需要迭代和调试代码以确保其正常工作",
        "- 在适当的时候使用 pandas、matplotlib 和其他 Python 库进行数据分析",
        "- 在被请求时创建适当的可视化，并将它们作为图像构件添加以显示为内联",
        "- 正确处理文件上传和下载",
        "- 详细解释您的方法和代码的功能",
        "- 使用代码和解释格式化响应以获得最大的清晰度",
        "- 优雅地处理错误并解释遇到的任何问题",
    ],
)

# 示例：生成斐波那契数列
agent.print_response(
    "编写 Python 代码以生成前 10 个斐波那契数并计算它们的总和与平均值"
)
```

## Toolkit 参数

| 参数            | 类型   | 默认值 | 描述                                        |
| --------------- | ------ | ------- | -------------------------------------------------- |
| `api_key`            | `str`  | `None`  | Daytona API 密钥。如果未提供，则使用 DAYTONA_API_KEY 环境变量。 |
| `api_url`            | `str`  | `None`  | Daytona API URL。如果未提供，则使用 DAYTONA_API_URL 环境变量或 Daytona 的默认值。 |
| `sandbox_language`   | `CodeLanguage`  | `Python`  | 在沙箱中运行的编程语言 |
| `sandbox_target_region` | `str` | `None`  | 将创建沙箱的区域 |
| `sandbox_os`         | `str`  | `None`  | 在沙箱中运行的操作系统（默认：ubuntu） |
| `sandbox_os_user`    | `str`  | `None`  | 在沙箱中运行的用户（默认：root） |
| `sandbox_env_vars`   | `dict` | `None`  | 要在沙箱中设置的环境变量 |
| `sandbox_labels`     | `dict` | `None`  | 要在沙箱中设置的标签 |
| `sandbox_public`     | `bool` | `None`  | 沙箱是否应公开 |
| `sandbox_auto_stop_interval` | `int` | `None`  | 沙箱将自动停止的间隔（秒） |
| `organization_id`    | `str`  | `None`  | 将用于沙箱的组织 ID |
| `timeout`            | `int`  | `300`   | 与沙箱通信的超时时间（秒）（默认：5 分钟） |

### 代码执行工具

| 函数           | 描述                                                  |
| -------------- | ----------------------------------------------------- |
| `run_python_code`  | 在上下文 Daytona 沙箱中运行 Python 代码 |
| `run_code`         | 在上下文 Daytona 沙箱中运行非 Python 代码 |
