アプリケーションの環境を構成するために、環境変数の設定が必要になる場合があります。このガイドでは、macOS (Shell) と Windows (PowerShell および Windows コマンド プロンプト) の両方で環境変数を設定する手順を説明します。

## macOS

### シェルでの環境変数の設定

#### 一時的な環境変数

これらの環境変数は、現在のシェルセッションでのみ利用可能です。

```shell
export VARIABLE_NAME="value"
```

環境変数を表示するには、次のようにします。

```shell
echo $VARIABLE_NAME
```

#### 永続的な環境変数

環境変数をセッション間で永続化するには、シェル設定ファイル (例: `.bashrc`、`.bash_profile`、`.zshrc`) に追加します。

Zsh の場合:

```shell
echo 'export VARIABLE_NAME="value"' >> ~/.zshrc
source ~/.zshrc
```

環境変数を表示するには、次のようにします。

```shell
echo $VARIABLE_NAME
```

## Windows

### PowerShell での環境変数の設定

#### 一時的な環境変数

これらの環境変数は、現在の PowerShell セッションでのみ利用可能です。

```powershell
$env:VARIABLE_NAME = "value"
```

環境変数を表示するには、次のようにします。

```powershell
echo $env:VARIABLE_NAME
```

#### 永続的な環境変数

環境変数をセッション間で永続化するには、PowerShell プロファイル スクリプト (例: `Microsoft.PowerShell_profile.ps1`) に追加します。

```powershell
notepad $PROFILE
```

プロファイル スクリプトに次の行を追加します。

```powershell
$env:VARIABLE_NAME = "value"
```

ファイルを保存して閉じ、プロファイルを再読み込みします。

```powershell
. $PROFILE
```

環境変数を表示するには、次のようにします。

```powershell
echo $env:VARIABLE_NAME
```

### Windows コマンド プロンプトでの環境変数の設定

#### 一時的な環境変数

これらの環境変数は、現在のコマンド プロンプト セッションでのみ利用可能です。

```cmd
set VARIABLE_NAME=value
```

環境変数を表示するには、次のようにします。

```cmd
echo %VARIABLE_NAME%
```

#### 永続的な環境変数

環境変数をセッション間で永続化するには、`setx` コマンドを使用できます。

```cmd
setx VARIABLE_NAME "value"
```

注: `setx` を使用して環境変数を設定した後、コマンド プロンプトまたは新しい環境変数を読み取る必要があるアプリケーションを再起動する必要があります。

新しいコマンド プロンプト セッションで環境変数を表示するには、次のようにします。

```cmd
echo %VARIABLE_NAME%
```

これらの手順に従うことで、macOS Shell、Windows コマンド プロンプト、および PowerShell で環境変数を効果的に設定および表示できます。これにより、アプリケーションの環境が正しく構成されるようになります。
