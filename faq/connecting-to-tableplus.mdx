pgvector コンテナを検査して、ストレージまたはナレッジベースを探索したい場合は、TablePlus を使用できます。次の手順に従ってください。

## ステップ 1: `pgvector` コンテナの起動

次のコマンドを実行して、`pgvector` コンテナをローカルで起動します。

```bash
docker run -d \
  -e POSTGRES_DB=ai \
  -e POSTGRES_USER=ai \
  -e POSTGRES_PASSWORD=ai \
  -e PGDATA=/var/lib/postgresql/data/pgdata \
  -v pgvolume:/var/lib/postgresql/data \
  -p 5532:5432 \
  --name pgvector \
  agno/pgvector:16
```

- `POSTGRES_DB=ai` はデフォルトのデータベース名を設定します。
- `POSTGRES_USER=ai` および `POSTGRES_PASSWORD=ai` はデータベースの認証情報を定義します。
- コンテナはポート `5432` を公開します (ローカルマシンでは `5532` にマッピングされます)。

## ステップ 2: TablePlus の設定

1. **TablePlus を開く**: TablePlus アプリケーションを起動します。
2. **新しい接続を作成する**: `+` アイコンをクリックして新しい接続を追加します。
3. **`PostgreSQL` を選択する**: データベースの種類として PostgreSQL を選択します。

次の接続詳細を入力します。

- **Host**: `localhost`
- **Port**: `5532`
- **Database**: `ai`
- **User**: `ai`
- **Password**: `ai`
