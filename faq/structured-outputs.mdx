## Structured Outputs vs. JSON Mode

言語モデルと連携する際、特定の構造に一致する応答を生成することは、信頼性の高いアプリケーションを構築するために不可欠です。Agno Agents は、この目標を達成するために **Structured Outputs** と **JSON mode** の 2 つの方法をサポートしています。

---

### Structured Outputs (サポートされている場合はデフォルト)

"Structured Outputs" は、モデルから整形され、スキーマに準拠した応答を抽出するための、**最も推奨される**、最も**信頼性の高い**方法です。モデルクラスがこれをサポートしている場合、Agno Agents はデフォルトで Structured Outputs を使用します。

Structured Outputs では、モデルにスキーマ（Pydantic または JSON Schema を使用）を提供し、モデルの応答はそのスキーマに**厳密に従う**ことが保証されます。これにより、フィールドの欠落、無効な enum 値、一貫性のないフォーマットといった多くの一般的な問題が解消されます。Structured Outputs は、エンティティ抽出、UI レンダリングのためのコンテンツ生成など、高い信頼性を持つ構造化された応答が必要な場合に最適です。

この場合、応答モデルはモデルのキーワード引数として渡されます。

## Example

```python
from pydantic import BaseModel
from agno.agent import Agent
from agno.models.openai import OpenAIChat

class User(BaseModel):
    name: str
    age: int
    email: str

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    description="You are a helpful assistant that can extract information from a user's profile.",
    response_model=User,
)
```

上記の例では、モデルは OpenAI の `gpt-4o` モデルを介して Structured Outputs を使用し、`User` スキーマに一致する応答を生成します。エージェントはその後、`User` オブジェクトをそのまま返します。

---

### JSON Mode

一部のモデルクラスは Structured Outputs を**サポートしていません**。あるいは、モデルが両方のオプションをサポートしている場合でも、JSON mode にフォールバックしたい場合があります。そのような場合は、`use_json_mode=True` を設定することで **JSON mode** を有効にできます。

JSON mode は、期待される JSON 構造の詳細な説明をシステムプロンプトに挿入することによって機能します。その後、モデルはこの構造に従った有効な JSON オブジェクトを返すように指示されます。Structured Outputs とは異なり、応答は API レベルでスキーマに対して**自動的に検証されません**。

## Example

```python
from pydantic import BaseModel
from agno.agent import Agent
from agno.models.openai import OpenAIChat

class User(BaseModel):
    name: str
    age: int
    email: str

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    description="You are a helpful assistant that can extract information from a user's profile.",
    response_model=User,
    use_json_mode=True,
)

```

### When to use

モデルがサポートしている場合は **Structured Outputs** を使用してください — これは信頼性が高く、クリーンで、自動的に検証されます。

**JSON mode** を使用する場合：

- モデルが structured outputs をサポートしていない場合。Agno agents はこれを自動的に処理します。
- より広範な互換性が必要で、手動での検証で問題ない場合。
- モデルが structured outputs を持つツールをサポートしていない場合。
