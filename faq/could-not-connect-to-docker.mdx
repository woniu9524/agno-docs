Dockerが起動しているにもかかわらず、以下のエラーが発生する場合は、以下をお読みください。

```bash
ERROR    Could not connect to docker. Please confirm docker is installed and running
ERROR    Error while fetching server API version: ('Connection aborted.', FileNotFoundError(2, 'No such file or directory'))
```

## クイックフィックス

以下のコマンドを使用して、`/var/run/docker.sock` のシンボリックリンクを作成してください。

```shell
sudo ln -s "$HOME/.docker/run/docker.sock" /var/run/docker.sock
```

ほとんどの場合、これで解決します。それでも解決しない場合は、以下を試してください。

```shell
sudo chown $USER /var/run/docker.sock
```

## 詳細

Agno はコンテナの実行に [docker-py](https://github.com/docker/docker-py) を使用しており、`/var/run/docker.sock` が存在しないか、パーミッションが正しくない場合、Docker に接続できません。

**修正するには、以下のコマンドを使用して `/var/run/docker.sock` ファイルを作成してください。**

```shell
sudo ln -s "$HOME/.docker/run/docker.sock" /var/run/docker.sock
```

それでも解決しない場合は、`ls -l /var/run/docker.sock` を使用してパーミッションを確認してください。

`/var/run/docker.sock` が存在しない場合は、`$HOME/.docker/run/docker.sock` ファイルが存在するか確認してください。もし存在しない場合は、Docker を再インストールしてください。

**上記の方法でも解決せず、`/var/run/docker.sock` が存在する場合:**

- `/var/run/docker.sock` ファイルに対するユーザーのパーミッションを付与してください。

```shell
sudo chown $USER /var/run/docker.sock
```

- Docker グループに対するユーザーのパーミッションを付与してください。

```shell
sudo usermod -a -G docker $USER
```
