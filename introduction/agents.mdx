従来のソフトウェアは、事前にプログラムされた一連のステップに従います。エージェントは、機械学習**モデル**を使用して動的に行動を決定します。そのコアコンポーネントは次のとおりです。

- **モデル:** 実行フローを制御します。推論、行動、または応答するかどうかを決定します。
- **ツール:** エージェントがアクションを実行し、外部システムと対話できるようにします。
- **指示:** エージェントをプログラムし、ツールの使用方法や応答方法を教える方法です。

エージェントはまた、**メモリ**、**ナレッジ**、**ストレージ**、および**推論**する能力を持っています。

- **推論:** エージェントが応答する前に「考え」、アクション（ツール呼び出しなど）の結果を「分析」できるようにします。これにより、応答の信頼性と品質が向上します。
- **ナレッジ:** エージェントが実行時に**検索**して、より良い意思決定を行い、正確な応答を提供できるドメイン固有の情報です（RAG）。ナレッジはベクトルデータベースに保存され、この**実行時の検索**パターンは、Agentic RAG/Agentic Searchとして知られています。
- **ストレージ:** エージェントがセッション履歴と状態をデータベースに保存するために使用されます。モデルAPIはステートレスであり、ストレージにより、中断したところから会話を続けることができます。これにより、エージェントはステートフルになり、マルチターン、長期的な会話が可能になります。
- **メモリ:** エージェントが以前の対話からの情報を保存および呼び出す能力を与え、ユーザーの好みを学習し、応答をパーソナライズできるようにします。

<Check>実際にいくつかエージェントを構築して、その仕組みを見てみましょう。</Check>

## レベル 1: ツールと指示を持つエージェント

最もシンプルなエージェントは、モデル、ツール、および指示を持っています。`yfinance`ライブラリを使用してデータを取得し、結果を表形式で表示する指示を持つエージェントを構築しましょう。

```python level_1_agent.py
from agno.agent import Agent
from agno.models.anthropic import Claude
from agno.tools.yfinance import YFinanceTools

agent = Agent(
    model=Claude(id="claude-sonnet-4-20250514"),
    tools=[YFinanceTools(stock_price=True)],
    instructions="Use tables to display data. Don't include any other text.",
    markdown=True,
)
agent.print_response("What is the stock price of Apple?", stream=True)
```

仮想環境を作成し、依存関係をインストールし、APIキーをエクスポートして、エージェントを実行します。

## レベル 2: ナレッジとストレージを持つエージェント

**ナレッジ:** モデルは大量のトレーニングデータを持っていますが、より良い意思決定を行い、正確な応答を提供するためには、ほぼ常にドメイン固有の情報を提供する必要があります（RAG）。この情報はベクトルデータベースに保存され、エージェントが実行時に**検索**できるようにします。

**ストレージ:** モデルAPIはステートレスであり、`Storage`ドライバーはチャット履歴と状態をデータベースに保存します。エージェントが実行されると、データベースからチャット履歴と状態を読み取り、メッセージリストに追加して、会話を再開し、エージェントをステートフルにします。

この例では、次を使用します。
- `UrlKnowledge` を使用して、OpenAIを埋め込みに使用し、AgnoドキュメントをLanceDBにロードします。
- `SqliteStorage` を使用して、エージェントのセッション履歴と状態をデータベースに保存します。

```python level_2_agent.py
from agno.agent import Agent
from agno.embedder.openai import OpenAIEmbedder
from agno.knowledge.url import UrlKnowledge
from agno.models.anthropic import Claude
from agno.storage.sqlite import SqliteStorage
from agno.vectordb.lancedb import LanceDb, SearchType

# ナレッジベースにAgnoドキュメントをロードします
# フルドキュメントには `https://docs.agno.com/llms-full.txt` も使用できます
knowledge = UrlKnowledge(
    urls=["https://docs.agno.com/introduction.md"],
    vector_db=LanceDb(
        uri="tmp/lancedb",
        table_name="agno_docs",
        search_type=SearchType.hybrid,
        # 埋め込みにはOpenAIを使用します
        embedder=OpenAIEmbedder(id="text-embedding-3-small", dimensions=1536),
    ),
)

# エージェントセッションをSQLiteデータベースに保存します
storage = SqliteStorage(table_name="agent_sessions", db_file="tmp/agent.db")

agent = Agent(
    name="Agno Assist",
    model=Claude(id="claude-sonnet-4-20250514"),
    instructions=[
        "Search your knowledge before answering the question.",
        "Only include the output in your response. No other text.",
    ],
    knowledge=knowledge,
    storage=storage,
    add_datetime_to_instructions=True,
    # チャット履歴をメッセージに追加します
    add_history_to_messages=True,
    # ヒストリ実行数
    num_history_runs=3,
    markdown=True,
)

if __name__ == "__main__":
    # ナレッジベースをロードします。初回実行後はコメントアウトしてください
    # 必要に応じてナレッジベースを再作成するには、recreateをTrueに設定します
    agent.knowledge.load(recreate=False)
    agent.print_response("What is Agno?", stream=True)
```

依存関係をインストールし、`OPENAI_API_KEY`をエクスポートして、エージェントを実行します。

    ```

    ```
    </CodeGroup>

  </Step>
  <Step title="エージェントの実行">

    ```shell

    ```

  </Step>
</Steps>

## レベル 3: メモリと推論を持つエージェント

- **推論:** エージェントが**「考え」＆「分析」**できるようにし、信頼性と品質を向上させます。`ReasoningTools`は、エージェントの応答品質を向上させるための最良のアプローチの1つです。
- **メモリ:** エージェントがユーザーの好みを分類、保存、呼び出し、応答をパーソナライズできるようにします。メモリは、エージェントがペルソナを構築し、以前の対話から学習するのに役立ちます。

```python level_3_agent.py
from agno.agent import Agent
from agno.memory.v2.db.sqlite import SqliteMemoryDb
from agno.memory.v2.memory import Memory
from agno.models.anthropic import Claude
from agno.tools.reasoning import ReasoningTools
from agno.tools.yfinance import YFinanceTools

memory = Memory(
    # メモリの作成と管理には任意のモデルを使用します
    model=Claude(id="claude-sonnet-4-20250514"),
    # メモリをSQLiteデータベースに保存します
    db=SqliteMemoryDb(table_name="user_memories", db_file="tmp/agent.db"),
    # デフォルトでは削除を無効にしています。必要に応じて有効にしてください
    delete_memories=True,
    clear_memories=True,
)

agent = Agent(
    model=Claude(id="claude-sonnet-4-20250514"),
    tools=[
        ReasoningTools(add_instructions=True),
        YFinanceTools(stock_price=True, analyst_recommendations=True, company_info=True, company_news=True),
    ],
    # メモリを保存するためのユーザーID。指定しない場合は`default`
    user_id="ava",
    instructions=[
        "Use tables to display data.",
        "Include sources in your response.",
        "Only include the report in your response. No other text.",
    ],
    memory=memory,
    # エージェントにメモリを管理させます
    enable_agentic_memory=True,
    markdown=True,
)

if __name__ == "__main__":
    # これにより、「ava」のお気に入りの株はNVIDIAとTSLAであるというメモリが作成されます
    agent.print_response(
        "My favorite stocks are NVIDIA and TSLA",
        stream=True,
        show_full_reasoning=True,
        stream_intermediate_steps=True,
    )
    # これにより、メモリを使用して質問に回答します
    agent.print_response(
        "Can you compare my favorite stocks?",
        stream=True,
        show_full_reasoning=True,
        stream_intermediate_steps=True,
    )
```

エージェントを実行します。

```shell
python level_3_agent.py
```

<Tip>`Memory`と`Reasoning`は別々に使用できます。一緒に使用する必要はありません。</Tip>
