AG-UI、または [Agent-User Interaction Protocol](https://github.com/ag-ui-protocol/ag-ui) は、AI エージェントがフロントエンドアプリケーションに接続する方法を標準化するプロトコルです。


## 使用例

<Steps>
  <Step title="バックエンドの依存関係をインストールする">
    ```bash
    pip install agno ag-ui-protocol
    ```
  </Step>

  <Step title="バックエンドを実行する">
    次に、Agno エージェントを公開する `AGUIApp` を実行しましょう。前のコードを使用できます！
  </Step>

  <Step title="フロントエンドを実行する">


AG-UI エージェントのフロントエンドとして使用できる、高度でカスタマイズ可能なオプションである [Dojo](https://github.com/ag-ui-protocol/ag-ui/tree/main/typescript-sdk/apps/dojo) を使用できます。

1. プロジェクトをクローンします: `git clone https://github.com/ag-ui-protocol/ag-ui.git`
2. [こちら](https://github.com/ag-ui-protocol/ag-ui/tree/main/typescript-sdk/apps/dojo) の指示に従って、必要な依存関係をインストールし、プロジェクトを実行する方法を学びます。
3. `/ag-ui/typescript-sdk` で依存関係を `pnpm install` でインストールし、`/integrations/agno` で Agno パッケージを `pnpm run build` でビルドすることを忘れないでください。
4. これで Dojo を実行できます！利用可能なオプションの 1 つとして Agno エージェントが表示されます。
</Step>

<Step title="Agno エージェントとチャットする">

完了です！Dojo をフロントエンドとして実行している場合は、ブラウザで http://localhost:3000 にアクセスして、Agno エージェントとチャットできます。

</Step>
</Steps>

その他の例については、[AG-UI 統合例](/examples/applications/ag-ui) セクションをご覧ください。

## コアコンポーネント

- `AGUIApp`: Agno エージェント/チームを FastAPI アプリにラップします。
- `serve`: Uvicorn を使用して FastAPI AG-UI アプリをサーブします。

`AGUIApp` はルーティングのためのヘルパー関数を使用します。

## `AGUIApp` クラス

Agno AG-UI アプリケーションのエントリポイントです。

### 初期化パラメータ

| パラメータ  | タイプ                      | デフォルト | 説明                                           |
| ---------- | ------------------------- | ------- | ----------------------------------------------------- |
| `agent`    | `Optional[Agent]`         | `None`  | Agno `Agent` インスタンス。                                |
| `team`     | `Optional[Team]`          | `None`  | Agno `Team` インスタンス。                                 |
| `settings` | `Optional[APIAppSettings]`| `None`  | API 設定。`None` の場合はデフォルト値を使用します。                |
| `api_app`  | `Optional[FastAPI]`       | `None`  | 既存の FastAPI アプリ。`None` の場合は新規作成されます。        |
| `router`   | `Optional[APIRouter]`     | `None`  | 既存の APIRouter。`None` の場合は新規作成されます。        |
| `app_id`    | `Optional[str]`           | `None`  | アプリの識別子（設定されていない場合は自動生成されます）。                         |
| `name`    | `Optional[str]`           | `None`  | アプリの名前。                         |
| `description`    | `Optional[str]`           | `None`  | アプリの説明。                         |

*`agent` または `team` のいずれかを提供してください。両方ではありません。*

### 主要メソッド

| メソッド | パラメータ | 戻り値の型 | 説明 |
|--------|------------|-------------|-------------|
| `get_app` | `use_async: bool = True` | `FastAPI` | 設定済みの FastAPI アプリを返します（デフォルトは非同期）。プレフィックス、エラーハンドラ、CORS、ドキュメントを設定します。 |


## エンドポイント

エンドポイントは指定された `prefix`（デフォルトは `/v1`）で利用可能です。

### 1. `POST /agui`

これは Agno エージェントまたはチームと対話するためのメインのエントリポイントです。

プロトコルで定義されている `RunAgentInput` オブジェクト（`ag-ui-protocol` パッケージから）を期待します。詳細については、[ドキュメント](https://docs.ag-ui.com/quickstart/server) を参照してください。


## アプリケーションのサーブ (`serve`)

Uvicorn を使用して FastAPI アプリをサーブします。

### パラメータ

| パラメータ | タイプ                | デフォルト     | 説明                                        |
| --------- | ------------------- | ----------- | -------------------------------------------------- |
| `app`     | `Union[str, FastAPI]` | `N/A`       | FastAPI アプリインスタンスまたはインポート文字列（必須）。 |
| `host`    | `str`               | `"localhost"` | バインドするホスト。                                      |
| `port`    | `int`               | `7777`      | バインドするポート。                                      |
| `reload`  | `bool`              | `False`     | 開発用に自動リロードを有効にします。                |


使用例については、[AG-UI 統合例](/examples/applications/ag-ui) セクションで確認できます。

また、[CopilotKit ドキュメント](https://docs.copilotkit.ai/agno) で Agno との連携について確認し、UI サイドの構築方法をさらに学ぶこともできます。
