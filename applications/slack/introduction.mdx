Slack アプリは、FastAPI サーバーを使用して Slack イベントを処理し、メッセージを送信することで、Slack 経由でエージェントまたはチームを提供するために使用されます。


## セットアップ手順

<Snippet file="setup-slack-app.mdx" />

### 使用例

エージェントを作成し、`SlackAPI` でラップして提供します。

```python
from agno.agent import Agent
from agno.app.slack.app import SlackAPI
from agno.models.openai import OpenAIChat

basic_agent = Agent(
    name="Basic Agent",
    model=OpenAIChat(id="gpt-4o"), # OPENAI_API_KEY が設定されていることを確認してください
    add_history_to_messages=True,
    num_history_responses=3,
    add_datetime_to_instructions=True,
)

slack_api_app = SlackAPI(
    agent=basic_agent,
)
app = slack_api_app.get_app()

if __name__ == "__main__":
    slack_api_app.serve("basic:app", port=8000, reload=True)
```

## コアコンポーネント

- `SlackAPI`: FastAPI を介した Slack 統合のために Agno エージェント/チームをラップします。
- `SlackAPI.serve`: Uvicorn を使用して FastAPI アプリをサーブします。Slack 用に設定されています。

## `SlackAPI` クラス

Agno Slack アプリケーションのエントリポイントです。

### 初期化パラメータ

| パラメータ  | タイプ                      | デフォルト | 説明                                           |
| ---------- | ------------------------- | ------- | ----------------------------------------------------- |
| `agent`    | `Optional[Agent]`         | `None`  | Agno `Agent` インスタンス。                                |
| `team`     | `Optional[Team]`          | `None`  | Agno `Team` インスタンス。                                 |
| `settings` | `Optional[APIAppSettings]`| `None`  | API 設定。`None` の場合はデフォルト値が使用されます。                |
| `api_app`  | `Optional[FastAPI]`       | `None`  | 既存の FastAPI アプリ。`None` の場合は新規作成されます。 |
| `router`   | `Optional[APIRouter]`     | `None`  | 既存の APIRouter。`None` の場合は新規作成されます。    |
| `app_id`   | `Optional[str]`           | `None`  | アプリ識別子（設定されていない場合は自動生成されます）。             |
| `name`     | `Optional[str]`           | `None`  | アプリの名前。                                     |
| `description` | `Optional[str]`         | `None`  | アプリの説明。                                     |

*`agent` または `team` のいずれかを提供してください。両方は提供しないでください。*

## エンドポイント

Slack 統合のメインエンドポイントです。

### `POST /slack/events`
- **説明**: メッセージやアプリメンションを含む、すべての Slack イベントを処理します。
- **セキュリティ**: 各リクエストの Slack 署名を検証します。
- **イベントタイプ**:
  - URL 検証チャレンジ
  - メッセージイベント
  - アプリメンションイベント
- **機能**:
  - スレッド会話
  - バックグラウンドタスク処理
  - 長い応答のためのメッセージ分割
  - ダイレクトメッセージとチャンネルインタラクションの両方のサポート

## 統合のテスト

1. アプリケーションをローカルで `python <my-app>.py` で起動します（ngrok が実行されていることを確認してください）。
2. `/invite @YourAppName` を使用してボットをチャンネルに招待します。
3. チャンネルでボットにメンションしてみてください: `@YourAppName hello`
4. ボットとの DM を開いてダイレクトメッセージをテストします。

## トラブルシューティング

- すべての環境変数が正しく設定されていることを確認してください。
- ボットに必要な権限があり、チャンネルに招待されていることを確認してください。
- ngrok 接続と URL 設定を確認してください。
- イベントサブスクリプションが正しく設定されていることを確認してください。
- 詳細なエラーメッセージについては、アプリケーションログを監視してください。
