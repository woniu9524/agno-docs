**ワークフローは、制御と柔軟性に関するものです。**

ワークフローロジックは単なるPython関数であるため、ワークフローロジックを完全に制御できます。以下のことが可能です。
- 処理前にインプットを検証する
- インプットに応じて、エージェントを生成し、並列で実行する
- 必要に応じて結果をキャッシュする
- 中間エラーを修正する
- 出力をストリーミングする
- 単一または複数の出力を返す

**このレベルの制御は、信頼性にとって不可欠です。**


## ストリーミング

ワークフローを構築する際は、Python関数を作成していることを理解することが重要です。つまり、関数が出力をストリーミングするかどうかを決定するのはあなたです。出力をストリーミングするには、ワークフローの`run()`メソッドから`Iterator[RunResponse]`をyieldします。

```python news_report_generator.py
# ワークフローを定義する
class GenerateNewsReport(Workflow):
    agent_1: Agent = ...

    agent_2: Agent = ...

    agent_3: Agent = ...

    def run(self, ...) -> Iterator[RunResponse]:
        # エージェントを実行し、レスポンスを収集する
        # これらはバッチレスポンスでも構いません。中間結果をストリーミングすることも可能です。
        final_agent_input = ...

        # ライターエージェントから最終レスポンスを生成する
        agent_3_response_stream: Iterator[RunResponse] = self.agent_3.run(final_agent_input, stream=True)

        # レスポンスをyieldする
        yield agent_3_response_stream

# ワークフローをインスタンス化する
generate_news_report = GenerateNewsReport()

# ワークフローを実行し、レスポンスをRunResponseオブジェクトのイテレータとして取得する
report_stream: Iterator[RunResponse] = generate_news_report.run(...)

# レスポンスを出力する
pprint_run_response(report_stream, markdown=True)
```

## バッチ

ワークフローの`run()`メソッドから`RunResponse`オブジェクトを返すだけで、単一の出力を返すことができます。

```python news_report_generator.py
# ワークフローを定義する
class GenerateNewsReport(Workflow):
    agent_1: Agent = ...

    agent_2: Agent = ...

    agent_3: Agent = ...

    def run(self, ...) -> RunResponse:
        # エージェントを実行し、レスポンスを収集する
        final_agent_input = ...

        # ライターエージェントから最終レスポンスを生成する
        agent_3_response: RunResponse = self.agent_3.run(final_agent_input)

        # レスポンスを返す
        return agent_3_response

# ワークフローをインスタンス化する
generate_news_report = GenerateNewsReport()

# ワークフローを実行し、レスポンスをRunResponseオブジェクトとして取得する
report: RunResponse = generate_news_report.run(...)

# レスポンスを出力する
pprint_run_response(report, markdown=True)
