---
title: ワークフローの状態
---

すべてのワークフローには、中間結果をキャッシュするために使用できる `session_state` 辞書が付属しています。`session_state` は `session_id` に紐付けられており、データベースに永続化できます。

ワークフローに `storage` を提供することで、データベースでのセッション状態の永続化を有効にできます。

たとえば、`SqliteWorkflowStorage` を使用して、Sqlite データベースに結果をキャッシュできます。

```python
# ワークフローを作成
generate_blog_post = BlogPostGenerator(
    # このデモのために session_id を固定
    session_id="my-session-id",
    storage=SqliteWorkflowStorage(
        table_name="generate_blog_post_workflows",
        db_file="tmp/workflows.db",
    ),
)
```

次に、`run()` メソッド内で、必要に応じて `session_state` から読み取ったり、追加したりできます。

```python

class BlogPostGenerator(Workflow):
    # ... エージェント
    def run(self, topic: str, use_cache: bool = True) -> Iterator[RunResponse]:
        # セッション状態キャッシュから読み取る
        if use_cache and "blog_posts" in self.session_state:
            logger.info("キャッシュされたブログ投稿が存在するか確認しています")
            for cached_blog_post in self.session_state["blog_posts"]:
                if cached_blog_post["topic"] == topic:
                    logger.info("キャッシュされたブログ投稿が見つかりました")
                    yield RunResponse(
                        run_id=self.run_id,
                        event=RunEvent.workflow_completed,
                        content=cached_blog_post["blog_post"],
                    )
                    return

        # ... ブログ投稿を生成

        # 将来の実行のためにセッション状態に保存
        if "blog_posts" not in self.session_state:
            self.session_state["blog_posts"] = []
        self.session_state["blog_posts"].append({"topic": topic, "blog_post": self.writer.run_response.content})
```

ワークフローが開始されると、その特定の `session_id` の `session_state` がデータベースから読み取られ、ワークフローが終了すると、`session_state` がデータベースに保存されます。

<Tip>

いつでも `self.write_to_storage()` を呼び出して、`session_state` をデータベースに保存できます。ワークフローを中止する必要があるが、中間結果を保存したい場合などに便利です。

</Tip>

セッション状態をキャッシュに使用する方法の例については、[ブログ投稿ジェネレーター](/workflows/introduction#full-example-blog-post-generator) を参照してください。
