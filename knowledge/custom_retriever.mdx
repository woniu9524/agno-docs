場合によっては、エージェントがナレッジベースから情報を取得する方法について、完全な制御が必要になることがあります。これは、カスタムリトリーバー関数を実装することで実現できます。カスタムリトリーバーを使用すると、ベクトルデータベースからドキュメントを検索および取得するためのロジックを定義できます。

## セットアップ

Qdrantをローカルにインストールするには、[Qdrantセットアップガイド](https://qdrant.tech/documentation/guides/installation/)の手順に従ってください。APIキーを取得するためのガイドは次のとおりです。[Qdrant APIキー](https://qdrant.tech/documentation/cloud/authentication/)。

### 例：PDFナレッジベース用のカスタムリトリーバー

以下は、`agno`ライブラリを使用してカスタムリトリーバー関数を実装する方法の詳細な例です。この例では、PDFドキュメントでナレッジベースをセットアップし、カスタムリトリーバーを定義して、エージェントで使用する方法を示しています。

```python
from typing import Optional
from agno.agent import Agent
from agno.embedder.openai import OpenAIEmbedder
from agno.knowledge.pdf_url import PDFUrlKnowledgeBase
from agno.vectordb.qdrant import Qdrant
from qdrant_client import QdrantClient

# ---------------------------------------------------------
# このセクションはナレッジベースをロードします。ナレッジベースが他の場所で
# ポピュレートされた場合はスキップしてください。
# エンベッダーを定義します
embedder = OpenAIEmbedder(id="text-embedding-3-small")
# ベクトルデータベース接続を初期化します
vector_db = Qdrant(collection="thai-recipes", url="http://localhost:6333", embedder=embedder)
# ナレッジベースをロードします
knowledge_base = PDFUrlKnowledgeBase(
    urls=["https://agno-public.s3.amazonaws.com/recipes/ThaiRecipes.pdf"],
    vector_db=vector_db,
)

# ナレッジベースをロードします
knowledge_base.load(recreate=True)  # 初回実行後にコメントアウトしてください
# ナレッジベースはロードされました
# ---------------------------------------------------------

# カスタムリトリーバーを定義します
# これは、エージェントがドキュメントを取得するために使用する関数です
def retriever(
    query: str, agent: Optional[Agent] = None, num_documents: int = 5, **kwargs
) -> Optional[list[dict]]:
    """
    ベクトルデータベースで関連ドキュメントを検索するためのカスタムリトリーバー関数。

    Args:
        query (str): 検索クエリ文字列
        agent (Agent): クエリを実行しているエージェントインスタンス
        num_documents (int): 取得するドキュメント数 (デフォルト: 5)
        **kwargs: 追加のキーワード引数

    Returns:
        Optional[list[dict]]: 取得されたドキュメントのリスト、または検索が失敗した場合はNone
    """
    try:
        qdrant_client = QdrantClient(url="http://localhost:6333")
        query_embedding = embedder.get_embedding(query)
        results = qdrant_client.query_points(
            collection_name="thai-recipes",
            query=query_embedding,
            limit=num_documents,
        )
        results_dict = results.model_dump()
        if "points" in results_dict:
            return results_dict["points"]
        else:
            return None
    except Exception as e:
        print(f"ベクトルデータベース検索中のエラー: {str(e)}")
        return None

def main():
    """エージェントの使用法を示すメイン関数。"""
    # カスタムリトリーバーでエージェントを初期化します
    # agentic_ragを使用するにはsearch_knowledge=Trueを設定するか、
    # 従来のRAGにはadd_reference=Trueを追加することを忘れないでください。
    # search_knowledge=Trueはナレッジベースを追加するとデフォルトになりますが、
    # ここでは必要です。
    agent = Agent(
        retriever=retriever,
        search_knowledge=True,
        instructions="ナレッジベースで情報を検索してください",
        show_tool_calls=True,
    )

    # クエリ例
    query = "List down the ingredients to make Massaman Gai"
    agent.print_response(query, markdown=True)

if __name__ == "__main__":
    main()
```

#### 非同期実装

```python
import asyncio
from typing import Optional
from agno.agent import Agent
from agno.embedder.openai import OpenAIEmbedder
from agno.knowledge.pdf_url import PDFUrlKnowledgeBase
from agno.vectordb.qdrant import Qdrant
from qdrant_client import AsyncQdrantClient

# ---------------------------------------------------------
# ナレッジベースのセットアップ (同期例と同じ)
embedder = OpenAIEmbedder(id="text-embedding-3-small")
vector_db = Qdrant(collection="thai-recipes", url="http://localhost:6333", embedder=embedder)
knowledge_base = PDFUrlKnowledgeBase(
    urls=["https://agno-public.s3.amazonaws.com/recipes/ThaiRecipes.pdf"],
    vector_db=vector_db,
)
# ---------------------------------------------------------

# カスタム非同期リトリーバーを定義します
async def retriever(
    query: str, agent: Optional[Agent] = None, num_documents: int = 5, **kwargs
) -> Optional[list[dict]]:
    """
    ベクトルデータベースで関連ドキュメントを検索するためのカスタム非同期リトリーバー関数。
    """
    try:
        qdrant_client = AsyncQdrantClient(path="tmp/qdrant")
        query_embedding = embedder.get_embedding(query)
        results = await qdrant_client.query_points(
            collection_name="thai-recipes",
            query=query_embedding,
            limit=num_documents,
        )
        results_dict = results.model_dump()
        if "points" in results_dict:
            return results_dict["points"]
        else:
            return None
    except Exception as e:
        print(f"ベクトルデータベース検索中のエラー: {str(e)}")
        return None

async def main():
    """エージェントの使用法を示す非同期メイン関数。"""
    agent = Agent(
        retriever=retriever,
        search_knowledge=True,
        instructions="ナレッジベースで情報を検索してください",
        show_tool_calls=True,
    )

    # ナレッジベースをロードします (初回実行時にコメント解除してください)
    await knowledge_base.aload(recreate=True)

    # クエリ例
    query = "List down the ingredients to make Massaman Gai"
    await agent.aprint_response(query, markdown=True)

if __name__ == "__main__":
    asyncio.run(main())
```

### 説明

1.  **エンベッダーとベクトルデータベースのセットアップ**: エンベッダーを定義し、ベクトルデータベースへの接続を初期化することから始めます。このセットアップは、クエリをエンベディングに変換し、それらをデータベースに保存するために不可欠です。

2.  **ナレッジベースのロード**: PDFドキュメントでナレッジベースがロードされます。このステップには、ドキュメントをエンベディングに変換し、それらをベクトルデータベースに保存することが含まれます。

3.  **カスタムリトリーバー関数**: `retriever`関数は、ドキュメントの取得を処理するために定義されます。クエリを受け取り、それをエンベディングに変換し、関連ドキュメントをベクトルデータベースで検索します。

4.  **エージェントの初期化**: カスタムリトリーバーでエージェントが初期化されます。エージェントはこのリトリーバーを使用してナレッジベースを検索し、情報を取得します。

5.  **クエリ例**: `main`関数は、エージェントを使用してクエリを実行し、ナレッジベースから情報を取得する方法を示しています。
