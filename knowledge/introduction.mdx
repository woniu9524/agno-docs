知识存储在向量数据库中，这种按需搜索模式称为 Agentic RAG。

<Accordion title="动态少样本学习：Text2Sql Agent" icon="database">
示例：如果我们正在构建一个 Text2Sql Agent，我们需要提供表模式、列名、数据类型、示例查询以及常见的“陷阱”，以帮助它生成最佳的 SQL 查询。

我们显然不会将所有这些都放入系统提示中，而是将这些信息存储在向量数据库中，让 Agent 在运行时查询它。

利用这些信息，Agent 可以生成最佳的 SQL 查询。这就是所谓的动态少样本学习。
</Accordion>

**Agno Agents 默认使用 Agentic RAG**，这意味着如果你向 Agent 添加 `knowledge`，它将在运行时搜索该知识库，以获取完成任务所需的特定信息。

将知识添加到 Agent 的伪步骤如下：

```python
from agno.agent import Agent, AgentKnowledge

# 为 Agent 创建一个知识库
knowledge_base = AgentKnowledge(vector_db=...)

# 向知识库添加信息
knowledge_base.load_text("天空是蓝色的")

# 将知识库添加到 Agent，并
# 提供一个工具让它在需要时搜索知识库
agent = Agent(knowledge=knowledge_base, search_knowledge=True)
```

我们可以通过以下方式让我们的 Agent 访问知识库：

- 我们可以设置 `search_knowledge=True` 来向 Agent 添加一个 `search_knowledge_base()` 工具。如果你向 Agent 添加 `knowledge`，`search_knowledge` **默认**为 `True`。
- 我们可以设置 `add_references=True` 来自动将知识库中的引用添加到 Agent 的提示中。这是传统的 2023 RAG 方法。

<Tip>

如果你需要完全控制知识库的搜索，可以传递一个具有以下签名的自定义 `retriever` 函数：

```python
def retriever(agent: Agent, query: str, num_documents: Optional[int], **kwargs) -> Optional[list[dict]]:
  ...
```

在调用 `search_knowledge_base()` 时会调用此函数，Agent 使用它从知识库中检索引用。
有关更多详细信息，请参阅 [自定义检索器](../knowledge/custom_retriever) 页面。

</Tip>

## 向量数据库

虽然任何类型的存储都可以作为知识库，但向量数据库提供了从密集信息中快速检索相关结果的最佳解决方案。以下是向量数据库与 Agent 的使用方式：

<Steps>
  <Step title="分块信息">
    将知识分解成更小的块，以确保我们的搜索查询
    只返回相关结果。
  </Step>
  <Step title="加载知识库">
    将块转换为嵌入向量并存储在向量数据库中。
  </Step>
  <Step title="搜索知识库">
    当用户发送消息时，我们将输入消息转换为嵌入，并在向量数据库中“搜索”最近邻。
  </Step>
</Steps>

## 加载知识库

在使用知识库之前，需要加载用于检索的嵌入。

### 异步加载

许多向量数据库支持异步操作，这可以显著提高加载大型知识库时的性能。你可以使用 `aload()` 方法来利用此功能：

```python
import asyncio

from agno.agent import Agent
from agno.knowledge.pdf import PDFKnowledgeBase, PDFReader
from agno.vectordb.qdrant import Qdrant

COLLECTION_NAME = "pdf-reader"

vector_db = Qdrant(collection=COLLECTION_NAME, url="http://localhost:6333")

# 使用 data/pdfs 目录中的 PDF 创建知识库
knowledge_base = PDFKnowledgeBase(
    path="data/pdf",
    vector_db=vector_db,
    reader=PDFReader(chunk=True),
)

# 创建一个带有知识库的 Agent
agent = Agent(
    knowledge=knowledge_base,
    search_knowledge=True,
)

if __name__ == "__main__":
    # 首次运行时注释掉
    asyncio.run(knowledge_base.aload(recreate=False))

    # 创建并使用 Agent
    asyncio.run(agent.aprint_response("How to make Thai curry?", markdown=True))
```

使用 `aload()` 可以充分利用非阻塞操作、并发处理以及异步向量数据库操作提供的低延迟。这在高吞吐量要求的生产环境中尤其有价值。

有关向量数据库异步功能的更多详细信息，请参阅 [向量数据库简介](/vectordb/introduction)。
