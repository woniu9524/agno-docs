Knowledgeはベクトルデータベースに保存され、このオンデマンド検索パターンはAgentic RAGと呼ばれます。

<Accordion title="動的な少数ショット学習：Text2Sqlエージェント" icon="database">
例：Text2Sqlエージェントを構築する場合、テーブルスキーマ、カラム名、データ型、クエリ例、一般的な「落とし穴」を提供して、可能な限り最良のSQLクエリを生成できるようにする必要があります。

これらすべてをシステムプロンプトに入れることは明らかにできないため、代わりにこの情報をベクトルデータベースに保存し、エージェントが実行時にクエリできるようにします。

この情報を使用して、エージェントは可能な限り最良のSQLクエリを生成できます。これは動的な少数ショット学習と呼ばれます。
</Accordion>

**AgnoエージェントはデフォルトでAgentic RAGを使用します**。これは、エージェントに`knowledge`を追加すると、実行時にそのタスクを達成するために必要な特定の情報を検索することを意味します。

エージェントに知識を追加するための疑似ステップは次のとおりです。

```python
from agno.agent import Agent, AgentKnowledge

# エージェント用の知識ベースを作成します
knowledge_base = AgentKnowledge(vector_db=...)

# 知識ベースに情報を追加します
knowledge_base.load_text("The sky is blue")

# エージェントに知識ベースを追加し、
# 必要に応じて知識ベースを検索するためのツールを提供します
agent = Agent(knowledge=knowledge_base, search_knowledge=True)
```

エージェントに知識ベースへのアクセスを次の方法で提供できます。

- `search_knowledge=True`を設定すると、エージェントに`search_knowledge_base()`ツールが追加されます。エージェントに`knowledge`を追加すると、`search_knowledge`はデフォルトで`True`になります。
- `add_references=True`を設定すると、知識ベースからの参照が自動的にエージェントのプロンプトに追加されます。これは従来の2023 RAGアプローチです。

<Tip>

知識ベースの検索を完全に制御する必要がある場合は、次のシグネチャを持つ独自の`retriever`関数を渡すことができます。

```python
def retriever(agent: Agent, query: str, num_documents: Optional[int], **kwargs) -> Optional[list[dict]]:
  ...
```

この関数は`search_knowledge_base()`中に呼び出され、エージェントが知識ベースから参照を取得するために使用されます。
詳細については、[カスタムリトリーバー](../knowledge/custom_retriever)ページをご覧ください。

</Tip>

## ベクトルデータベース

あらゆる種類のストレージが知識ベースとして機能できますが、ベクトルデータベースは、密な情報から関連性の高い結果を迅速に取得するための最良のソリューションを提供します。ベクトルデータベースとエージェントの使用方法は次のとおりです。

<Steps>
  <Step title="情報をチャンク化する">
    検索クエリが関連性の高い結果のみを返すように、知識をより小さなチャンクに分割します。
  </Step>
  <Step title="知識ベースをロードする">
    チャンクを埋め込みベクトルに変換し、ベクトルデータベースに保存します。
  </Step>
  <Step title="知識ベースを検索する">
    ユーザーがメッセージを送信すると、入力メッセージを埋め込みに変換し、ベクトルデータベースで最近傍を「検索」します。
  </Step>
</Steps>

## 知識ベースのロード

知識ベースを使用する前に、取得に使用される埋め込みでロードする必要があります。

### 非同期ロード

多くのベクトルデータベースは非同期操作をサポートしており、これにより大規模な知識ベースのロード時のパフォーマンスが大幅に向上します。`aload()`メソッドを使用してこの機能を利用できます。

```python
import asyncio

from agno.agent import Agent
from agno.knowledge.pdf import PDFKnowledgeBase, PDFReader
from agno.vectordb.qdrant import Qdrant

COLLECTION_NAME = "pdf-reader"

vector_db = Qdrant(collection=COLLECTION_NAME, url="http://localhost:6333")

# data/pdfsディレクトリのPDFから知識ベースを作成します
knowledge_base = PDFKnowledgeBase(
    path="data/pdf",
    vector_db=vector_db,
    reader=PDFReader(chunk=True),
)

# 知識ベースでエージェントを作成します
agent = Agent(
    knowledge=knowledge_base,
    search_knowledge=True,
)

if __name__ == "__main__":
    # 初回実行後にコメントアウトしてください
    asyncio.run(knowledge_base.aload(recreate=False))

    # エージェントを作成して使用します
    asyncio.run(agent.aprint_response("タイカレーの作り方は？", markdown=True))
```

`aload()`を使用すると、非同期ベクトルデータベース操作が提供するノンブロッキング操作、並列処理、および低遅延を最大限に活用できます。これは、高いスループット要件を持つ本番環境で特に価値があります。

ベクトルデータベースの非同期機能の詳細については、[ベクトルデータベースの紹介](/vectordb/introduction)を参照してください。
