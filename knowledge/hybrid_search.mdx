通过混合搜索，您可以获得精确匹配的精确度以及语义理解的智能性。在许多情况下，结合这两种方法将提供更全面、更相关的结果。

## 什么是混合搜索？

**混合搜索**是一种检索技术，它结合了**向量搜索**（语义搜索）和**关键词搜索**（词汇搜索）的优点，以查找与查询最相关的结果。

- 向量搜索使用嵌入（密集向量）来捕获文本的语义含义，使系统能够找到意义相似的结果，即使确切的词语不匹配。
- 关键词搜索（BM25、TF-IDF 等）根据查询中确切词语或短语的出现和频率来匹配文档。

混合搜索将这些方法融合在一起，通常通过对两种方法的结果进行评分和/或排名，以最大限度地提高精确度和召回率。


## 关键词搜索 vs 向量搜索 vs 混合搜索

| 特征                  | 关键词搜索                     | 向量搜索                       | 混合搜索                              |
|-----------------------|------------------------------------|--------------------------------------|--------------------------------------------|
| 基于                  | 词汇匹配 (BM25, TF-IDF)     | 嵌入相似度 (余弦, 点积)   | 两者皆有                                        |
| 优势                  | 精确匹配，相关性            | 上下文含义                   | 平衡相关性 + 含义                |
| 劣势                  | 无语义理解           | 遗漏精确关键词                | 计算量稍大                 |
| 示例匹配           | "chicken soup" = *chicken soup*     | "chicken soup" = *hot broth with chicken* | 字面意思和相关概念皆可       |
| 最佳用例           | 法律文档，结构化数据         | 聊天机器人，问答，语义搜索       | 多模态，真实世界混乱的用户查询   |

<Note>
为什么混合搜索可能更适合您的应用程序-

- **提高召回率**：捕获纯关键词或向量搜索遗漏的更多相关结果。
- **平衡精确度**：精确匹配获得优先，同时也包含语义相关的结果。
- **应对模糊性**：处理拼写变体、同义词和模糊的用户意图。
- **兼顾两者优点**：关键词在应该时起作用，含义在需要时起作用。

**非常适合**食谱搜索、客户支持、法律发现等**真实世界应用**。
</Note>

## 示例：使用 `pgvector` 进行混合搜索

```python
from agno.knowledge.pdf_url import PDFUrlKnowledgeBase
from agno.vectordb.pgvector import PgVector, SearchType

# 数据库 URL
db_url = "postgresql+psycopg://ai:ai@localhost:5532/ai"

# 初始化混合向量数据库
hybrid_db = PgVector(
    table_name="recipes",
    db_url=db_url,
    search_type=SearchType.hybrid  # 混合搜索
)

# 使用混合搜索加载 PDF 知识库
knowledge_base = PDFUrlKnowledgeBase(
    urls=["https://agno-public.s3.amazonaws.com/recipes/ThaiRecipes.pdf"],
    vector_db=hybrid_db,
)

# 将数据加载到数据库中（首次设置）
knowledge_base.load(recreate=True, upsert=True)

# 运行混合搜索查询
results = hybrid_db.search("chicken coconut soup", limit=5)
print("混合搜索结果:", results)
```
