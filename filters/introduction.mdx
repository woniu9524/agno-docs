# ナレッジフィルター

ナレッジフィルターを使用すると、ユーザーID、ドキュメントタイプ、年などのメタデータを使用して、ナレッジベース内の検索を制限および絞り込むことができます。この機能は、大量のドキュメントコレクションがあり、特定のユーザーまたはコンテキストに関連する情報を取得したい場合に特に役立ちます。

## ナレッジフィルターを使用する理由

- **パーソナライゼーション:** 特定のユーザーまたはグループの情報を取得します。
- **セキュリティ:** 機密ドキュメントへのアクセスを制限します。
- **効率性:** 検索結果を絞り込むことで、ノイズを減らします。

## ナレッジフィルターの仕組み

ドキュメントをナレッジベースにロードする際に、メタデータ（ユーザーID、ドキュメントタイプ、年など）を添付できます。その後、クエリを実行する際に、特定の基準に一致するドキュメントのみを検索するようにフィルターを指定できます。

**メタデータの例:**
```python
{
    "user_id": "jordan_mitchell",
    "document_type": "cv",
    "year": 2025,
}
```

## フィルターの適用方法

ナレッジフィルターは、主に2つの方法で適用できます。
1. **手動フィルター:** クエリ時に明示的にフィルターを渡します。
2. **エージェントによるフィルター:** エージェントにクエリからフィルターを自動的に抽出させます。

> **ヒント:** 複数のフィルターを組み合わせて、より正確な結果を得ることができます！

## 従来のRAGとエージェントによるRAGでのフィルター

エージェントを設定する際には、ユースケースに適したアプローチを選択することが重要です。Agnoエージェントを使用したRAGには、従来のRAGとエージェントによるRAGの2つの広範なアプローチがあります。従来のRAGアプローチでは、`add_references=True` を設定して、参照がLLMに送信されるシステムメッセージに含まれるようにします。エージェントによるRAGでは、`search_knowledge=True` を設定して、エージェントがナレッジベースに直接アクセスする機能を活用します。

例:
```python
agent = Agent(
    name="KnowledgeFilterAgent",
    search_knowledge=False,  # エージェントによる検索を使用しない
    add_references=True,     # ナレッジベースの参照をシステムプロンプトに追加する
    knowledge_filters={"user_id": "jordan_mitchell"}, # このようにフィルターを渡す
)
```

<Check>
これらの構成は一度に1つだけ使用し、もう一方はfalseに設定してください。デフォルトでは、`search_knowledge=True` が推奨されます。これは、より動的でインタラクティブなエクスペリエンスを提供するからです。
従来のRAGシステムでナレッジフィルターを設定する方法については、[こちら](/examples/concepts/knowledge/filters/filtering-traditional-RAG)の例をご覧ください。
</Check>

## ベストプラクティス

- プロンプトは説明的にしてください（例: ユーザー名、ドキュメントタイプ、年を含める）。
- 対話型アプリケーションやチャットボットには、エージェントによるフィルターを使用してください。

## 手動フィルターとエージェントによるフィルター

| 手動フィルター | エージェントによるフィルター |
|------------------|------------------|
| コード内の明示的なフィルター | クエリテキストから推測されるフィルター |
| 完全な制御 | より自然で、コード量が少ない |
| 自動化に適している | ユーザー向けのアプリケーションに適している |
