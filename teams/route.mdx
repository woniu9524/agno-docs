**Route モード**では、チームリーダーはリクエストの内容に基づいて、ユーザーのクエリを最も適切なチームメンバーにルーティングします。

チームリーダーはスマートルーターとして機能し、クエリを分析して、それに対処するのに最も適したエージェントを選択します。メンバーの応答は、直接ユーザーに返されます。

## Route モードの仕組み

「route」モードでは：

1. チームはユーザーのクエリを受け取ります。
2. チームリーダーがクエリを分析し、どのチームメンバーが適切な専門知識を持っているかを判断します。
3. クエリは選択されたチームメンバーに転送されます。
4. チームメンバーからの応答は、直接ユーザーに返されます。

このモードは、専門分野が異なるエージェントがいて、クエリを適切な専門家に自動的に向けたい場合に特に役立ちます。

## Route モードでの構造化出力

Route モードの強力な機能の 1 つは、メンバーエージェントからの構造化された出力を維持できることです。
Pydantic モデルを応答に使用すると、選択されたチームメンバーからの応答は、指定された構造に自動的に解析されます。

### 構造化出力モデルの定義

```python
from pydantic import BaseModel
from typing import List, Optional
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.team import Team


class StockAnalysis(BaseModel):
    symbol: str
    company_name: str
    analysis: str

class CompanyAnalysis(BaseModel):
    company_name: str
    analysis: str

stock_searcher = Agent(
    name="Stock Searcher",
    model=OpenAIChat("gpt-4o"),
    response_model=StockAnalysis,
    role="Searches for information on stocks and provides price analysis.",
    tools=[
        YFinanceTools(
            stock_price=True,
            analyst_recommendations=True,
        )
    ],
)

company_info_agent = Agent(
    name="Company Info Searcher",
    model=OpenAIChat("gpt-4o"),
    role="Searches for information about companies and recent news.",
    response_model=CompanyAnalysis,
    tools=[
        YFinanceTools(
            stock_price=False,
            company_info=True,
            company_news=True,
        )
    ],
)

team = Team(
    name="Stock Research Team",
    mode="route",
    model=OpenAIChat("gpt-4o"),
    members=[stock_searcher, company_info_agent],
    markdown=True,
)

# This should route to the stock_searcher
response = team.run("What is the current stock price of NVDA?")
assert isinstance(response.content, StockAnalysis)
