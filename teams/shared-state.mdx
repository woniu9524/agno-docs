チームメンバー間で状態を共有するには、いくつかの方法があります。

## 共有チームの状態

チームセッションの状態は、共有状態とプライベート状態の両方の機能を持つ、エージェントチーム間の洗練された状態管理を可能にします。

チームは、独自のプライベートメトリクスや構成を維持しながら、共有情報（買い物リストなど）で調整する必要があることがよくあります。Agnoは、このためのエレガントな3層の状態システムを提供します。

Agnoのチーム状態管理は、3つの異なるレベルを提供します。

- チームの `team_session_state` - すべてのチームメンバーがアクセスできる共有状態。
- チームの `session_state` - チームリーダーのみがアクセスできるプライベート状態。
- エージェントの `session_state` - 各エージェントメンバーのプライベート状態。

<Check>
チームの状態は、ネストされたチーム構造全体にも伝播します。
</Check>

### チームセッションの状態の使用方法

チームメンバー間で状態を共有するには、`Team` の `team_session_state` パラメータを設定できます。
この状態はすべてのチームメンバーが利用でき、それらの間で同期されます。

例：

```python
team = Team(
    members=[agent1, agent2, agent3],
    team_session_state={"shopping_list": []},
)
```

メンバーは、ツール内の `team_session_state` 属性を使用して共有状態にアクセスできます。

例：

```python
def add_item(agent: Agent, item: str) -> str:
    """買い物リストにアイテムを追加し、確認を返します。

    Args:
        item (str): 買い物リストに追加するアイテム。
    """
    # アイテムが既にリストにない場合に追加します
    if item.lower() not in [
        i.lower() for i in agent.team_session_state["shopping_list"]
    ]:
        agent.team_session_state["shopping_list"].append(item)
        return f"'{item}' を買い物リストに追加しました"
    else:
        return f"'{item}' は既に買い物リストにあります"
```

### 例

共有の買い物リストを管理するチームの簡単な例を以下に示します。


```python team_session_state.py
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.team import Team


# 共有チームの状態で動作するツールを定義します
def add_item(agent: Agent, item: str) -> str:
    """買い物リストにアイテムを追加します。"""
    if item.lower() not in [
        i.lower() for i in agent.team_session_state["shopping_list"]
    ]:
        agent.team_session_state["shopping_list"].append(item)
        return f"'{item}' を買い物リストに追加しました"
    else:
        return f"'{item}' は既に買い物リストにあります"


def remove_item(agent: Agent, item: str) -> str:
    """買い物リストからアイテムを削除します。"""
    for i, list_item in enumerate(agent.team_session_state["shopping_list"]):
        if list_item.lower() == item.lower():
            agent.team_session_state["shopping_list"].pop(i)
            return f"'{list_item}' を買い物リストから削除しました"
    
    return f"'{item}' は買い物リストに見つかりませんでした"


# 買い物リストを管理するエージェントを作成します
shopping_agent = Agent(
    name="Shopping List Agent",
    role="買い物リストを管理する",
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[add_item, remove_item],
)


# チームレベルのツールを定義します
def list_items(team: Team) -> str:
    """買い物リストのすべてのアイテムを一覧表示します。"""
    # 共有状態にアクセスします（プライベート状態ではありません）
    shopping_list = team.team_session_state["shopping_list"]
    
    if not shopping_list:
        return "買い物リストは空です。"
    
    items_text = "\n".join([f"- {item}" for item in shopping_list])
    return f"現在の買い物リスト:\n{items_text}"


def add_chore(team: Team, chore: str) -> str:
    """完了した家事をチームのプライベートログに追加します。"""
    # チームのプライベート状態にアクセスします
    if "chores" not in team.session_state:
        team.session_state["chores"] = []
    
    team.session_state["chores"].append(chore)
    return f"ログされた家事: {chore}"


# 共有状態とプライベート状態の両方を持つチームを作成します
shopping_team = Team(
    name="Shopping Team",
    mode="coordinate",
    model=OpenAIChat(id="gpt-4o-mini"),
    members=[shopping_agent],
    # 共有状態 - すべてのメンバーがアクセス可能
    team_session_state={"shopping_list": []},
    # チームのプライベート状態 - チームのみがアクセス可能
    session_state={"chores": []},
    tools=[list_items, add_chore],
    instructions=[
        "買い物リストを管理します。",
        "追加/削除のリクエストはShopping List Agentに転送します。",
        "list_items を使用して現在のリストを表示します。",
        "add_chore を使用して完了したタスクをログします。",
    ],
    show_tool_calls=True,
)

# 使用例
shopping_team.print_response("牛乳、卵、パンを追加してください", stream=True)
print(f"共有状態: {shopping_team.team_session_state}")

shopping_team.print_response("私のリストに何がありますか？", stream=True)

shopping_team.print_response("卵を手に入れました", stream=True)
print(f"共有状態: {shopping_team.team_session_state}")
print(f"チームのプライベート状態: {shopping_team.session_state}")
```

<Tip> 
共有ツールが `agent.team_session_state` を使用していることに注意してください。これにより、状態がチーム全体に伝播および永続化されます。サブチーム内でも同様です。これにより、すべてのメンバーに対して一貫した共有状態が保証されます。

対照的に、チーム固有のツールは `team.session_state` を使用し、プライベートでチーム固有の状態を可能にします。たとえば、チームリーダーのツールは、`team.session_state` を使用して独自のセッション状態を維持します。
</Tip>


完全な例は[こちら](/examples/teams/shared_state/team_shared_state)でご覧ください。

## エージェントコンテキスト

チームリーダーは、エージェントによって更新される（つまり、チームリーダーによって更新される）共有コンテキストを維持し、必要に応じてチームメンバーに送信されます。

エージェントコンテキストは、エージェント間の効果的な情報共有とコラボレーションに不可欠であり、チームの応答の質は、チームリーダーがこの共有エージェントコンテキストをどの程度うまく管理するかに依存します。
これには、チームリーダーが応答の質を確保するために、より高品質なモデルが必要になる場合があります。

<Note>
チームメンバーのタスクと応答は自動的にチームコンテキストに追加されますが、エージェントコンテキストは開発者によって有効にする必要があります。
</Note>

### エージェントコンテキストの有効化

チームリーダーがエージェントコンテキストを維持できるようにするには、`enable_agentic_context=True` を設定します。

これにより、チームリーダーは実行中にチームコンテキストを維持および更新できます。

```python
team = Team(
    members=[agent1, agent2, agent3],
    enable_agentic_context=True,  # チームリーダーがエージェントコンテキストを維持できるようにする
)
```

### チームメンバーのインタラクション

エージェントチームはメンバー間のインタラクションを共有でき、エージェントはお互いの出力から学習できます。

```python
team = Team(
    members=[agent1, agent2, agent3],
    share_member_interactions=True,  # インタラクションを共有する
)
