有多种方法可以在团队成员之间共享状态。

## 共享团队状态

团队会话状态支持跨代理团队进行复杂的状态管理，并具备共享和私有状态的功能。

团队通常需要协调共享信息（例如购物清单），同时维护自己的私有指标或配置。Agno 为此提供了一个优雅的三层状态系统。

Agno 的团队状态管理提供三个不同的级别：

- Team's team_session_state - 所有团队成员都可以访问的共享状态。
- Team's session_state - 仅团队领导者可以访问的私有状态。
- Agent's session_state - 每个代理成员的私有状态。

<Check>
团队状态也会通过嵌套的团队结构进行传播。
</Check>

### 如何使用团队会话状态

您可以设置 `Team` 上的 `team_session_state` 参数来在团队成员之间共享状态。
此状态对所有团队成员都可用，并在他们之间进行同步。

例如：

```python
team = Team(
    members=[agent1, agent2, agent3],
    team_session_state={"shopping_list": []},
)
```

成员可以通过工具中的 `team_session_state` 属性访问共享状态。

例如：

```python
def add_item(agent: Agent, item: str) -> str:
    """将商品添加到购物清单并返回确认信息。

    Args:
        item (str): 要添加到购物清单的商品。
    """
    # 如果商品已存在于列表中，则不添加
    if item.lower() not in [
        i.lower() for i in agent.team_session_state["shopping_list"]
    ]:
        agent.team_session_state["shopping_list"].append(item)
        return f"已将 '{item}' 添加到购物清单"
    else:
        return f"'{item}' 已在购物清单中"
```

### 示例

这是一个关于团队管理共享购物清单的简单示例：


```python team_session_state.py
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.team import Team


# 定义处理共享团队状态的工具
def add_item(agent: Agent, item: str) -> str:
    """将商品添加到购物清单。"""
    if item.lower() not in [
        i.lower() for i in agent.team_session_state["shopping_list"]
    ]:
        agent.team_session_state["shopping_list"].append(item)
        return f"已将 '{item}' 添加到购物清单"
    else:
        return f"'{item}' 已在购物清单中"


def remove_item(agent: Agent, item: str) -> str:
    """从购物清单中移除商品。"""
    for i, list_item in enumerate(agent.team_session_state["shopping_list"]):
        if list_item.lower() == item.lower():
            agent.team_session_state["shopping_list"].pop(i)
            return f"已从购物清单中移除 '{list_item}'"
    
    return f"'{item}' 未在购物清单中找到"


# 创建一个管理购物清单的代理
shopping_agent = Agent(
    name="Shopping List Agent",
    role="Manage the shopping list",
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[add_item, remove_item],
)


# 定义团队级别的工具
def list_items(team: Team) -> str:
    """列出购物清单中的所有商品。"""
    # 访问共享状态（非私有状态）
    shopping_list = team.team_session_state["shopping_list"]
    
    if not shopping_list:
        return "购物清单为空。"
    
    items_text = "\n".join([f"- {item}" for item in shopping_list])
    return f"当前购物清单:\n{items_text}"


def add_chore(team: Team, chore: str) -> str:
    """将已完成的家务添加到团队的私有日志中。"""
    # 访问团队的私有状态
    if "chores" not in team.session_state:
        team.session_state["chores"] = []
    
    team.session_state["chores"].append(chore)
    return f"已记录家务：{chore}"


# 创建一个同时具有共享和私有状态的团队
shopping_team = Team(
    name="Shopping Team",
    mode="coordinate",
    model=OpenAIChat(id="gpt-4o-mini"),
    members=[shopping_agent],
    # 共享状态 - 所有成员均可访问
    team_session_state={"shopping_list": []},
    # 团队私有状态 - 仅团队可访问
    session_state={"chores": []},
    tools=[list_items, add_chore],
    instructions=[
        "你负责管理购物清单。",
        "将添加/移除请求转发给购物清单代理。",
        "使用 list_items 显示当前列表。",
        "使用 add_chore 记录已完成的任务。",
    ],
    show_tool_calls=True,
)

# 示例用法
shopping_team.print_response("添加牛奶、鸡蛋和面包", stream=True)
print(f"共享状态：{shopping_team.team_session_state}")

shopping_team.print_response("我的列表里有什么？", stream=True)

shopping_team.print_response("我拿到了鸡蛋", stream=True)
print(f"共享状态：{shopping_team.team_session_state}")
print(f"团队私有状态：{shopping_team.session_state}")
```

<Tip>
请注意，共享工具使用 `agent.team_session_state`，这使得状态可以在整个团队中传播和持久化——即使是团队内的子团队。这确保了所有成员的共享状态一致。

相比之下，特定于团队的工具使用 `team.session_state`，允许私有、特定于团队的状态。例如，团队领导者的工具将使用 `team.session_state` 来维护自己的会话状态。
</Tip>


请参阅完整的示例 [此处](/examples/teams/shared_state/team_shared_state)。

## 代理上下文

团队领导者维护一个共享上下文，该上下文由代理（即由团队领导者）更新，并在需要时发送给团队成员。

代理上下文对于代理之间的有效信息共享和协作至关重要，而团队响应的质量取决于团队领导者管理此共享代理上下文的程度。
这可能需要团队领导者使用更高质量的模型来确保团队响应的质量。

<Note>
团队成员的任务和响应会自动添加到团队上下文中，但代理上下文需要由开发者启用。
</Note>

### 启用代理上下文

要启用团队领导者维护代理上下文，请设置 `enable_agentic_context=True`。

这将允许团队领导者在运行期间维护和更新团队上下文。

```python
team = Team(
    members=[agent1, agent2, agent3],
    enable_agentic_context=True,  # 启用团队领导者维护代理上下文
)
```

### 团队成员交互

代理团队可以共享成员之间的交互，使代理能够从彼此的输出来学习：

    share_member_interactions=True,  # 共享交互
