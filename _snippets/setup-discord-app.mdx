## セットアップと構成
<Steps>

<Step title="前提条件">
以下が揃っていることを確認してください：
- Python 3.7以降
- サーバー管理権限を持つDiscordアカウント
- 必要なPythonパッケージ（後のステップでインストールします）
</Step>

<Step title="Discordアプリケーションの作成">
1. [Discord Developer Portal](https://discord.com/developers/applications)にアクセスします
2. 「New Application」をクリックします
3. アプリケーション名を入力します（例：「My Agno Bot」）
4. Developer Terms of Serviceに同意します
5. 「Create」をクリックします
</Step>

<Step title="ボットユーザーの作成">
1. アプリケーションの設定で、「Bot」セクションに移動します
2. 「Add Bot」をクリックします
3. 「Yes, do it!」をクリックして確認します
4. 「Token」セクションで、「Copy」をクリックしてボットトークンをコピーします
5. このトークンは安全に保存してください（後で必要になります）
</Step>

<Step title="ボットの権限とインテントの構成">
1. ボットの設定で、「Privileged Gateway Intents」までスクロールします
2. 以下を有効にします：
   - **Presence Intent**（オプション、ユーザーのステータス用）
   - **Server Members Intent**（メンバー関連イベント用）
   - **Message Content Intent**（メッセージコンテンツの読み取りに必要）
3. 「Bot Permissions」で、ボットが以下を持っていることを確認します：
   - Send Messages
   - Read Message History
   - Create Public Threads
   - Use Slash Commands（オプション）
</Step>

<Step title="環境変数のセットアップ">
プロジェクトのルートに`.envrc`ファイルを作成し、プレースホルダーを実際のボットトークンに置き換えて、以下の内容を記述します：
```bash
export DISCORD_BOT_TOKEN="your_bot_token_here"
```

ボットトークンは、Discord Developer Portalの「Bot」>「Token」で確認できます。

このファイルがシェルによってソースされていることを確認してください（例：「direnv allow」を使用）。
</Step>

<Step title="必要なパッケージのインストール">
必要なPythonパッケージをインストールします：
```bash
pip install discord.py agno
```


</Step>

<Step title="ボットをDiscordサーバーに招待する">
1. アプリケーションの設定で、「OAuth2」>「URL Generator」に移動します
2. 「Scopes」で以下を選択します：
   - `bot`
   - `applications.commands`（スラッシュコマンドを使用する場合）
3. 「Bot Permissions」で、ボットが必要とする権限を選択します：
   - **Send Messages**
   - **Create Public Threads**
   - **Read Message History**
   - **Attach Files**
   - **Embed Links**
   - **Use External Emojis**（オプション）
4. 生成されたURLをコピーし、ブラウザでアクセスして、ボットを追加したいサーバーを選択します
</Step>

<Step title="ボットのテスト">
1. ボットアプリケーションを起動します
2. Discordサーバーに移動します
3. ボットがアクセスできる任意のチャンネルでメッセージを送信します
4. ボットは自動的にスレッドを作成して応答するはずです
5. メディアボットを使用している場合は、画像をアップロードしてメディア処理をテストしてみてください
</Step>

</Steps>

<Callout type="warning">
**セキュリティに関する注意**: Discordボットトークンをバージョン管理システムにコミットしないでください。機密データには、常に環境変数またはセキュアな設定管理を使用してください。
</Callout>
