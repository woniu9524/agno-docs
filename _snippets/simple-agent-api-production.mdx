## 本番環境での実行

このリポジトリには、アプリケーションの本番稼働に対応したコンテナイメージをビルドするための `Dockerfile` が含まれています。

本番環境で実行するための一般的なプロセスは以下の通りです。

1.  `scripts/build_image.sh` ファイルを更新し、`IMAGE_NAME` および `IMAGE_TAG` 変数を設定します。
2.  イメージをビルドし、コンテナレジストリにプッシュします。
    ```bash
    ./scripts/build_image.sh
    ```
3.  選択したクラウドプロバイダーで実行します。

### 詳細な手順

#### 1. 本番環境の設定

-   本番環境の環境変数（例: `OPENAI_API_KEY`、データベース接続文字列）が安全に管理されていることを確認してください。ほとんどのクラウドプロバイダーは、デプロイされたサービスに対してこれらの環境変数を設定する方法を提供しています。
-   `/agents` ディレクトリ内のエージェント設定を確認し、本番環境のニーズ（例: 正しいモデルバージョン、本番環境固有の設定）に合わせて設定されていることを確認してください。

#### 2. 本番用 Docker イメージのビルド

`scripts/build_image.sh` スクリプトを更新して、希望する `IMAGE_NAME` と `IMAGE_TAG`（例: `your-repo/agent-api:v1.0.0`）を設定します。

スクリプトを実行して、イメージをビルドしプッシュします。

```bash
./scripts/build_image.sh
```

#### 3. クラウドサービスへのデプロイ

レジストリにイメージがあれば、コンテナ化されたアプリケーションをサポートするさまざまなクラウドサービスにデプロイできます。一般的なオプションをいくつか紹介します。

**サーバーレスコンテナプラットフォーム:**

-   [Google Cloud Run](https://cloud.google.com/run): ステートレスコンテナを自動的にスケーリングする、フルマネージドプラットフォームです。
-   [AWS App Runner](https://aws.amazon.com/apprunner/): コンテナ化されたWebアプリケーションやAPIを簡単にスケーリングしてデプロイできます。
-   [Azure Container Apps](https://azure.microsoft.com/en-us/products/container-apps): サーバーレスコンテナを使用して最新のアプリやマイクロサービスをビルドおよびデプロイします。

**コンテナオーケストレーションサービス:**

-   [Amazon Elastic Container Service (ECS)](https://aws.amazon.com/ecs/): サーバーレスコンピューティングの場合はAWS Fargate、またはEC2インスタンスと組み合わせてよく使用されます。
-   [Google Kubernetes Engine (GKE)](https://cloud.google.com/kubernetes-engine): マネージドKubernetesサービスです。
-   [Azure Kubernetes Service (AKS)](https://azure.microsoft.com/en-us/services/kubernetes-service/): マネージドKubernetesサービスです。

**Dockerサポート付きのPlatform as a Service (PaaS):**

-   [Railway.app](https://railway.app/): Dockerfileからの簡単なデプロイ。
-   [Render](https://render.com/): Dockerコンテナ、データベース、静的サイトのデプロイを簡素化します。
-   [Heroku](https://www.heroku.com/): Dockerコンテナのデプロイをサポートします。

**専門プラットフォーム:**

-   [Modal](https://modal.com/): クラウドでPythonコードを実行するためのプラットフォームで、Webエンドポイントを提供できます。

具体的なデプロイ手順は、選択したプロバイダーによって異なります。一般的には、サービスにレジストリ内のコンテナイメージを指定し、ポートマッピング（アプリケーションはデフォルトでポート `8000` で実行されます）、環境変数、スケーリング、データベース接続を設定します。

#### 4. データベースの設定

デフォルトの `docker-compose.yml` は、ローカル開発用にPostgreSQLデータベースを設定します。本番環境では、マネージドデータベースサービス（例: AWS RDS、Google Cloud SQL、Azure Database for PostgreSQL）を使用してください。
デプロイされたアプリケーションが、通常は環境変数を通じて、本番データベースの正しいデータベース接続URLで設定されていることを確認してください。
