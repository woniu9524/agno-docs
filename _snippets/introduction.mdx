## Agnoエージェント

- 言語モデルを使用して推論と計画を行います。
- ツールを使用してアクションを実行し、外部システムと対話します。
- メモリと状態をデータベースに保存し、チャット履歴、ユーザー設定、会話の要約を記憶します。
- ベクトルデータベースに知識を保存し、ドメインの専門知識でトレーニングデータを補強します。

## 例：リサーチエージェント

Webを検索してトップリンクを読み取ることで、リサーチレポートを作成できるエージェントを作成しましょう。`description`と`instructions`を使用してエージェントに**「プロンプト」**します。

<Steps>
  <Step title="リサーチエージェントの作成">
    ファイル `research_agent.py` を作成します。

    ```python research_agent.py
    from agno.agent import Agent
    from agno.models.openai import OpenAIChat
    from agno.tools.duckduckgo import DuckDuckGoTools
    from agno.tools.newspaper4k import Newspaper4kTools

    agent = Agent(
        model=OpenAIChat(id="gpt-4o"),
        tools=[DuckDuckGoTools(), Newspaper4kTools()],
        description="あなたは、あるトピックについて記事を書いているシニアNYTリサーチャーです。",
        instructions=[
            "指定されたトピックについて、トップ5件のリンクを検索してください。",
            "次に、各URLを読み取り、記事のテキストを抽出してください。URLが利用できない場合は無視してください。",
            "情報に基づいて、NYTにふさわしい記事を分析して準備してください。",
        ],
        markdown=True,
        show_tool_calls=True,
        add_datetime_to_instructions=True,
        # debug_mode=True,
    )
    agent.print_response("シミュレーション理論", stream=True)
    ```

  </Step>
  <Step title="エージェントの実行">
    ライブラリをインストールします。

    ```shell
    pip install agno openai duckduckgo-search newspaper4k lxml_html_clean
    ```

    エージェントを実行します。

    ```shell
    python research_agent.py
    ```

  </Step>
</Steps>

<Note>
内部的には、descriptionとinstructionsはシステムプロンプトに変換され、入力はユーザープロンプトとして渡されます。

生のメッセージを表示するには `debug_mode=True` を使用します。

</Note>

## エージェントの応答を変数として返す

`Agent.run()` 関数を使用して、応答を変数としてキャプチャします。

```python
from agno.agent import Agent, RunResponse
from agno.utils.pprint import pprint_run_response

agent = Agent(...)

# エージェントを実行し、応答を変数として返します
response: RunResponse = agent.run("シミュレーション理論")
# 応答をマークダウン形式で表示します
pprint_run_response(response, markdown=True)
```

デフォルトでは `stream=False` です。ストリームを返すには `stream=True` を設定します。

```python
from typing import Iterator

# エージェントを実行し、応答ストリームを返します
response_stream: Iterator[RunResponse] = agent.run("シミュレーション理論", stream=True)
# 応答ストリームをマークダウン形式で表示します
pprint_run_response(response_stream, markdown=True, show_time=True)
```

## RunResponse

`Agent.run()` 関数は、`stream=True` の場合は `RunResponse` オブジェクトまたは `RunResponse` オブジェクトのストリームを返します。

### 属性

<ResponseField name="content" type="Any" default="None">
  応答のコンテンツ。
</ResponseField>
<ResponseField name="content_type" type="str" default="str">
  コンテンツのデータ型を指定します。
</ResponseField>
<ResponseField name="context" type="List[MessageContext]" default="None">
  RAGのために応答に追加されたコンテキスト。
</ResponseField>
<ResponseField name="event" type="str" default="RunEvent.run_response.value">
  応答のイベントタイプ。
</ResponseField>
<ResponseField name="event_data" type="Dict[str, Any]" default="None">
  イベントに関連付けられたデータ。
</ResponseField>
<ResponseField name="messages" type="List[Message]" default="None">
  応答に含まれるメッセージのリスト。
</ResponseField>
<ResponseField name="metrics" type="Dict[str, Any]" default="None">
  実行の使用状況メトリクス。
</ResponseField>
<ResponseField name="model" type="str" default="None">
  実行に使用されたモデル。
</ResponseField>
<ResponseField name="run_id" type="str" default="None">
  実行ID。
</ResponseField>
<ResponseField name="agent_id" type="str" default="None">
  実行のエージェントID。
</ResponseField>
<ResponseField name="session_id" type="str" default="None">
  実行のセッションID。
</ResponseField>
<ResponseField name="tools" type="List[Dict[str, Any]]" default="None">
  モデルに提供されたツールのリスト。
</ResponseField>
<ResponseField name="created_at" type="int">
  応答作成のUnixタイムスタンプ。
</ResponseField>
