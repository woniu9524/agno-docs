## AWS リソースの作成

以下を使用して AWS リソースを作成します。

<CodeGroup>

```bash terminal
ag ws up --env prd --infra aws
```

```bash shorthand
ag ws up prd:aws
```

</CodeGroup>

これにより、以下が作成されます。

1. [ECS クラスター](https://docs.aws.amazon.com/AmazonECS/latest/developerguide/clusters.html) アプリケーション用。
2. [ECS タスク定義](https://docs.aws.amazon.com/AmazonECS/latest/developerguide/task_definitions.html) および [サービス](https://docs.aws.amazon.com/AmazonECS/latest/developerguide/ecs_services.html) ECS クラスター上でアプリケーションを実行します。
3. [ロードバランサー](https://aws.amazon.com/elasticloadbalancing/) アプリケーションへのトラフィックをルーティングします。
4. [セキュリティグループ](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-security-groups.html) 送受信トラフィックを制御します。
5. [シークレット](https://aws.amazon.com/secrets-manager/) アプリケーションおよびデータベースのシークレットを管理します。
6. [RDS データベース](https://aws.amazon.com/rds/) ナレッジベースおよびストレージ用。

確認してリソースの起動を待つには **Enter** を押します。

- RDS データベースのアクティブ化には約 5 分かかります。
- これらのリソースは `workspace/prd_resources.py` ファイルで定義されています。
- [ECS コンソール](https://us-east-1.console.aws.amazon.com/ecs/v2/clusters) を使用してサービスとログを表示します。
- [RDS コンソール](https://us-east-1.console.aws.amazon.com/rds/home?#databases:) を使用してデータベースインスタンスを表示します。
