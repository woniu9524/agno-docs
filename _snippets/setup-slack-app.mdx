## 设置与配置
<Steps>

<Step title="先决条件">
请确保您已具备以下条件：
- 拥有管理员权限的 Slack 工作区
- ngrok（用于开发）
- Python 3.7+
</Step>

<Step title="创建 Slack 应用">
1. 前往 [Slack 应用目录](https://api.slack.com/apps)
2. 点击“创建新应用”
3. 选择“从头开始”
4. 提供：
    - 应用名称
    - 要安装到的工作区
5. 点击“创建应用”
</Step>

<Step title="配置 OAuth 与权限">
1. 在 Slack 应用设置中导航至“OAuth 与权限”
2. 在“范围”下，点击“添加 OAuth 范围”
3. 添加以下 Bot Token 范围：
   - `app_mention`
   - `chat:write`
   - `chat:write.customize`
   - `chat:write.public`
   - `im:history`
   - `im:read`
   - `im:write`
4. 滚动到顶部，点击“安装到工作区”
5. 点击“允许”以授权应用
</Step>

<Step title="设置环境变量">
在项目根目录下创建一个 `.envrc` 文件，并包含以下内容，将占位符值替换为您实际的凭据：
```bash
export SLACK_TOKEN="xoxb-your-bot-user-token"  # Bot User OAuth Token
export SLACK_SIGNING_SECRET="your-signing-secret"  # App Signing Secret
```

在 Slack 应用设置中查找这些值：
- Bot User OAuth Token：位于“OAuth 与权限”下
- Signing Secret：位于“基本信息” > “应用凭据”下

确保您的 shell 已加载此文件（例如，通过使用 `direnv allow`）。
</Step>

<Step title="使用 ngrok 设置 Webhook">
1. 对于本地开发，请使用 ngrok 将您的本地服务器暴露给互联网：
   ```bash
   ngrok http 8000
   # 或者，如果您拥有付费 ngrok 计划并使用静态域名：
   # ngrok http --domain=your-custom-domain.ngrok-free.app 8000
   ```
2. 复制 ngrok 提供的 `https://` URL
3. 在您的 Slack 应用设置中，转到“事件订阅”
4. 通过切换开关启用事件
5. 添加您的请求 URL：
   - 格式：`https://your-ngrok-url.ngrok.io/slack/events`
6. 等待 Slack 验证端点（您的应用必须正在运行）
</Step>

<Step title="配置事件订阅">
1. 在“事件订阅”的“订阅机器人事件”下：
2. 点击“添加机器人用户事件”并添加：
   - `app_mention`
   - `message.im`
   - `message.channels`
   - `message.groups`
3. 点击“保存更改”
4. 重新安装您的应用以应用新权限
</Step>

<Step title="启用应用主页">
1. 在 Slack 应用设置中转到“应用主页”
2. 在“显示标签页”下：
   - 启用“消息标签页”
   - 勾选“允许用户从消息标签页发送斜杠命令和消息”
3. 保存更改
</Step>

<Step title="最终安装">
1. 返回 Slack 应用设置中的“安装应用”
2. 点击“重新安装到工作区”
3. 使用新权限授权应用
4. 您的应用现已准备就绪！
</Step>

</Steps>
