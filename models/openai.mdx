GPT 模型是同类最佳的 LLM，被 **Agents** 用作默认 LLM。OpenAI 支持多种世界一流的模型。在此处查看其模型：[https://platform.openai.com/docs/models](https://platform.openai.com/docs/models)。

我们建议进行实验，以找到最适合您用例的模型。以下是一些通用建议：

- `gpt-4o` 适用于大多数通用用例。
- `gpt-4o-mini` 模型适用于小型任务和更快的推理。
- `o1` 模型适用于复杂的推理和多步任务。
- `o3-mini` 是一个强大的推理模型，支持工具调用和结构化输出，但成本要低得多。

OpenAI 具有基于层级的速率限制。有关更多信息，请参阅[文档](https://platform.openai.com/docs/guides/rate-limits/usage-tiers)。

## 认证

设置您的 `OPENAI_API_KEY` 环境变量。您可以在此处从 OpenAI 获取一个：[https://platform.openai.com/account/api-keys](https://platform.openai.com/account/api-keys)。

<CodeGroup>

```bash Mac
export OPENAI_API_KEY=sk-***
```

```bash Windows
setx OPENAI_API_KEY sk-***
```

</CodeGroup>

## 示例

在您的 `Agent` 中使用 `OpenAIChat`：

<CodeGroup>

```python agent.py

from agno.agent import Agent, RunResponse
from agno.models.openai import OpenAIChat

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    markdown=True
)

# 在终端中打印响应
agent.print_response("分享一个两句话的恐怖故事。")

```

## Prompt 缓存

使用我们的 `OpenAIChat` 模型将自动进行 Prompt 缓存。您可以在其文档中详细了解 OpenAI 如何处理缓存：[https://platform.openai.com/docs/guides/prompt-caching](https://platform.openai.com/docs/guides/prompt-caching)。

</CodeGroup>
