[LiteLLM](https://docs.litellm.ai/docs/) 提供了一个统一的接口，用于各种 LLM 提供商，使您能够使用相同的代码来使用不同的模型。

Agno 通过两种方式与 LiteLLM 集成：
1. **直接 SDK 集成** - 使用 LiteLLM Python SDK
2. **代理服务器集成** - 使用 LiteLLM 作为兼容 OpenAI 的代理

## 先决条件

对于两种集成方法，您需要：

```shell
# 安装所需的包
pip install agno litellm
```

设置您的 API 密钥：
无论使用何种模型（OpenAI、Hugging Face 或 XAI），API 密钥都引用为 `LITELLM_API_KEY`。
```shell
export LITELLM_API_KEY=your_api_key_here
```

## SDK 集成

`LiteLLM` 类提供了与 LiteLLM Python SDK 的直接集成。

### 基本用法

```python
from agno.agent import Agent
from agno.models.litellm import LiteLLM

# 使用 GPT-4o 创建一个代理
agent = Agent(
    model=LiteLLM(
        id="gpt-4o",  # 要使用的模型 ID
        name="LiteLLM",  # 可选的显示名称
    ),
    markdown=True,
)

# 获取响应
agent.print_response("Share a 2 sentence horror story")
```

### 使用 Hugging Face 模型

LiteLLM 也可以与 Hugging Face 模型一起使用：

```python
from agno.agent import Agent
from agno.models.litellm import LiteLLM

agent = Agent(
    model=LiteLLM(
        id="huggingface/mistralai/Mistral-7B-Instruct-v0.2",
        top_p=0.95,
    ),
    markdown=True,
)

agent.print_response("What's happening in France?")
```

### 配置选项

`LiteLLM` 类接受以下参数：

| 参数 | 类型 | 描述 | 默认值 |
| --- | --- | --- | --- |
| `id` | str | 模型标识符（例如，“gpt-4o”或“huggingface/mistralai/Mistral-7B-Instruct-v0.2”） | "gpt-4o" |
| `name` | str | 模型的显示名称 | "LiteLLM" |
| `provider` | str | 提供商名称 | "LiteLLM" |
| `api_key` | Optional[str] | API 密钥（回退到 LITELLM_API_KEY 环境变量） | None |
| `api_base` | Optional[str] | API 请求的基础 URL | None |
| `max_tokens` | Optional[int] | 响应中的最大 token 数 | None |
| `temperature` | float | 采样温度 | 0.7 |
| `top_p` | float | Top-p 采样值 | 1.0 |
| `request_params` | Optional[Dict[str, Any]] | 其他请求参数 | None |
