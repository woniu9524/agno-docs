[LiteLLM](https://docs.litellm.ai/docs/) は、さまざまな LLM プロバイダーに対する統一されたインターフェイスを提供し、同じコードで異なるモデルを使用できるようにします。

Agno は 2 つの方法で LiteLLM と統合されます。
1. **直接 SDK 統合** - LiteLLM Python SDK を使用
2. **プロキシサーバー統合** - LiteLLM を OpenAI 互換プロキシとして使用

## 前提条件

どちらの統合方法でも、以下が必要です。

```shell
# 必要なパッケージをインストールします
pip install agno litellm
```

API キーを設定します。
使用するモデル（OpenAI、Hugging Face、または XAI）に関係なく、API キーは `LITELLM_API_KEY` として参照されます。
```shell
export LITELLM_API_KEY=your_api_key_here
```

## SDK 統合

`LiteLLM` クラスは、LiteLLM Python SDK との直接統合を提供します。

### 基本的な使い方

```python
from agno.agent import Agent
from agno.models.litellm import LiteLLM

# GPT-4o でエージェントを作成します
agent = Agent(
    model=LiteLLM(
        id="gpt-4o",  # 使用するモデル ID
        name="LiteLLM",  # オプションの表示名
    ),
    markdown=True,
)

# レスポンスを取得します
agent.print_response("Share a 2 sentence horror story")
```

### Hugging Face モデルの使用

LiteLLM は Hugging Face モデルでも動作します。

```python
from agno.agent import Agent
from agno.models.litellm import LiteLLM

agent = Agent(
    model=LiteLLM(
        id="huggingface/mistralai/Mistral-7B-Instruct-v0.2",
        top_p=0.95,
    ),
    markdown=True,
)

agent.print_response("What's happening in France?")
```

### 設定オプション

`LiteLLM` クラスは、次のパラメーターを受け入れます。

| パラメーター | タイプ | 説明 | デフォルト |
| --- | --- | --- | --- |
| `id` | str | モデル識別子（例：「gpt-4o」または「huggingface/mistralai/Mistral-7B-Instruct-v0.2」） | "gpt-4o" |
| `name` | str | モデルの表示名 | "LiteLLM" |
| `provider` | str | プロバイダー名 | "LiteLLM" |
| `api_key` | Optional[str] | API キー（`LITELLM_API_KEY` 環境変数にフォールバックします） | None |
| `api_base` | Optional[str] | API リクエストのベース URL | None |
| `max_tokens` | Optional[int] | レスポンスの最大トークン数 | None |
| `temperature` | float | サンプリング温度 | 0.7 |
| `top_p` | float | Top-p サンプリング値 | 1.0 |
| `request_params` | Optional[Dict[str, Any]] | 追加のリクエストパラメーター | None |
