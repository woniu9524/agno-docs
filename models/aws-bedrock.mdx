AWS Bedrock を使用して、AWS 上のさまざまな基盤モデルにアクセスします。モデルへのアクセスは [ポータル](https://us-east-1.console.aws.amazon.com/bedrock/home?region=us-east-1#/model-catalog)で管理してください。

[AWS Bedrock の基盤モデル](https://docs.aws.amazon.com/bedrock/latest/userguide/models-supported.html)をすべて参照してください。すべての Bedrock モデルがすべての機能をサポートしているわけではありません。各モデルの[サポートされている機能](https://docs.aws.amazon.com/bedrock/latest/userguide/conversation-inference-supported-models-features.html)を参照してください。

ユースケースに最適なモデルを見つけるために、実験することをお勧めします。一般的な推奨事項は次のとおりです。

- 一般的に優れたパフォーマンスを持つ Mistral モデルについては、「`mistral.mistral-large-2402-v1:0`」を参照してください。
- Amazon Nova モデルを試すことができます。一般的なタスクには「`amazon.nova-pro-v1:0`」を使用してください。
- Claude モデルについては、[Claude の統合](/models/aws-claude)を参照してください。

<Warning>
AWS Bedrock の非同期使用はまだサポートされていません。`Agent` で `AwsBedrock` を使用する場合、`agent.run` と `agent.print_response` のみを使用できます。
</Warning>

## 認証

`AWS_ACCESS_KEY_ID`、`AWS_SECRET_ACCESS_KEY`、および `AWS_REGION` 環境変数を設定します。

キーは[こちら](https://us-east-1.console.aws.amazon.com/iam/home?region=us-east-1#/home)から取得してください。

<CodeGroup>

```bash Mac
export AWS_ACCESS_KEY_ID=***
export AWS_SECRET_ACCESS_KEY=***
export AWS_REGION=***
```

```bash Windows
setx AWS_ACCESS_KEY_ID ***
setx AWS_SECRET_ACCESS_KEY ***
setx AWS_REGION ***
```

</CodeGroup>

## 例

`Agent` で `AwsBedrock` を使用します。

<CodeGroup>

```python agent.py
from agno.agent import Agent
from agno.models.aws import AwsBedrock

agent = Agent(
    model=AwsBedrock(id="mistral.mistral-large-2402-v1:0"),
    markdown=True
)

# 応答をターミナルに出力します
agent.print_response("Share a 2 sentence horror story.")
```

</CodeGroup>

<Note> その他の例は[こちら](/examples/models/aws/bedrock)でご覧ください。 </Note>
