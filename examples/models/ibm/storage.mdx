## コード

```python cookbook/models/ibm/watsonx/storage.py
from agno.agent import Agent
from agno.models.ibm import WatsonX
from agno.storage.postgres import PostgresStorage
from agno.tools.duckduckgo import DuckDuckGoTools

db_url = "postgresql+psycopg://ai:ai@localhost:5532/ai"

agent = Agent(
    model=WatsonX(id="ibm/granite-20b-code-instruct"),
    storage=PostgresStorage(table_name="agent_sessions", db_url=db_url),
    tools=[DuckDuckGoTools()],
    add_history_to_messages=True,
)
agent.print_response("カナダには何人住んでいますか？")
agent.print_response("彼らの国歌は何と呼ばれていますか？")
```

## 使用方法

<Steps>
  <Snippet file="create-venv-step.mdx" />

  <Step title="APIキーを設定する">
    ```bash
    export IBM_WATSONX_API_KEY=xxx
    export IBM_WATSONX_PROJECT_ID=xxx
    ```
  </Step>

  <Step title="ライブラリをインストールする">
    ```bash
    pip install -U ibm-watsonx-ai sqlalchemy psycopg duckduckgo-search agno
    ```
  </Step>

  <Step title="PostgreSQLを設定する">
    PostgreSQLデータベースが実行されていることを確認してください。コード内の`db_url`をデータベース構成に合わせて調整できます。
  </Step>

  <Step title="エージェントを実行する">
    <CodeGroup>
    ```bash Mac
    python cookbook/models/ibm/watsonx/storage.py
    ```

    ```bash Windows
    python cookbook\models\ibm\watsonx\storage.py
    ```
    </CodeGroup>
  </Step>
</Steps>

この例では、IBM WatsonXでPostgreSQLストレージを使用して、複数の対話間で会話の状態を維持する方法を示しています。PostgreSQLストレージバックエンドを持つエージェントを作成し、複数のメッセージを送信します。会話履歴はそれらの間で保持されます。

注意：`sqlalchemy`パッケージをインストールし、PostgreSQLデータベースを利用可能にする必要があります。
