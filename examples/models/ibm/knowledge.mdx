## コード

```python cookbook/models/ibm/watsonx/knowledge.py
from agno.agent import Agent
from agno.knowledge.pdf_url import PDFUrlKnowledgeBase
from agno.models.ibm import WatsonX
from agno.vectordb.pgvector import PgVector

db_url = "postgresql+psycopg://ai:ai@localhost:5532/ai"

knowledge_base = PDFUrlKnowledgeBase(
    urls=["https://agno-public.s3.amazonaws.com/recipes/ThaiRecipes.pdf"],
    vector_db=PgVector(table_name="recipes", db_url=db_url),
)
knowledge_base.load(recreate=True)  # 初回実行後にコメントアウトしてください

agent = Agent(
    model=WatsonX(id="ibm/granite-20b-code-instruct"),
    knowledge=knowledge_base,
    show_tool_calls=True,
)
agent.print_response("How to make Thai curry?", markdown=True)
```

## 使用方法

<Steps>
  <Snippet file="create-venv-step.mdx" />

  <Step title="API キーの設定">
    ```bash
    export IBM_WATSONX_API_KEY=xxx
    export IBM_WATSONX_PROJECT_ID=xxx
    ```
  </Step>

  <Step title="ライブラリのインストール">
    ```bash
    pip install -U ibm-watsonx-ai sqlalchemy pgvector psycopg pypdf openai agno
    ```
  </Step>

  <Step title="pgvector を使用した PostgreSQL のセットアップ">
    pgvector 拡張機能がインストールされた PostgreSQL データベースが必要です。コード内の `db_url` をデータベース構成に合わせて調整してください。
  </Step>

  <Step title="エージェントの実行">
    <CodeGroup>
    ```bash Mac
    python cookbook/models/watsonx/knowledge.py
    ```

    ```bash Windows
    python cookbook\models\watsonx\knowledge.py
    ```
    </CodeGroup>
  </Step>

  <Step title="後続の実行について">
    初回実行後、PDF の再読み込みを避けるために `knowledge_base.load(recreate=True)` 行をコメントアウトしてください。
  </Step>
</Steps>

この例では、知識ベースを IBM WatsonX と統合する方法を示しています。URL から PDF を読み込み、それをベクトルデータベース（この場合は pgvector を使用した PostgreSQL）に処理し、その知識ベースにクエリを実行できるエージェントを作成します。

注意: いくつかのパッケージ (`pgvector`, `pypdf` など) をインストールし、pgvector 拡張機能が利用可能な PostgreSQL データベースを用意する必要があります。
