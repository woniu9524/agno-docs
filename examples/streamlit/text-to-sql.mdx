title: SQL Agent

この例では、以下を行うテキストからSQLへのシステムを構築する方法を示します。
1. エージェント型RAGを使用して、テーブルメタデータ、サンプルクエリ、およびより良いSQLクエリを作成するためのルールを検索します。
2. 動的な少数ショットの例とルールを使用して、クエリ作成を改善します。
3. ユーザーがデータベースにクエリを実行するためのインタラクティブなStreamlit UIを提供します。

例としてF1データセットを使用しますが、他のデータセットにも簡単に拡張できます。

### 主要な機能
- 自然言語からSQLへの変換
- エージェント型RAGを使用したテーブルメタデータ、サンプルクエリ、ルールの取得
- 動的な少数ショットの例とルールによるクエリ作成の改善
- インタラクティブなStreamlit UI

### 試してみる簡単なクエリ
- 「レースで最も多く勝利したトップ5のドライバーは誰ですか？」
- 「コンストラクターズチャンピオンシップにおけるメルセデス対フェラーリのパフォーマンスを比較してください」
- 「モンツァでのファステストラップの進行状況を表示してください」
- 「複数のチームでチャンピオンシップを獲得したドライバーは誰ですか？」
- 「最も多くのレースを開催したトラックは何ですか？」
- 「ルイス・ハミルトンのシーズンごとの勝率を表示してください」

### テーブル結合を使用した高度なクエリ
- 「チャンピオンシップ優勝者は毎年何レース勝ちましたか？」
- 「2019年のコンストラクターのレース勝利数とチャンピオンシップ順位を比較してください」
- 「ルイス・ハミルトンの年ごとのレース勝利数とチャンピオンシップ順位を表示してください」
- 「モナコでレースに勝利し、ファステストラップを記録したドライバーは誰ですか？」
- 「2015年から2020年までのフェラーリのレース勝利数とコンストラクターチャンピオンシップ順位を表示してください」

## コード

完全なコードは[Agnoリポジトリ](https://github.com/agno-agi/agno)で入手できます。

## 使用方法

<Steps>
  <Step title="リポジトリをクローンする">
    ```bash
    git clone https://github.com/agno-agi/agno.git
    cd agno
    ```
  </Step>

  <Step title="仮想環境を作成する">
    ```bash
    python3 -m venv .venv
    source .venv/bin/activate
    ```
  </Step>

  <Step title="依存関係をインストールする">
    ```bash
    pip install -r cookbook/examples/streamlit_apps/sql_agent/requirements.txt
    ```
  </Step>

  <Step title="PgVectorを実行する">
    まず、[Docker Desktop](https://docs.docker.com/desktop/install/mac-install/)をインストールします。

    次に、ヘルパースクリプトを使用して実行するか、
    ```bash
    ./cookbook/scripts/run_pgvector.sh
    ```

    またはDockerで直接実行します。
    ```bash
    docker run -d \
      -e POSTGRES_DB=ai \
      -e POSTGRES_USER=ai \
      -e POSTGRES_PASSWORD=ai \
      -e PGDATA=/var/lib/postgresql/data/pgdata \
      -v pgvolume:/var/lib/postgresql/data \
      -p 5532:5432 \
      --name pgvector \
      agnohq/pgvector:16
    ```
  </Step>

  <Step title="F1データをロードする">
    ```bash
    python cookbook/examples/streamlit_apps/sql_agent/load_f1_data.py
    ```
  </Step>

  <Step title="ナレッジベースをロードする">
    ナレッジベースには、エージェントがより良い応答を生成するのに役立つテーブルメタデータ、ルール、およびサンプルクエリが含まれています。

    ```bash
    python cookbook/examples/streamlit_apps/sql_agent/load_knowledge.py
    ```

    ナレッジベースを強化するためのヒント:
    - クエリ形式に関するエージェントをガイドするために `table_rules` と `column_rules` を追加します
    - `cookbook/examples/apps/sql_agent/knowledge_base/sample_queries.sql` にサンプルクエリを追加します
  </Step>

  <Step title="APIキーを設定する">
    ```bash
    # 必須
    export OPENAI_API_KEY=***

    # オプション
    export ANTHROPIC_API_KEY=***
    export GOOGLE_API_KEY=***
    export GROQ_API_KEY=***
    ```
    最適なパフォーマンスを得るには、gpt-4oの使用をお勧めします。
  </Step>

  <Step title="アプリを起動する">
    ```bash
    streamlit run cookbook/examples/streamlit_apps/sql_agent/app.py
    ```
    [localhost:8501](http://localhost:8501) を開いてSQLエージェントの使用を開始します。
  </Step>
</Steps>

ヘルプが必要ですか？サポートについては、[Discourseコミュニティ](https://community.agno.com)にご参加ください！
