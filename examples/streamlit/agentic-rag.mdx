title: Agentic RAG
---

此示例应用程序展示了如何构建一个复杂的 RAG（检索增强生成）系统，该系统利用知识库搜索和 LLM 来提供对数据的深度洞察。


## 该代理可以：
- 处理和理解来自多个来源（PDF、网站、文本文件）的文档
- 使用向量嵌入构建可搜索的知识库
- 维护跨会话的对话上下文和记忆
- 为其响应提供相关的引用和来源
- 生成摘要并提取关键见解
- 回答后续问题和澄清

## 该代理使用：
- 向量相似性搜索以检索相关文档
- 对话记忆以提供上下文响应
- 引用跟踪以进行来源归属
- 动态知识库更新

## 示例查询：
- “本文档的关键要点是什么？”
- “你能总结主要论点和支持证据吗？”
- “重要的统计数据和发现是什么？”
- “这与[主题 X]有何关系？”
- “此分析的局限性或差距是什么？”
- “你能更详细地解释[概念 X]吗？”
- “还有哪些其他来源支持或反驳这些说法？”

## 用法

<Steps>
  <Step title="克隆存储库">
    ```bash
    git clone https://github.com/agno-agi/agno.git
    cd agno
    ```
  </Step>

  <Step title="创建虚拟环境">
    ```bash
    python3 -m venv .venv
    source .venv/bin/activate
    ```
  </Step>

  <Step title="安装依赖项">
    ```bash
    pip install -r cookbook/examples/streamlit_apps/agentic_rag/requirements.txt
    ```
  </Step>

  <Step title="运行 PgVector">
    首先，安装 [Docker Desktop](https://docs.docker.com/desktop/install/mac-install/)。

    然后使用辅助脚本运行：
    ```bash
    ./cookbook/scripts/run_pgvector.sh
    ```

    或者直接使用 Docker 运行：
    ```bash
    docker run -d \
      -e POSTGRES_DB=ai \
      -e POSTGRES_USER=ai \
      -e POSTGRES_PASSWORD=ai \
      -e PGDATA=/var/lib/postgresql/data/pgdata \
      -v pgvolume:/var/lib/postgresql/data \
      -p 5532:5432 \
      --name pgvector \
      agnohq/pgvector:16
    ```
  </Step>


  <Step title="设置 API 密钥">
    ```bash
    # Required
    export OPENAI_API_KEY=***
    # Optional
    export ANTHROPIC_API_KEY=***
    export GOOGLE_API_KEY=***

    ```
    我们建议使用 gpt-4o 以获得最佳性能。
  </Step>

  <Step title="启动应用程序">
    ```bash
    streamlit run cookbook/examples/streamlit_apps/agentic_rag/app.py
    ```
    打开 [localhost:8501](http://localhost:8501) 开始使用 Agentic RAG。
  </Step>
</Steps>
