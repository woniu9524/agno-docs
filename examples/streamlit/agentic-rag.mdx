このサンプルアプリケーションは、知識ベースの検索とLLMを活用してデータに関する深い洞察を提供する、洗練されたRAG（Retrieval Augmented Generation）システムを構築する方法を示しています。


## エージェントができること：
- 複数のソース（PDF、ウェブサイト、テキストファイル）からのドキュメントを処理・理解する
- ベクトル埋め込みを使用して検索可能な知識ベースを構築する
- セッション間で会話のコンテキストとメモリを維持する
- 回答に関連する引用元とソースを提供する
- 要約を生成し、主要な洞察を抽出する
- フォローアップの質問や明確化に回答する

## エージェントが使用するもの：
- 関連ドキュメント検索のためのベクトル類似性検索
- コンテキスト応答のための会話メモリ
- ソース帰属のための引用追跡
- 動的な知識ベース更新

## コード

完全なコードは[Agnoリポジトリ](https://github.com/agno-agi/agno)で入手できます。

## 使用方法

<Steps>
  <Step title="リポジトリをクローンする">
    ```bash
    git clone https://github.com/agno-agi/agno.git
    cd agno
    ```
  </Step>

  <Step title="仮想環境を作成する">
    ```bash
    python3 -m venv .venv
    source .venv/bin/activate
    ```
  </Step>

  <Step title="依存関係をインストールする">
    ```bash
    pip install -r cookbook/examples/streamlit_apps/agentic_rag/requirements.txt
    ```
  </Step>

  <Step title="PgVectorを実行する">
    まず、[Docker Desktop](https://docs.docker.com/desktop/install/mac-install/)をインストールしてください。

    次に、ヘルパースクリプトを使用して実行するか、
    ```bash
    ./cookbook/scripts/run_pgvector.sh
    ```

    またはDockerで直接実行します：
    ```bash
    docker run -d \
      -e POSTGRES_DB=ai \
      -e POSTGRES_USER=ai \
      -e POSTGRES_PASSWORD=ai \
      -e PGDATA=/var/lib/postgresql/data/pgdata \
      -v pgvolume:/var/lib/postgresql/data \
      -p 5532:5432 \
      --name pgvector \
      agnohq/pgvector:16
    ```
  </Step>


  <Step title="APIキーを設定する">
    ```bash
    # 必須
    export OPENAI_API_KEY=***
    # オプション
    export ANTHROPIC_API_KEY=***
    export GOOGLE_API_KEY=***

    ```
    最適なパフォーマンスのためには、gpt-4oの使用をお勧めします。
  </Step>

  <Step title="アプリを起動する">
    ```bash
    streamlit run cookbook/examples/streamlit_apps/agentic_rag/app.py
    ```
    [localhost:8501](http://localhost:8501)を開いてAgentic RAGの使用を開始してください。
  </Step>
</Steps>

ヘルプが必要ですか？サポートについては、[Discourseコミュニティ](https://community.agno.com)にご参加ください！
