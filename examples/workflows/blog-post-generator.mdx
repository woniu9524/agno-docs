この高度な例では、Webリサーチ機能とプロフェッショナルなライティングスキルを組み合わせた、洗練されたブログ記事ジェネレーターの構築方法を実演します。このワークフローは、マルチステージアプローチを採用しています。
1. 知的なWebリサーチとソース収集
2. コンテンツ抽出と処理
3. 適切な引用を伴うプロフェッショナルなブログ記事ライティング

主な機能：
- 高度なWebリサーチとソース評価
- コンテンツスクレイピングと処理
- SEO最適化を伴うプロフェッショナルライティング
- 効率化のための自動コンテンツキャッシュ
- ソースの帰属と事実検証

試してみるブログトピックの例：
- 「人工汎用知能の台頭：最新のブレークスルー」
- 「量子コンピューティングがいかにサイバーセキュリティに革命を起こしているか」
- 「2024年の持続可能な生活：カーボンフットプリントを削減するための実践的なヒント」
- 「仕事の未来：AIと人間のコラボレーション」
- 「宇宙旅行：SFから現実へ」
- 「デジタル時代のマインドフルネスとメンタルヘルス」
- 「電気自動車の進化：現状と将来のトレンド」

依存関係をインストールするには、`pip install openai duckduckgo-search newspaper4k lxml_html_clean sqlalchemy agno` を実行してください。

```python blog_post_generator.py
import json
from textwrap import dedent
from typing import Dict, Iterator, Optional

from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.storage.sqlite import SqliteStorage
from agno.tools.duckduckgo import DuckDuckGoTools
from agno.tools.newspaper4k import Newspaper4kTools
from agno.utils.log import logger
from agno.utils.pprint import pprint_run_response
from agno.workflow import RunEvent, RunResponse, Workflow
from pydantic import BaseModel, Field


class NewsArticle(BaseModel):
    title: str = Field(..., description="記事のタイトル。")
    url: str = Field(..., description="記事へのリンク。")
    summary: Optional[str] = Field(
        ..., description="利用可能な場合の記事の要約。"
    )


class SearchResults(BaseModel):
    articles: list[NewsArticle]


class ScrapedArticle(BaseModel):
    title: str = Field(..., description="記事のタイトル。")
    url: str = Field(..., description="記事へのリンク。")
    summary: Optional[str] = Field(
        ..., description="利用可能な場合の記事の要約。"
    )
    content: Optional[str] = Field(
        ...,
        description="Markdown形式の完全な記事コンテンツ。コンテンツが利用できない場合はNone。",
    )


class BlogPostGenerator(Workflow):
    """適切なリサーチと引用を伴うプロフェッショナルなブログ記事を生成するための高度なワークフロー。"""

    description: str = dedent("""\
    インテリジェントなブログ記事ジェネレーターで、魅力的で十分にリサーチされたコンテンツを作成します。
    このワークフローは、複数のAIエージェントをオーケストレーションして、
    ジャーナリスティックな厳密さと魅力的なストーリーテリングを組み合わせた、説得力のあるブログ記事をリサーチ、分析、作成します。
    このシステムは、情報提供とデジタル消費の両方に最適化されたコンテンツの作成に優れています。
    """)

    # Search Agent: インテリジェントなWeb検索とソース収集を処理します
    searcher: Agent = Agent(
        model=OpenAIChat(id="gpt-4o-mini"),
        tools=[DuckDuckGoTools()],
        description=dedent("""\
        あなたはBlogResearch-Xです。魅力的なブログコンテンツの高品質なソースを発見することに特化したエリートリサーチアシスタントです。あなたの専門知識には以下が含まれます：

        - 権威がありトレンドのソースを見つける
        - コンテンツの信頼性と関連性を評価する
        - 多様な視点と専門家の意見を特定する
        - ユニークな角度と洞察を発見する
        - 包括的なトピックカバレッジを保証する\
        """),
        instructions=dedent("""\
        1. 検索戦略 🔍
           - 関連性の高いソースを10〜15件見つけ、最も良い5〜7件を選択する
           - 最近の権威あるコンテンツを優先する
           - ユニークな角度と専門家の洞察を探す
        2. ソース評価 📊
           - ソースの信頼性と専門知識を確認する
           - 最新性のために公開日を確認する
           - コンテンツの深さと独自性を評価する
        3. 多様な視点 🌐
           - 異なる視点を含める
           - メインストリームと専門家の意見の両方を収集する
           - 裏付けとなるデータと統計を見つける\
        """),
        response_model=SearchResults,
    )

    # Content Scraper: 記事コンテンツを抽出し処理します
    article_scraper: Agent = Agent(
        model=OpenAIChat(id="gpt-4o-mini"),
        tools=[Newspaper4kTools()],
        description=dedent("""\
        あなたはContentBot-Xです。ブログ作成のためのコンテンツ抽出と処理の専門家です。あなたの専門知識には以下が含まれます：

        - 効率的なコンテンツ抽出
        - スマートなフォーマットと構造化
        - 主要情報の特定
        - 引用と統計の保持
        - ソースの帰属の維持\
        """),
        instructions=dedent("""\
        1. コンテンツ抽出 📑
           - 記事からコンテンツを抽出する
           - 重要な引用と統計を保持する
           - 適切な帰属を維持する
           - ペイウォールをうまく処理する
        2. コンテンツ処理 🔄
           - テキストをクリーンなMarkdownでフォーマットする
           - 主要情報を保持する
           - コンテンツを論理的に構造化する
        3. 品質管理 ✅
           - コンテンツの関連性を確認する
           - 正確な抽出を保証する
           - 可読性を維持する\
        """),
        response_model=ScrapedArticle,
    )

    # Content Writer Agent: リサーチから魅力的なブログ記事を作成します
    writer: Agent = Agent(
        model=OpenAIChat(id="gpt-4o"),
        description=dedent("""\
        あなたはBlogMaster-Xです。ジャーナリスティックな卓越性とデジタルマーケティングの専門知識を組み合わせたエリートコンテンツクリエイターです。あなたの強みは以下を含みます：

        - バイラル級の見出しを作成する
        - 魅力的な導入文を書く
        - デジタル消費のためにコンテンツを構造化する
        - リサーチをシームレスに組み込む
        - 品質を維持しながらSEOを最適化する
        - シェア可能な結論を作成する\
        """),
        instructions=dedent("""\
        1. コンテンツ戦略 📝
           - 注意を引く見出しを作成する
           - 説得力のある導入文を書く
           - エンゲージメントのためにコンテンツを構造化する
           - 関連する小見出しを含める
        2. ライティングの卓越性 ✍️
           - 専門知識とアクセシビリティのバランスをとる
           - 明確で魅力的な言語を使用する
           - 関連する例を含める
           - 統計を自然に組み込む
        3. ソース統合 🔍
           - ソースを適切に引用する
           - 専門家の引用を含める
           - 事実の正確性を維持する
        4. デジタル最適化 💻
           - スキャン可能性のために構造化する
           - シェア可能な要点を追加する
           - SEOを最適化する
           - 魅力的な小見出しを追加する\
        """),
        expected_output=dedent("""\
        # {バイラル級の見出し}

        ## 導入
        {魅力的なフックとコンテキスト}

        ## {説得力のあるセクション1}
        {主要な洞察と分析}
        {専門家の引用と統計}

        ## {魅力的なセクション2}
        {より深い探求}
        {実例}

        ## {実践的なセクション3}
        {実行可能な洞察}
        {専門家の推奨事項}

        ## 主要な要点
        - {共有可能な洞察1}
        - {実践的な要点2}
        - {注目すべき発見3}

        ## ソース
        {適切に帰属表示されたソースとリンク}\
        """),
        markdown=True,
    )

    def run(
        self,
        topic: str,
        use_search_cache: bool = True,
        use_scrape_cache: bool = True,
        use_cached_report: bool = True,
    ) -> Iterator[RunResponse]:
        logger.info(f"ブログ記事を生成中: {topic}")

        # use_cacheがTrueの場合、キャッシュされたブログ記事を使用します
        if use_cached_report:
            cached_blog_post = self.get_cached_blog_post(topic)
            if cached_blog_post:
                yield RunResponse(
                    content=cached_blog_post, event=RunEvent.workflow_completed
                )
                return

        # トピックに関する記事をWebで検索します
        search_results: Optional[SearchResults] = self.get_search_results(
            topic, use_search_cache
        )
        # トピックのsearch_resultsが見つからない場合は、ワークフローを終了します
        if search_results is None or len(search_results.articles) == 0:
            yield RunResponse(
                event=RunEvent.workflow_completed,
                content=f"申し訳ありませんが、トピックに関する記事が見つかりませんでした: {topic}",
            )
            return

        # 検索結果をスクレイピングします
        scraped_articles: Dict[str, ScrapedArticle] = self.scrape_articles(
            topic, search_results, use_scrape_cache
        )

        # ライターへの入力を準備します
        writer_input = {
            "topic": topic,
            "articles": [v.model_dump() for v in scraped_articles.values()],
        }

        # ライターを実行し、レスポンスをyieldします
        yield from self.writer.run(json.dumps(writer_input, indent=4), stream=True)

        # ブログ記事をキャッシュに保存します
        self.add_blog_post_to_cache(topic, self.writer.run_response.content)

    def get_cached_blog_post(self, topic: str) -> Optional[str]:
        logger.info("キャッシュされたブログ記事が存在するか確認中")

        return self.session_state.get("blog_posts", {}).get(topic)

    def add_blog_post_to_cache(self, topic: str, blog_post: str):
        logger.info(f"トピックのブログ記事を保存中: {topic}")
        self.session_state.setdefault("blog_posts", {})
        self.session_state["blog_posts"][topic] = blog_post

    def get_cached_search_results(self, topic: str) -> Optional[SearchResults]:
        logger.info("キャッシュされた検索結果が存在するか確認中")
        search_results = self.session_state.get("search_results", {}).get(topic)
        return (
            SearchResults.model_validate(search_results)
            if search_results and isinstance(search_results, dict)
            else search_results
        )

    def add_search_results_to_cache(self, topic: str, search_results: SearchResults):
        logger.info(f"トピックの検索結果を保存中: {topic}")
        self.session_state.setdefault("search_results", {})
        self.session_state["search_results"][topic] = search_results

    def get_cached_scraped_articles(
        self, topic: str
    ) -> Optional[Dict[str, ScrapedArticle]]:
        logger.info("キャッシュされたスクレイピング済み記事が存在するか確認中")
        scraped_articles = self.session_state.get("scraped_articles", {}).get(topic)
        return (
            ScrapedArticle.model_validate(scraped_articles)
            if scraped_articles and isinstance(scraped_articles, dict)
            else scraped_articles
        )

    def add_scraped_articles_to_cache(
        self, topic: str, scraped_articles: Dict[str, ScrapedArticle]
    ):
        logger.info(f"トピックのスクレイピング済み記事を保存中: {topic}")
        self.session_state.setdefault("scraped_articles", {})
        self.session_state["scraped_articles"][topic] = scraped_articles

    def get_search_results(
        self, topic: str, use_search_cache: bool, num_attempts: int = 3
    ) -> Optional[SearchResults]:
        # use_search_cacheがTrueの場合、セッションステートからキャッシュされたsearch_resultsを取得します
        if use_search_cache:
            try:
                search_results_from_cache = self.get_cached_search_results(topic)
                if search_results_from_cache is not None:
                    search_results = SearchResults.model_validate(
                        search_results_from_cache
                    )
                    logger.info(
                        f"キャッシュから{len(search_results.articles)}件の記事が見つかりました。"
                    )
                    return search_results
            except Exception as e:
                logger.warning(f"キャッシュから検索結果を読み取れませんでした: {e}")

        # キャッシュされたsearch_resultsがない場合、searcherを使用して最新の記事を見つけます
        for attempt in range(num_attempts):
            try:
                searcher_response: RunResponse = self.searcher.run(topic)
                if (
                    searcher_response is not None
                    and searcher_response.content is not None
                    and isinstance(searcher_response.content, SearchResults)
                ):
                    article_count = len(searcher_response.content.articles)
                    logger.info(
                        f"{attempt + 1}回目の試行で{article_count}件の記事が見つかりました"
                    )
                    # 検索結果をキャッシュします
                    self.add_search_results_to_cache(topic, searcher_response.content)
                    return searcher_response.content
                else:
                    logger.warning(
                        f"{attempt + 1}/{num_attempts}回目の試行が失敗しました: 無効なレスポンスタイプ"
                    )
            except Exception as e:
                logger.warning(f"{attempt + 1}/{num_attempts}回目の試行が失敗しました: {str(e)}")

        logger.error(f"{num_attempts}回の試行後、検索結果の取得に失敗しました")
        return None

    def scrape_articles(
        self, topic: str, search_results: SearchResults, use_scrape_cache: bool
    ) -> Dict[str, ScrapedArticle]:
        scraped_articles: Dict[str, ScrapedArticle] = {}

        # use_scrape_cacheがTrueの場合、セッションステートからキャッシュされたscraped_articlesを取得します
        if use_scrape_cache:
            try:
                scraped_articles_from_cache = self.get_cached_scraped_articles(topic)
                if scraped_articles_from_cache is not None:
                    scraped_articles = scraped_articles_from_cache
                    logger.info(
                        f"キャッシュから{len(scraped_articles)}件のスクレイピング済み記事が見つかりました。"
                    )
                    return scraped_articles
            except Exception as e:
                logger.warning(f"キャッシュからスクレイピング済み記事を読み取れませんでした: {e}")

        # キャッシュにない記事をスクレイピングします
        for article in search_results.articles:
            if article.url in scraped_articles:
                logger.info(f"キャッシュにスクレイピング済み記事が見つかりました: {article.url}")
                continue

            article_scraper_response: RunResponse = self.article_scraper.run(
                article.url
            )
            if (
                article_scraper_response is not None
                and article_scraper_response.content is not None
                and isinstance(article_scraper_response.content, ScrapedArticle)
            ):
                scraped_articles[article_scraper_response.content.url] = (
                    article_scraper_response.content
                )
                logger.info(f"スクレイピング済み記事: {article_scraper_response.content.url}")

        # セッションステートにスクレイピング済み記事を保存します
        self.add_scraped_articles_to_cache(topic, scraped_articles)
        return scraped_articles


# スクリプトが直接実行された場合にワークフローを実行します
if __name__ == "__main__":
    import random

    from rich.prompt import Prompt

    # ジェネレーターの汎用性を示すための楽しい例のプロンプト
    example_prompts = [
        "なぜ猫は秘密裏にインターネットを運営しているのか",
        "ピザが午前2時に美味しくなる科学的根拠",
        "タイムトラベラーのための現代ソーシャルメディアガイド",
        "ラバーダックがいかにソフトウェア開発に革命をもたらしたか",
        "オフィス植物の秘密結社：サバイバルガイド",
        "犬が私たちを臭いに弱いと思っている理由",
        "コーヒーショップのWiFiパスワードのアンダーグラウンドエコノミー",
        "時代を超えたダジャレの歴史的分析",
    ]

    # ユーザーからトピックを取得します
    topic = Prompt.ask(
        "[bold]ブログ記事のトピックを入力してください[/bold] (またはランダムな例を表示するにはEnterキーを押してください)\n✨",
        default=random.choice(example_prompts),
    )

    # セッションIDで使用するためにトピックをURLセーフな文字列に変換します
    url_safe_topic = topic.lower().replace(" ", "-")

    # ブログ記事ジェネレーターワークフローを初期化します
    # - トピックに基づいて一意のセッションIDを作成します
    # - 結果のキャッシュのためにSQLiteストレージを設定します
    generate_blog_post = BlogPostGenerator(
        session_id=f"generate-blog-post-on-{url_safe_topic}",
        storage=SqliteStorage(
            table_name="generate_blog_post_workflows",
            db_file="tmp/agno_workflows.db",
        ),
        debug_mode=True,
    )

    # キャッシュを有効にしてワークフローを実行します
    # 生成されたコンテンツを含むRunResponseオブジェクトのイテレーターを返します
    blog_post: Iterator[RunResponse] = generate_blog_post.run(
        topic=topic,
        use_search_cache=True,
        use_scrape_cache=True,
        use_cached_report=True,
    )

    # レスポンスを表示します
    pprint_run_response(blog_post, markdown=True)
```
