この例では、以下の機能を組み合わせた高度なリサーチワークフローの構築方法を示します。
🔍 関連ソースを見つけるためのウェブ検索機能
📚 コンテンツの抽出と処理
✍️ 学術的なレポート生成
💾 パフォーマンス向上のためのスマートキャッシュ

無料で利用可能な以下のツールを使用しました。
- DuckDuckGoTools: 関連する記事をウェブで検索します
- Newspaper4kTools: 記事のコンテンツをスクレイピングして処理します

試すことができる研究トピックの例：
- "量子コンピューティングの最新動向は？"
- "人工意識の現状を調査する"
- "核融合エネルギーにおける最近のブレークスルーを分析する"
- "宇宙旅行の環境への影響を調査する"
- "長寿研究の最新の発見を探る"

## コード

```python research_workflow.py
import json
from textwrap import dedent
from typing import Dict, Iterator, Optional

from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.storage.workflow.sqlite import SqliteWorkflowStorage
from agno.tools.duckduckgo import DuckDuckGoTools
from agno.tools.newspaper4k import Newspaper4kTools
from agno.utils.log import logger
from agno.utils.pprint import pprint_run_response
from agno.workflow import RunEvent, RunResponse, Workflow
from pydantic import BaseModel, Field


class Article(BaseModel):
    title: str = Field(..., description="記事のタイトル。")
    url: str = Field(..., description="記事へのリンク。")
    summary: Optional[str] = Field(
        ..., description="利用可能な場合は記事の要約。"
    )


class SearchResults(BaseModel):
    articles: list[Article]


class ScrapedArticle(BaseModel):
    title: str = Field(..., description="記事のタイトル。")
    url: str = Field(..., description="記事へのリンク。")
    summary: Optional[str] = Field(
        ..., description="利用可能な場合は記事の要約。"
    )
    content: Optional[str] = Field(
        ...,
        description="利用可能な場合はマークダウン形式のコンテンツ。コンテンツが利用できないか、意味をなさない場合はNoneを返します。",
    )


class ResearchReportGenerator(Workflow):
    description: str = dedent("""\
    学術的な厳密さと魅力的なストーリーテリングを組み合わせた包括的な研究レポートを生成します。このワークフローは、複数のAIエージェントをオーケストレーションして、多様なソースからの情報を検索、分析、合成し、構造化されたレポートにまとめます。
    """)

    web_searcher: Agent = Agent(
        model=OpenAIChat(id="gpt-4o-mini"),
        tools=[DuckDuckGoTools()],
        description=dedent("""\
        あなたは、学術的および科学的なソースを発見し評価する専門家であるResearchBot-Xです。
        """),
        instructions=dedent("""\
        あなたは、ソース評価の専門知識を持つ、細心の注意を払うリサーチアシスタントです！🔍
        10〜15件のソースを検索し、最も権威があり関連性の高い5〜7件を特定してください。
        優先順位：
        - 査読付き論文および学術論文
        - 信頼できる機関からの最近の開発状況
        - 権威あるニュースソースおよび専門家の解説
        - 認識されている専門家からの多様な視点
        意見記事や権威のないソースは避けてください。
        """),
        response_model=SearchResults,
    )

    article_scraper: Agent = Agent(
        model=OpenAIChat(id="gpt-4o-mini"),
        tools=[Newspaper4kTools()],
        description=dedent("""\
        あなたは、学術コンテンツの抽出と構造化の専門家であるContentBot-Xです。
        """),
        instructions=dedent("""\
        あなたは、学術的な詳細に注意を払う正確なコンテンツキュレーターです！📚
        コンテンツを処理する際：
           - 記事からコンテンツを抽出する
           - 学術的な引用と参考文献を保持する
           - 技術的な用語の正確性を維持する
           - 明確なセクションでコンテンツを論理的に構造化する
           - 主要な発見と方法論の詳細を抽出する
           - ペイウォールコンテンツを適切に処理する
        読みやすさを最適化するために、すべてをクリーンなマークダウン形式でフォーマットしてください。
        """),
        response_model=ScrapedArticle,
    )

    writer: Agent = Agent(
        model=OpenAIChat(id="gpt-4o"),
        description=dedent("""\
        あなたは、学術的な厳密さと魅力的なナラティブスタイルを組み合わせた著名なAIリサーチサイエンティストであるProfessor X-2000です。
        """),
        instructions=dedent("""\
        世界クラスの学術研究者の専門知識をチャネルしてください！
        🎯 分析フェーズ：
          - ソースの信頼性と関連性を評価する
          - ソース間の発見を相互参照する
          - 主要なテーマとブレークスルーを特定する
        💡 合成フェーズ：
          - 一貫したナラティブフレームワークを開発する
          - 離散的な発見を接続する
          - 矛盾またはギャップを強調する
        ✍️ ライティングフェーズ：
          - 魅力的なエグゼクティブサマリーで始め、読者を引き込む
          - 複雑なアイデアを明確に提示する
          - すべての主張を引用で裏付ける
          - 深さとアクセシビリティのバランスを取る
          - 学術的なトーンを維持しつつ、読みやすさを確保する
          - 影響と将来の方向性で締めくくる
        """),
        expected_output=dedent("""\
        # {魅力的な学術タイトル}

        ## エグゼクティブサマリー
        {主要な発見と重要性の簡潔な概要}

        ## 導入
        {研究の文脈と背景}
        {分野の現状}

        ## 方法論
        {検索および分析アプローチ}
        {ソース評価基準}

        ## 主要な発見
        {主要な発見と開発}
        {裏付けとなる証拠と分析}
        {対立する見解}

        ## 分析
        {発見の批判的評価}
        {複数の視点の統合}
        {パターンとトレンドの特定}

        ## 影響
        {学術的および実践的な重要性}
        {将来の研究の方向性}
        {潜在的な応用}

        ## 主要なポイント
        - {重要な発見1}
        - {重要な発見2}
        - {重要な発見3}

        ## 参考文献
        {適切にフォーマットされた学術引用}

        ---
        レポート生成者：Professor X-2000
        先端研究部門
        日付：{current_date}
        """),
        markdown=True,
    )

    def run(
        self,
        topic: str,
        use_search_cache: bool = True,
        use_scrape_cache: bool = True,
        use_cached_report: bool = True,
    ) -> Iterator[RunResponse]:
        """
        指定されたトピックに関する包括的なニュースレポートを生成します。

        この関数は、記事を検索し、そのコンテンツをスクレイピングし、最終レポートを生成するワークフローをオーケストレーションします。パフォーマンスを最適化するためにキャッシュメカニズムを利用します。

        Args:
            topic (str): ニュースレポートを生成するトピック。
            use_search_cache (bool, optional): キャッシュされた検索結果を使用するかどうか。デフォルトはTrue。
            use_scrape_cache (bool, optional): キャッシュされたスクレイピング済み記事を使用するかどうか。デフォルトはTrue。
            use_cached_report (bool, optional): 同じトピックに関する以前に生成されたレポートを使用するかどうか。デフォルトはFalse。

        Returns:
            Iterator[RunResponse]: 生成されたレポートまたはステータス情報を含むオブジェクトのストリーム。

        ステップ：
        1. use_cached_reportがTrueの場合、キャッシュされたレポートを確認します。
        2. トピックに関する記事をウェブで検索します：
            - キャッシュされた検索結果が利用可能でuse_search_cacheがTrueの場合、それを使用します。
            - それ以外の場合は、新しいウェブ検索を実行します。
        3. 各記事のコンテンツをスクレイピングします：
            - キャッシュされたスクレイピング済み記事が利用可能でuse_scrape_cacheがTrueの場合、それを使用します。
            - キャッシュにない新しい記事をスクレイピングします。
        4. スクレピングされた記事のコンテンツを使用して最終レポートを生成します。

        この関数は、`session_state`を使用してキャッシュデータを保存および取得します。
        """
        logger.info(f"レポートを生成中: {topic}")

        # use_cached_reportがTrueの場合、キャッシュされたレポートを使用します
        if use_cached_report:
            cached_report = self.get_cached_report(topic)
            if cached_report:
                yield RunResponse(
                    content=cached_report, event=RunEvent.workflow_completed
                )
                return

        # トピックに関する記事をウェブで検索します
        search_results: Optional[SearchResults] = self.get_search_results(
            topic, use_search_cache
        )
        # トピックのsearch_resultsが見つからない場合は、ワークフローを終了します
        if search_results is None or len(search_results.articles) == 0:
            yield RunResponse(
                event=RunEvent.workflow_completed,
                content=f"申し訳ありませんが、トピック '{topic}' に関する記事が見つかりませんでした。",
            )
            return

        # 検索結果をスクレイピングします
        scraped_articles: Dict[str, ScrapedArticle] = self.scrape_articles(
            search_results, use_scrape_cache
        )

        # 研究レポートを作成します
        yield from self.write_research_report(topic, scraped_articles)

    def get_cached_report(self, topic: str) -> Optional[str]:
        logger.info("キャッシュされたレポートが存在するか確認中")
        return self.session_state.get("reports", {}).get(topic)

    def add_report_to_cache(self, topic: str, report: str):
        logger.info(f"トピック '{topic}' のレポートを保存中")
        self.session_state.setdefault("reports", {})
        self.session_state["reports"][topic] = report
        # レポートをストレージに保存します
        self.write_to_storage()

    def get_cached_search_results(self, topic: str) -> Optional[SearchResults]:
        logger.info("キャッシュされた検索結果が存在するか確認中")
        return self.session_state.get("search_results", {}).get(topic)

    def add_search_results_to_cache(self, topic: str, search_results: SearchResults):
        logger.info(f"トピック '{topic}' の検索結果を保存中")
        self.session_state.setdefault("search_results", {})
        self.session_state["search_results"][topic] = search_results.model_dump()
        # 検索結果をストレージに保存します
        self.write_to_storage()

    def get_cached_scraped_articles(
        self, topic: str
    ) -> Optional[Dict[str, ScrapedArticle]]:
        logger.info("キャッシュされたスクレイピング済み記事が存在するか確認中")
        return self.session_state.get("scraped_articles", {}).get(topic)

    def add_scraped_articles_to_cache(
        self, topic: str, scraped_articles: Dict[str, ScrapedArticle]
    ):
        logger.info(f"トピック '{topic}' のスクレイピング済み記事を保存中")
        self.session_state.setdefault("scraped_articles", {})
        self.session_state["scraped_articles"][topic] = scraped_articles
        # スクレピング済み記事をストレージに保存します
        self.write_to_storage()

    def get_search_results(
        self, topic: str, use_search_cache: bool, num_attempts: int = 3
    ) -> Optional[SearchResults]:
        # use_search_cacheがTrueの場合、セッションステートからキャッシュされたsearch_resultsを取得します
        if use_search_cache:
            try:
                search_results_from_cache = self.get_cached_search_results(topic)
                if search_results_from_cache is not None:
                    search_results = SearchResults.model_validate(
                        search_results_from_cache
                    )
                    logger.info(
                        f"キャッシュから{len(search_results.articles)}件の記事が見つかりました。"
                    )
                    return search_results
            except Exception as e:
                logger.warning(f"キャッシュから検索結果を読み取れませんでした: {e}")

        # キャッシュされたsearch_resultsがない場合、web_searcherを使用して最新の記事を見つけます
        for attempt in range(num_attempts):
            try:
                searcher_response: RunResponse = self.web_searcher.run(topic)
                if (
                    searcher_response is not None
                    and searcher_response.content is not None
                    and isinstance(searcher_response.content, SearchResults)
                ):
                    article_count = len(searcher_response.content.articles)
                    logger.info(
                        f"{attempt + 1}回目の試行で{article_count}件の記事が見つかりました"
                    )
                    # 検索結果をキャッシュします
                    self.add_search_results_to_cache(topic, searcher_response.content)
                    return searcher_response.content
                else:
                    logger.warning(
                        f"{attempt + 1}/{num_attempts}回目の試行に失敗しました: 無効なレスポンスタイプ"
                    )
            except Exception as e:
                logger.warning(f"{attempt + 1}/{num_attempts}回目の試行に失敗しました: {str(e)}")

        logger.error(f"{num_attempts}回の試行後、検索結果の取得に失敗しました")
        return None

    def scrape_articles(
        self, search_results: SearchResults, use_scrape_cache: bool
    ) -> Dict[str, ScrapedArticle]:
        scraped_articles: Dict[str, ScrapedArticle] = {}

        # use_scrape_cacheがTrueの場合、セッションステートからキャッシュされたscraped_articlesを取得します
        if use_scrape_cache:
            try:
                scraped_articles_from_cache = self.get_cached_scraped_articles(topic)
                if scraped_articles_from_cache is not None:
                    scraped_articles = scraped_articles_from_cache
                    logger.info(
                        f"キャッシュから{len(scraped_articles)}件のスクレイピング済み記事が見つかりました。"
                    )
                    return scraped_articles
            except Exception as e:
                logger.warning(f"キャッシュからスクレイピング済み記事を読み取れませんでした: {e}")

        # キャッシュにない記事をスクレイピングします
        for article in search_results.articles:
            if article.url in scraped_articles:
                logger.info(f"キャッシュにスクレイピング済み記事が見つかりました: {article.url}")
                continue

            article_scraper_response: RunResponse = self.article_scraper.run(
                article.url
            )
            if (
                article_scraper_response is not None
                and article_scraper_response.content is not None
                and isinstance(article_scraper_response.content, ScrapedArticle)
            ):
                scraped_articles[article_scraper_response.content.url] = (
                    article_scraper_response.content
                )
                logger.info(f"スクレイピングされた記事: {article_scraper_response.content.url}")

        # スクレピングされた記事をセッションステートに保存します
        self.add_scraped_articles_to_cache(topic, scraped_articles)
        return scraped_articles

    def write_research_report(
        self, topic: str, scraped_articles: Dict[str, ScrapedArticle]
    ) -> Iterator[RunResponse]:
        logger.info("研究レポートを作成中")
        # ライターへの入力を準備します
        writer_input = {
            "topic": topic,
            "articles": [v.model_dump() for v in scraped_articles.values()],
        }
        # ライターを実行し、レスポンスをyieldします
        yield from self.writer.run(json.dumps(writer_input, indent=4), stream=True)
        # 研究レポートをキャッシュに保存します
        self.add_report_to_cache(topic, self.writer.run_response.content)


# スクリプトが直接実行された場合にワークフローを実行します
if __name__ == "__main__":
    from rich.prompt import Prompt

    # 例の研究トピック
    example_topics = [
        "quantum computing breakthroughs 2024",
        "artificial consciousness research",
        "fusion energy developments",
        "space tourism environmental impact",
        "longevity research advances",
    ]

    topics_str = "\n".join(
        f"{i + 1}. {topic}" for i, topic in enumerate(example_topics)
    )

    print(f"\n📚 例の研究トピック:\n{topics_str}\n")

    # ユーザーからトピックを取得します
    topic = Prompt.ask(
        "[bold]研究トピックを入力してください[/bold]\n✨",
        default="quantum computing breakthroughs 2024",
    )

    # セッションIDに使用するためにトピックをURLセーフな文字列に変換します
    url_safe_topic = topic.lower().replace(" ", "-")

    # ニュースレポート生成ワークフローを初期化します
    generate_research_report = ResearchReportGenerator(
        session_id=f"generate-report-on-{url_safe_topic}",
        storage=SqliteWorkflowStorage(
            table_name="generate_research_report_workflow",
            db_file="tmp/workflows.db",
        ),
    )

    # キャッシュを有効にしてワークフローを実行します
    report_stream: Iterator[RunResponse] = generate_research_report.run(
        topic=topic,
        use_search_cache=True,
        use_scrape_cache=True,
        use_cached_report=True,
    )

    # レスポンスを表示します
    pprint_run_response(report_stream, markdown=True)
```
