title: Agent State
---

この例では、インタラクション間で状態を維持するエージェントを作成する方法を示します。単純なカウンターメカニズムを実証していますが、このパターンは、会話コンテキスト、ユーザー設定の維持、または複数ステップのプロセスの追跡などの、より複雑な状態管理に拡張できます。

試してみるプロンプトの例：
- 「カウンターを3回増やして、最終的なカウントを教えてください」
- 「現在のカウントはいくつですか？さらに2つ追加してください」
- 「カウンターを5回増やしてください。ただし、各ステップを教えてください」
- 「カウントに4を追加して、開始時点を思い出させてください」
- 「カウンターを2回増やして、私たちの旅を要約してください」

## コード

```python agent_state.py
from textwrap import dedent

from agno.agent import Agent
from agno.models.openai import OpenAIChat


# カウンターを増やして新しい値を返すツールを定義します
def increment_counter(agent: Agent) -> str:
    """セッションカウンターを増やして新しい値を返します。"""
    agent.session_state["count"] += 1
    return f"The count is now {agent.session_state['count']}"


# 状態を維持するステートマネージャーエージェントを作成します
agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    # カウンターを0から開始してセッション状態を初期化します
    session_state={"count": 0},
    tools=[increment_counter],
    # セッション状態の変数を指示で使用できます
    instructions=dedent("""\
        あなたはステートマネージャーです。状態管理の熱心なガイドです！🔄
        あなたの仕事は、単純なカウンターの例を通じてユーザーが状態管理を理解するのを助けることです。

        すべてのインタラクションで次のガイドラインに従ってください：
        1. 関連する場合は常に現在の状態（カウント）を認識してください
        2. increment_counter ツールを使用して状態を変更してください
        3. 状態の変更を明確かつ魅力的な方法で説明してください

        応答を次のように構成してください：
        - 現在の状態ステータス
        - 状態変換アクション
        - 最終状態と観察

        開始状態（カウント）は次のとおりです： {count}\
    """),
    show_tool_calls=True,
    markdown=True,
)

# 使用例
agent.print_response(
    "カウンターを3回増やして、状態の変化を観察しましょう！",
    stream=True,
)

# 試してみる追加のプロンプト：
"""
これらの魅力的な状態管理シナリオを試してください：
1. 「状態を4回更新して、変更を追跡してください」
2. 「カウンターを2回変更して、状態遷移を説明してください」
3. 「3回増やして、状態がどのように永続するかを示してください」
4. 「観察を伴う状態更新を5回実行しましょう」
5. 「カウントに3を追加して、状態管理の概念を説明してください」
"""

print(f"Final session state: {agent.session_state}")
```
