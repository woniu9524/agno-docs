title: データベーステーブル

Agnoテンプレートには、データベースを管理するための[SqlAlchemy](https://www.sqlalchemy.org/)と[alembic](https://alembic.sqlalchemy.org/en/latest/)がプリセットされています。テーブルを追加する一般的なワークフローは以下の通りです。

1. `db/tables` ディレクトリにテーブル定義を追加します。
2. `db/tables/__init__.py` ファイルにテーブルクラスをインポートします。
3. データベースマイグレーションを作成します。
4. データベースマイグレーションを実行します。

## テーブル定義

`UsersTable` を作成しましょう。以下のコードを `db/tables/user.py` にコピーしてください。

```python db/tables/user.py
from datetime import datetime
from typing import Optional

from sqlalchemy.orm import Mapped, mapped_column
from sqlalchemy.sql.expression import text
from sqlalchemy.types import BigInteger, DateTime, String

from db.tables.base import Base


class UsersTable(Base):
    """ユーザーデータを格納するテーブル。"""

    __tablename__ = "dim_users"

    id_user: Mapped[int] = mapped_column(
        BigInteger, primary_key=True, autoincrement=True, nullable=False, index=True
    )
    email: Mapped[str] = mapped_column(String)
    is_active: Mapped[bool] = mapped_column(default=True)
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True), server_default=text("now()")
    )
    updated_at: Mapped[Optional[datetime]] = mapped_column(
        DateTime(timezone=True), onupdate=text("now()")
    )
```

`db/tables/__init__.py` ファイルを更新してください。

```python db/tables/__init__.py
from db.tables.base import Base
from db.tables.user import UsersTable
```

## データベースリビジョンの作成

開発コンテナでデータベースマイグレーションを作成するために、alembic コマンドを実行します。

```bash
docker exec -it ai-api alembic -c db/alembic.ini revision --autogenerate -m "Initialize DB"
```

## 開発データベースのマイグレーション

開発データベースをマイグレーションするために、alembic コマンドを実行します。

```bash
docker exec -it ai-api alembic -c db/alembic.ini upgrade head
```

### オプション: テストユーザーの追加

次にテストユーザーを追加しましょう。以下のコードを `db/tables/test_add_user.py` にコピーしてください。

```python db/tables/test_add_user.py
from typing import Optional
from sqlalchemy.orm import Session

from db.session import SessionLocal
from db.tables.user import UsersTable
from utils.log import logger


def create_user(db_session: Session, email: str) -> UsersTable:
    """新しいユーザーを作成します。"""
    new_user = UsersTable(email=email)
    db_session.add(new_user)
    return new_user


def get_user(db_session: Session, email: str) -> Optional[UsersTable]:
    """メールアドレスでユーザーを取得します。"""
    return db_session.query(UsersTable).filter(UsersTable.email == email).first()


if __name__ == "__main__":
    test_user_email = "test@test.com"
    with SessionLocal() as sess, sess.begin():
        logger.info(f"Creating user: {test_user_email}")
        create_user(db_session=sess, email=test_user_email)
        logger.info(f"Getting user: {test_user_email}")
        user = get_user(db_session=sess, email=test_user_email)
        if user:
            logger.info(f"User created: {user.id_user}")
        else:
            logger.info(f"User not found: {test_user_email}")

```

テストユーザー追加スクリプトを実行します。

```bash
docker exec -it ai-api python db/tables/test_add_user.py
```

## 本番データベースのマイグレーション

本番データベースのマイグレーションは、環境変数 `MIGRATE_DB = True` を設定し、本番サービスを再起動することをお勧めします。これにより、コンテナ起動時のエントリーポイントスクリプトから `alembic -c db/alembic.ini upgrade head` が実行されます。

### `workspace/prd_resources.py` ファイルの更新

```python workspace/prd_resources.py
...
# -*- Build container environment
container_env = {
    ...
    # Migrate database on startup using alembic
    "MIGRATE_DB": ws_settings.prd_db_enabled,
}
...
```

### ECS タスク定義の更新

環境変数を更新したため、タスク定義を更新する必要があります。

<CodeGroup>

```bash terminal
ag ws patch --env prd --infra aws --name td
```

```bash shorthand
ag ws patch -e prd -i aws -n td
```

</CodeGroup>

### ECS サービスの更新

タスク定義を更新した後、本番アプリケーションを再デプロイします。

<CodeGroup>

```bash terminal
ag ws patch --env prd --infra aws --name service
```

```bash shorthand
ag ws patch -e prd -i aws -n service
```

</CodeGroup>

## 本番データベースを手動でマイグレーション

別の方法として、本番コンテナに SSH 接続して手動でマイグレーションを実行します。ECS タスクはすでに SSH アクセスが有効になっています。本番データベースをマイグレーションするために alembic コマンドを実行します。

```bash
ECS_CLUSTER=ai-app-prd-cluster
TASK_ARN=$(aws ecs list-tasks --cluster ai-app-prd-cluster --query "taskArns[0]" --output text)
CONTAINER_NAME=ai-api-prd

aws ecs execute-command --cluster $ECS_CLUSTER \
    --task $TASK_ARN \
    --container $CONTAINER_NAME \
    --interactive \
    --command "alembic -c db/alembic.ini upgrade head"
```

## マイグレーションディレクトリの作成方法

<Note>

これらのコマンドは実行済みであり、完全性のために説明されています。

</Note>

マイグレーションディレクトリは、以下のコマンドを使用して作成されました。

```bash
docker exec -it ai-api cd db && alembic init migrations
```

- 上記コマンドを実行すると、`db/migrations` ディレクトリが作成されます。
- `alembic.ini` を更新します。
  - `script_location = db/migrations` を設定します。
  - `[post_write_hooks]` の `black` フックのコメントを解除します。
- `db/migrations/env.py` ファイルを [このリンク](https://alembic.sqlalchemy.org/en/latest/autogenerate.html) に従って更新します。
- `configure` 関数に以下を追加して、`target_metadata` に含まれるテーブルのみを含めるようにします。

```python db/migrations/env.py
# -*- Only include tables that are in the target_metadata
def include_name(name, type_, parent_names):
    if type_ == "table":
        return name in target_metadata.tables
    else:
        return True
...
