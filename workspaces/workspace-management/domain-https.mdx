## カスタムドメインの使用

1. [Route 53](https://us-east-1.console.aws.amazon.com/route53/)でドメインを登録します。
2. ドメインをロードバランサーのDNSに向けます。

### Streamlitアプリのカスタムドメイン

Route53コンソールでレコードを作成し、`app.[YOUR_DOMAIN]`をStreamlitアプリに向けます。

<img src="/images/llm-app-aidev-run.png" alt="llm-app-aidev-run" />

アプリは[http://app.aidev.run](http://app.aidev.run)でアクセスできます。

<Note>ドメイン名に`http`が含まれていることに注意してください。</Note>

### FastAPIアプリのカスタムドメイン

Route53コンソールでレコードを作成し、`api.[YOUR_DOMAIN]`をFastAPIアプリに向けます。

<img src="/images/llm-api-aidev-run.png" alt="llm-api-aidev-run" />

APIは[http://api.aidev.run](http://api.aidev.run)でアクセスできます。

<Note>ドメイン名に`http`が含まれていることに注意してください。</Note>

## HTTPSの追加

HTTPSを追加するには：

1. [AWS ACM](https://us-east-1.console.aws.amazon.com/acm)を使用して証明書を作成します。`*.[YOUR_DOMAIN]`の証明書をリクエストします。

<img src="/images/llm-app-request-cert.png" alt="llm-app-request-cert" />

2. Route 53にレコードを作成します。

<img src="/images/llm-app-validate-cert.png" alt="llm-app-validate-cert" />

3. 証明書のARNをアプリに追加します。

<Note>アプリに追加する前に、証明書が`Issued`（発行済み）であることを確認してください。</Note>

`llm-app/workspace/prd_resources.py`ファイルを更新し、`FastAPI`および`Streamlit`アプリに`load_balancer_certificate_arn`を追加します。

```python workspace/prd_resources.py

# -*- Streamlit running on ECS
prd_streamlit = Streamlit(
    ...
    # HTTPSを有効にするには、ACM証明書を作成し、以下のARNを追加してください:
    load_balancer_enable_https=True,
    load_balancer_certificate_arn="arn:aws:acm:us-east-1:497891874516:certificate/6598c24a-d4fc-4f17-8ee0-0d3906eb705f",
    ...
)

# -*- FastAPI running on ECS
prd_fastapi = FastApi(
    ...
    # HTTPSを有効にするには、ACM証明書を作成し、以下のARNを追加してください:
    load_balancer_enable_https=True,
    load_balancer_certificate_arn="arn:aws:acm:us-east-1:497891874516:certificate/6598c24a-d4fc-4f17-8ee0-0d3906eb705f",
    ...
)
```

4. 新しいロードバランサーリスナーを作成します。

HTTPS構成を取得するために、ロードバランサーに新しいリスナーを作成します。

<CodeGroup>

```bash terminal
ag ws up --env prd --infra aws --name listener
```

```bash shorthand
ag ws up -e prd -i aws -n listener
```

</CodeGroup>

<Note>適用する前に、証明書が`Issued`（発行済み）である必要があります。</Note>

これにより、カスタムドメインで`https`が機能するようになります。

5. 既存のリスナーを更新してHTTPからHTTPSへのリダイレクトを行います。

<CodeGroup>

```bash terminal
ag ws patch --env prd --infra aws --name listener
```

```bash shorthand
ag ws patch -e prd -i aws -n listener
```

</CodeGroup>

これにより、すべてのHTTPリクエストが自動的にHTTPSにリダイレクトされるようになります。
