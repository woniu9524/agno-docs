---
title: Pythonライブラリの追加
---

Agnoテンプレートは、[pyproject.toml](https://packaging.python.org/en/latest/specifications/declaring-project-metadata/#declaring-project-metadata)ファイルを使用して依存関係を管理するように設定されています。このファイルは、[uv](https://github.com/astral-sh/uv)または[pip-tools](https://pip-tools.readthedocs.io/en/latest/)を使用して`requirements.txt`ファイルを生成するために使用されます。**

Pythonライブラリの追加または更新は、次の2つのステップで行います。

1. `pyproject.toml`ファイルにライブラリを追加する
2. `requirements.txt`ファイルを自動生成する

<Warning>

このプロセスを使用して`requirements.txt`ファイルを自動生成することを強くお勧めします。

</Warning>

## pyproject.tomlの更新

- `pyproject.toml`ファイルを開きます。
- 依存関係セクションに新しいライブラリを追加します。

## requirementsの生成

`pyproject.toml`ファイルで`dependencies`を更新した後、ヘルパースクリプトを実行するか、直接`pip-compile`を実行して`requirements.txt`ファイルを自動生成します。

<CodeGroup>

```bash terminal
./scripts/generate_requirements.sh
```

```bash pip compile
pip-compile \
    --no-annotate \
    --pip-args "--no-cache-dir" \
    -o requirements.txt pyproject.toml
```

</CodeGroup>

すべてのPythonライブラリを最新バージョンにアップグレードしたい場合は、次を実行します。

<CodeGroup>

```bash terminal
./scripts/generate_requirements.sh upgrade
```

```bash pip compile
pip-compile \
    --upgrade \
    --no-annotate \
    --pip-args "--no-cache-dir" \
    -o requirements.txt pyproject.toml
```

</CodeGroup>

## イメージの再構築

`requirements.txt`ファイルを更新した後、イメージを再構築します。

### 開発用イメージの再構築

<CodeGroup>

```bash terminal
ag ws up --env dev --infra docker --type image
```

```bash short options
ag ws up -e dev -i docker -t image
```

</CodeGroup>

### 本番用イメージの再構築

<Note>

必要に応じて、[ECRへの認証](workspaces/workspace-management/production-app#ecr-images)を忘れないでください。

</Note>

<CodeGroup>

```bash terminal
ag ws up --env prd --infra aws --type image
```

```bash short options
ag ws up -e prd -i aws -t image
```

</CodeGroup>

## リソースの再作成

イメージを再構築した後、リソースを再作成します。

### 開発用コンテナの再作成

<CodeGroup>

```bash terminal
ag ws restart --env dev --infra docker --type container
```

```bash short options
ag ws restart -e dev -c docker -t container
```

</CodeGroup>

### ECSサービスの更新

<CodeGroup>

```bash terminal
ag ws patch --env prd --infra aws --name service
```

```bash short options
ag ws patch -e prd -i aws -n service
```

</CodeGroup>
