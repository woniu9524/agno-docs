---
title: Python ライブラリの追加
---

Agno テンプレートは、[pyproject.toml](https://packaging.python.org/en/latest/specifications/declaring-project-metadata/#declaring-project-metadata) ファイルを使用して依存関係を管理するように設定されており、**このファイルは [pip-tools](https://pip-tools.readthedocs.io/en/latest/) を使用して `requirements.txt` ファイルを生成するために使用されます。**

Python ライブラリを追加または更新するプロセスは、次の 2 つのステップで行います。

1. `pyproject.toml` ファイルにライブラリを追加する
2. `requirements.txt` ファイルを自動生成する

<Warning>

このプロセスを使用して `requirements.txt` ファイルを自動生成することを強くお勧めします。

</Warning>

## pyproject.toml の更新

- `pyproject.toml` ファイルを開きます。
- `dependencies` セクションに新しいライブラリを追加します。

## requirements の生成

`pyproject.toml` ファイルの `dependencies` を更新した後、ヘルパースクリプトを実行するか、直接 `pip-compile` を実行して `requirements.txt` ファイルを自動生成します。

<CodeGroup>

```bash terminal
./scripts/upgrade.sh
```

```bash pip compile
pip-compile \
    --no-annotate \
    --pip-args "--no-cache-dir" \
    -o requirements.txt pyproject.toml
```

</CodeGroup>

すべての Python ライブラリを最新バージョンにアップグレードしたい場合は、次を実行します。

<CodeGroup>

```bash terminal
./scripts/upgrade.sh all
```

```bash pip compile
pip-compile \
    --upgrade \
    --no-annotate \
    --pip-args "--no-cache-dir" \
    -o requirements.txt pyproject.toml
```

</CodeGroup>

## イメージの再構築

`requirements.txt` ファイルを更新した後、イメージを再構築します。

### 開発イメージの再構築

<CodeGroup>

```bash terminal
ag ws up --env dev --infra docker --type image
```

```bash short options
ag ws up -e dev -i docker -t image
```

</CodeGroup>

### 本番イメージの再構築

<Note>

必要に応じて、[ECR との認証](/workspaces/workspace-management/production-app#ecr-images) を忘れないでください。

</Note>

<CodeGroup>

```bash terminal
ag ws up --env prd --infra docker --type image
```

```bash short options
ag ws up -e prd -i docker -t image
```

</CodeGroup>

## リソースの再作成

イメージの再構築後、リソースを再作成します。

### 開発コンテナの再作成

<CodeGroup>

```bash terminal
ag ws restart --env dev --infra docker --type container
```

```bash short options
ag ws restart -e dev -c docker -t container
```

</CodeGroup>

### ECS サービスの再起動

<CodeGroup>

```bash terminal
ag ws patch --env prd --infra aws --name service
```

```bash short options
ag ws patch -e prd -i aws -n service
```

</CodeGroup>
