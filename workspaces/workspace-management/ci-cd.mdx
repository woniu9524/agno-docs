## 在每次 PR 时进行测试和验证

每当针对 `main` 分支打开 PR 时，都会运行一个验证脚本，该脚本可确保

1. 更改已使用 ruff 进行格式化
2. 所有单元测试均通过
3. 更改没有任何类型或 linting 错误。

请查看 `.github/workflows/validate.yml` 文件了解更多信息。

## 通过 Github Releases 构建 Docker 镜像

如果您使用 [Dockerhub](https://hub.docker.com/) 来存储镜像，则可以通过 Github Release 构建和推送镜像。此操作在 `.github/workflows/docker-images.yml` 文件中定义。

1. 为 Github Actions 创建一个 [Docker 访问令牌](https://hub.docker.com/settings/security)

<img src="/images/docker-access-token.png" alt="docker-access-token" />

2. 在您的 github 仓库中创建秘密变量 `DOCKERHUB_REPO`、`DOCKERHUB_TOKEN` 和 `DOCKERHUB_USERNAME`。这些变量由 `.github/workflows/docker-images.yml` 中的操作使用

<img
  src="/images/github-actions-docker-secrets.png"
  alt="github-actions-docker-secrets"
/>

3. 使用 Github Release 运行工作流

此工作流配置为在创建 Release 时运行。使用以下命令创建新 Release：

<Note>

在运行之前，请确认 `.github/workflows/docker-images.yml` 文件中的镜像名称

</Note>

<CodeGroup>

```bash Mac
gh release create v0.1.0 --title "v0.1.0" -n ""
```

```bash Windows
gh release create v0.1.0 --title "v0.1.0" -n ""
```

</CodeGroup>

<img
  src="/images/github-actions-build-docker.png"
  alt="github-actions-build-docker"
/>

<Note>

您也可以使用 `gh workflow run` 来运行工作流

</Note>

## 通过 Github Releases 构建 ECR 镜像

如果您使用 ECR 来存储镜像，则可以通过 Github Release 构建和推送镜像。此操作在 `.github/workflows/ecr-images.yml` 文件中定义，并使用新的 OpenID Connect (OIDC) 方法来请求访问令牌，而无需使用 IAM 访问密钥。

我们将遵循此 [指南](https://aws.amazon.com/blogs/security/use-iam-roles-to-connect-github-actions-to-actions-in-aws/) 来创建将由 github 操作使用的 IAM 角色。

1. 打开 IAM 控制台。
2. 在左侧导航菜单中，选择“身份提供程序”。
3. 在“身份提供程序”窗格中，选择“添加提供程序”。
4. 对于“提供程序类型”，选择“OpenID Connect”。
5. 对于“提供程序 URL”，请输入 GitHub OIDC IdP 的 URL：https://token.actions.githubusercontent.com
6. 获取指纹以验证服务器证书
7. 对于“受众”，请输入 sts.amazonaws.com。

验证信息与下图匹配，然后添加提供程序

<img src="/images/github-oidc-provider.png" alt="github-oidc-provider" />

8. 将角色分配给提供程序。

<img
  src="/images/github-oidc-provider-assign-role.png"
  alt="github-oidc-provider-assign-role"
/>

9. 创建新角色。

<img
  src="/images/github-oidc-provider-create-new-role.png"
  alt="github-oidc-provider-create-new-role"
/>

10. 确认“Web 身份”已选为受信任实体，并且“身份提供程序”字段已填充了 IdP。在“受众”列表中，选择 `sts.amazonaws.com`，然后选择“下一步”。

<img
  src="/images/github-oidc-provider-trusted-entity.png"
  alt="github-oidc-provider-trusted-entity"
/>

11. 将 `AmazonEC2ContainerRegistryPowerUser` 权限添加到此角色。

12. 以 `GithubActionsRole` 的名称创建角色。

13. 找到 `GithubActionsRole` 角色并复制其 ARN。

<img src="/images/github-oidc-role.png" alt="github-oidc-role" />

14. 创建 ECR 存储库：`llm` 和 `jupyter-llm`，它们将由工作流构建。

<img src="/images/create-ecr-image.png" alt="create-ecr-image" />

15. 使用 `GithubActionsRole` ARN 和 ECR 存储库更新工作流。

```yaml .github/workflows/ecr-images.yml
name: Build ECR Images

on:
  release:
    types: [published]

permissions:
  # For AWS OIDC Token access as per https://docs.github.com/en/actions/deployment/security-hardening-your-deployments/configuring-openid-connect-in-amazon-web-services#updating-your-github-actions-workflow
  id-token: write # This is required for requesting the JWT
  contents: read # This is required for actions/checkout

env:
  ECR_REPO: [YOUR_ECR_REPO]
  # Create role using https://aws.amazon.com/blogs/security/use-iam-roles-to-connect-github-actions-to-actions-in-aws/
  AWS_ROLE: [GITHUB_ACTIONS_ROLE_ARN]
  AWS_REGION: us-east-1
```

16. 更新 `docker-images` 工作流以 **不** 在 Release 时运行

```yaml .github/workflows/docker-images.yml
name: Build Docker Images

on: workflow_dispatch
```

17. 使用 Github Release 运行工作流

<CodeGroup>

```bash Mac
gh release create v0.2.0 --title "v0.2.0" -n ""
```

```bash Windows
gh release create v0.2.0 --title "v0.2.0" -n ""
```

</CodeGroup>
