## PR ごとのテストと検証

`main` ブランチに対する PR が開かれるたびに、以下のことを保証する検証スクリプトが実行されます。

1. 変更は ruff を使用してフォーマットされていること
2. すべての単体テストがパスすること
3. 変更に型エラーや lint エラーがないこと

詳細については、`.github/workflows/validate.yml` ファイルを確認してください。

## Github Releases を使用した Docker イメージのビルド

イメージに [Dockerhub](https://hub.docker.com/) を使用している場合、Github Release を通じてイメージをビルドおよびプッシュできます。このアクションは `.github/workflows/docker-images.yml` ファイルで定義されています。

1. Github Actions 用の [Docker アクセストークン](https://hub.docker.com/settings/security)を作成します。

<img src="/images/docker-access-token.png" alt="docker-access-token" />

2. GitHub リポジトリに `DOCKERHUB_REPO`、`DOCKERHUB_TOKEN`、`DOCKERHUB_USERNAME` というシークレット変数を作成します。これらの変数は、`.github/workflows/docker-images.yml` のアクションによって使用されます。

<img
  src="/images/github-actions-docker-secrets.png"
  alt="github-actions-docker-secrets"
/>

3. Github Release を使用してワークフローを実行します。

このワークフローは、リリースが作成されたときに実行されるように構成されています。新しいリリースを作成するには、以下を使用します。

<Note>

実行前に `.github/workflows/docker-images.yml` ファイルでイメージ名を確認してください。

</Note>

<CodeGroup>

```bash Mac
gh release create v0.1.0 --title "v0.1.0" -n ""
```

```bash Windows
gh release create v0.1.0 --title "v0.1.0" -n ""
```

</CodeGroup>

<img
  src="/images/github-actions-build-docker.png"
  alt="github-actions-build-docker"
/>

<Note>

`gh workflow run` を使用してワークフローを実行することもできます。

</Note>

## Github Releases を使用した ECR イメージのビルド

イメージに ECR を使用している場合、Github Release を通じてイメージをビルドおよびプッシュできます。このアクションは `.github/workflows/ecr-images.yml` ファイルで定義されており、IAM アクセスキーを使用せずにアクセストークンを要求する新しい OpenID Connect (OIDC) アプローチを使用します。

この [ガイド](https://aws.amazon.com/blogs/security/use-iam-roles-to-connect-github-actions-to-actions-in-aws/) に従って、GitHub アクションが使用する IAM ロールを作成します。

1. IAM コンソールを開きます。
2. 左側のナビゲーションメニューで、[ID プロバイダー] を選択します。
3. [ID プロバイダー] ペインで、[プロバイダーの追加] を選択します。
4. [プロバイダータイプ] で、[OpenID Connect] を選択します。
5. [プロバイダー URL] に、GitHub OIDC IdP の URL を入力します: https://token.actions.githubusercontent.com
6. サムプリントを取得してサーバー証明書を検証します。
7. [オーディエンス] に、sts.amazonaws.com と入力します。

情報が以下のスクリーンショットと一致することを確認し、[プロバイダーの追加] を選択します。

<img src="/images/github-oidc-provider.png" alt="github-oidc-provider" />

8. プロバイダーにロールを割り当てます。

<img
  src="/images/github-oidc-provider-assign-role.png"
  alt="github-oidc-provider-assign-role"
/>

9. 新しいロールを作成します。

<img
  src="/images/github-oidc-provider-create-new-role.png"
  alt="github-oidc-provider-create-new-role"
/>

10. [信頼されたエンティティ] で Web ID が選択されており、[ID プロバイダー] フィールドに IdP が入力されていることを確認します。[オーディエンス] リストで sts.amazonaws.com を選択し、[次へ] を選択します。

<img
  src="/images/github-oidc-provider-trusted-entity.png"
  alt="github-oidc-provider-trusted-entity"
/>

11. このロールに `AmazonEC2ContainerRegistryPowerUser` 権限を追加します。

12. `GithubActionsRole` という名前でロールを作成します。

13. `GithubActionsRole` ロールを見つけて、ARN をコピーします。

<img src="/images/github-oidc-role.png" alt="github-oidc-role" />

14. ワークフローによってビルドされる ECR リポジトリ `llm` および `jupyter-llm` を作成します。

<img src="/images/create-ecr-image.png" alt="create-ecr-image" />

15. ワークフローを `GithubActionsRole` ARN および ECR リポジトリで更新します。

```yaml .github/workflows/ecr-images.yml
name: Build ECR Images

on:
  release:
    types: [published]

permissions:
  # For AWS OIDC Token access as per https://docs.github.com/en/actions/deployment/security-hardening-your-deployments/configuring-openid-connect-in-amazon-web-services#updating-your-github-actions-workflow
  id-token: write # This is required for requesting the JWT
  contents: read # This is required for actions/checkout

env:
  ECR_REPO: [YOUR_ECR_REPO]
  # Create role using https://aws.amazon.com/blogs/security/use-iam-roles-to-connect-github-actions-to-actions-in-aws/
  AWS_ROLE: [GITHUB_ACTIONS_ROLE_ARN]
  AWS_REGION: us-east-1
```

16. `docker-images` ワークフローを、リリース時に実行されないように更新します。

```yaml .github/workflows/docker-images.yml
name: Build Docker Images

on: workflow_dispatch
```

17. Github Release を使用してワークフローを実行します。

<CodeGroup>

```bash Mac
gh release create v0.2.0 --title "v0.2.0" -n ""
```

```bash Windows
gh release create v0.2.0 --title "v0.2.0" -n ""
```

</CodeGroup>

<img
  src="/images/github-actions-build-ecr.png"
  alt="github-actions-build-ecr"
/>

<Note>

`gh workflow run` を使用してワークフローを実行することもできます。

</Note>
