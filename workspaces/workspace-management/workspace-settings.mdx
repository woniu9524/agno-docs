---
title: Workspace Settings
---

`workspace/settings.py` ファイルの `WorkspaceSettings` オブジェクトは、アプリとリソースで共通して使用される設定を定義します。以下に更新をお勧めする設定を示します。

```python workspace/settings.py
ws_settings = WorkspaceSettings(
    # プロジェクト名に更新してください
    ws_name="ai",
    # AWSサブネットを追加してください
    subnet_ids=["subnet-xyz", "subnet-xyz"],
    # イメージリポジトリを追加してください
    image_repo="[ACCOUNT_ID].dkr.ecr.us-east-1.amazonaws.com",
    # ローカルでイメージをビルドするにはTrueに設定します
    build_images=True,
    # ビルド後にイメージをプッシュするにはTrueに設定します
    push_images=True,
)
```

<Note>

`WorkspaceSettings` は、環境変数または `.env` ファイルを使用して更新することもできます。

例については、`example.env` ファイルを確認してください。

</Note>

### Workspace Name

`ws_name` は、アプリとリソースの名前付けに使用されます。プロジェクト名またはチーム名に変更してください。例：

- `ws_name="booking-ai"`
- `ws_name="reddit-ai"`
- `ws_name="vantage-ai"`

`ws_name` は以下に使用されます：

- アプリケーションのイメージ
- db、streamlitアプリ、FastAPIサーバーなどのアプリ
- バケット、シークレット、ロードバランサーなどのリソース

どのように使用されているかについては、`workspace/dev_resources.py` および `workspace/prd_resources.py` ファイルを確認してください。

## Image Repository

`image_repo` は、イメージのリポジトリを定義します。

- dockerhubを使用する場合、`agno` のようになります。
- ECRを使用する場合、`[ACCOUNT_ID].dkr.ecr.us-east-1.amazonaws.com` のようになります。

どのように使用されているかについては、`workspace/dev_resources.py` の `dev_image` および `workspace/prd_resources.py` の `prd_image` を確認してください。

## Build Images

`build_images=True` を設定すると、`ag ws up dev:docker` または `ag ws up prd:docker` を実行したときにローカルでイメージがビルドされます。

どのように使用されているかについては、`workspace/dev_resources.py` の `dev_image` および `workspace/prd_resources.py` の `prd_image` を確認してください。

以下について続きを読む：

- [開発イメージのビルド](/workspaces/workspace-management/development-app#build-your-development-image)
- [本番イメージのビルド](/workspaces/workspace-management/production-app#build-your-production-image)

## Push Images

`push_images=True` を設定すると、`ag ws up dev:docker` または `ag ws up prd:docker` を実行したときにビルド後にイメージがプッシュされます。

どのように使用されているかについては、`workspace/dev_resources.py` の `dev_image` および `workspace/prd_resources.py` の `prd_image` を確認してください。

以下について続きを読む：

- [開発イメージのビルド](/workspaces/workspace-management/development-app#build-your-development-image)
- [本番イメージのビルド](/workspaces/workspace-management/production-app#build-your-production-image)

## AWS Settings

`aws_region` および `subnet_ids` は、本番リソースの作成に使用される値を提供します。どのように使用されているかについては、`workspace/prd_resources.py` ファイルを確認してください。
