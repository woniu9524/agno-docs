**Agno Workspaces** は、Docker を使用して Agentic Systems をローカルで実行し、AWS 上で本番稼働させるための標準化されたコードベースです。これにより、Agentic System をコードとして管理することができます。

![workspace](/images/workspace.png)

## 新しいワークスペースを作成する

`ag ws create` を実行して新しいワークスペースを作成します。コマンドは、スターターテンプレートとワークスペース名を尋ねます。

<CodeGroup>

```bash Create Workspace
ag ws create
```

```bash Create Agent App
ag ws create -t agent-app-aws -n agent-app
```

```bash Create Agent API
ag ws create -t agent-api-aws -n agent-api
```

</CodeGroup>

## ワークスペースリソースを開始する

`ag ws up` を実行してワークスペースリソースを開始（作成）します。

<CodeGroup>

```bash terminal
ag ws up
```

```bash shorthand
ag ws up dev:docker
```

```bash full options
ag ws up --env dev --infra docker
```

```bash short options
ag ws up -e dev -i docker
```

</CodeGroup>

## ワークスペースリソースを停止する

<CodeGroup>

```bash terminal

```

```bash shorthand

```

```bash full options
ag ws patch --env dev --infra docker
```


<Note>

</Note>

<CodeGroup>


```bash shorthand

```

```bash full options
ag ws restart --env dev --infra docker
```


## 既存のワークスペースをセットアップする

コードベースを直接クローンした場合（例：同僚が作成した場合）は、`ag ws setup` を実行してローカルでセットアップします。

<CodeGroup>

```bash terminal
ag ws setup
```

```bash with debug logs
ag ws setup -d
```

</CodeGroup>

## コマンドオプション

<Note>すべてのオプションを表示するには `ag ws up --help` を実行してください。</Note>

### 環境 (`--env`)

`--env` または `-e` フラグを使用して、環境（dev/prd）でフィルタリングします。

<CodeGroup>

```bash flag
ag ws up --env dev
```

```bash shorthand
ag ws up dev
```

```bash short options
ag ws up -e dev
```

</CodeGroup>

### インフラストラクチャ (`--infra`)

`--infra` または `-i` フラグを使用して、インフラストラクチャ（docker/aws/k8s）でフィルタリングします。

<CodeGroup>

```bash flag
ag ws up --infra docker
```

```bash shorthand
ag ws up :docker
```

```bash short options
ag ws up -i docker
```

</CodeGroup>

### グループ (`--group`)

`--group` または `-g` フラグを使用して、リソースグループでフィルタリングします。

<CodeGroup>

```bash flag
ag ws up --group app
```

```bash full options
ag ws up \
  --env dev \
  --infra docker \
  --group app
```

```bash shorthand
ag ws up dev:docker:app
```

```bash short options
ag ws up \
  -e dev \
  -i docker \
  -g app
```

</CodeGroup>

### 名前 (`--name`)

`--name` または `-n` フラグを使用して、リソース名でフィルタリングします。

<CodeGroup>

```bash flag
ag ws up --name app
```

```bash full options
ag ws up \
  --env dev \
  --infra docker \
  --name app
```

```bash shorthand
ag ws up dev:docker::app
```

```bash short options
ag ws up \
  -e dev \
  -i docker \
  -n app
```

</CodeGroup>

### タイプ (`--type`)

`--type` または `-t` フラグを使用して、リソースタイプでフィルタリングします。

<CodeGroup>

```bash flag
ag ws up --type container
```

```bash full options
ag ws up \
  --env dev \
  --infra docker \
  --type container
```

```bash shorthand
ag ws up dev:docker:app::container
```

```bash short options
ag ws up \
  -e dev \
  -i docker \
  -t container
```

</CodeGroup>

### ドライラン (`--dry-run`)

`--dry-run` または `-dr` フラグを使用して、コマンドを**ドライラン**できます。`ag ws up -dr` はリソースを作成せずに、リソースのみを表示します。

<CodeGroup>

```bash flag
ag ws up --dry-run
```

```bash full options
ag ws up \
  --env dev \
  --infra docker \
  --dry-run
```

```bash shorthand
ag ws up dev:docker -dr
```

```bash short options
ag ws up \
  -e dev \
  -i docker \
  -dr
```

</CodeGroup>

`--debug` または `-d` フラグを使用して、デバッグログを表示します。

<CodeGroup>

```bash flag

```

```bash full options
ag ws up \
```

`--force` または `-f` フラグを使用して、イメージとコンテナを強制的に再作成します。

<CodeGroup>

```bash flag

```

```bash full options

  --env dev \
  --infra docker \
  -f
```

```bash shorthand
ag ws up dev:docker -f
```


</CodeGroup>
