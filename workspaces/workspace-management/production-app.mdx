本番アプリケーションはAWS上で実行され、そのリソースは `workspace/prd_resources.py` ファイルで定義されています。このガイドでは、以下の方法を示します。

1. [本番イメージのビルド](#build-your-production-image)
2. [ECSタスク定義の更新](#ecs-task-definition)
3. [ECSサービスの更新](#ecs-service)

## ワークスペース設定

`workspace/settings.py` ファイルの `WorkspaceSettings` オブジェクトは、ワークスペースのアプリとリソースで共通して使用される設定を定義します。

## 本番イメージのビルド

デフォルトでは、アプリケーションは `agno` イメージを使用します。独自のイメージを使用するには：

- `ECR` でリポジトリを作成し、認証を行うか、`Dockerhub` を使用します。
- `workspace/settings.py` ファイルを開きます。
- `image_repo` をイメージリポジトリに更新します。
- `build_images=True` および `push_images=True` を設定します。
- オプション - リポジトリ内の既存のイメージを使用するには、`build_images=False` および `push_images=False` を設定します。

### ECRリポジトリの作成

ECRを使用するには、イメージをプッシュする前に**イメージリポジトリを作成し、ECRで認証を行ってください**。

**1. ECRでイメージリポジトリを作成する**

リポジトリ名は `ws_name` と一致する必要があります。つまり、デフォルトのワークスペース名を使用している場合、リポジトリ名は `ai` になります。

<img src="/images/create-ecr-image.png" alt="create-ecr-image" />

**2. ECRで認証する**

```bash ECRで認証する
aws ecr get-login-password --region [region] | docker login --username AWS --password-stdin [account].dkr.ecr.[region].amazonaws.com
```

フルコマンドを実行するのを避けるために、ヘルパースクリプトを使用することもできます。

<Note>

実行前にスクリプトをECRリポジトリで更新してください。

</Note>

<CodeGroup>

```bash Mac
./scripts/auth_ecr.sh
```

</CodeGroup>

### `WorkspaceSettings` の更新

```python workspace/settings.py
ws_settings = WorkspaceSettings(
    ...
    # aws_region内のサブネットID
    subnet_ids=["subnet-xyz", "subnet-xyz"],
    # -*- イメージ設定
    # イメージのリポジトリ
    image_repo="your-image-repo",
    # ローカルでイメージをビルドする
    build_images=True,
    # ビルド後にイメージをプッシュする
    push_images=True,
)
```

<Note>

`image_repo` はイメージのリポジトリを定義します。

- dockerhubを使用する場合、`agno` のようになります。
- ECRを使用する場合、`[ACCOUNT_ID].dkr.ecr.us-east-1.amazonaws.com` のようになります。

</Note>

### 新しいイメージのビルド

以下を使用して本番イメージをビルドします。

<CodeGroup>

```bash ターミナル
ag ws up --env prd --infra docker --type image
```

```bash ショートハンド
ag ws up -e prd -i docker -t image
```

</CodeGroup>

イメージを`強制`的に再ビルドするには、`--force` または `-f` フラグを使用します。

<CodeGroup>

```bash ターミナル
ag ws up --env prd --infra docker --type image --force
```

```bash ショートハンド
ag ws up -e prd -i docker -t image -f
```

</CodeGroup>

本番環境のDockerリソースはDockerイメージのみであるため、以下を使用することもできます。

<CodeGroup>

```bash イメージのビルド
ag ws up prd:docker
```

```bash イメージの強制ビルド
ag ws up prd:docker -f
```

</CodeGroup>

## ECSタスク定義

イメージ、CPU、メモリ、または環境変数を更新した場合は、以下を使用してタスク定義を更新します。

<CodeGroup>

```bash ターミナル
ag ws patch --env prd --infra aws --name td
```

```bash ショートハンド
ag ws patch -e prd -i aws -n td
```

</CodeGroup>

## ECSサービス

本番アプリケーションを再デプロイするには、以下を使用してECSサービスを更新します。

<CodeGroup>

```bash ターミナル
ag ws patch --env prd --infra aws --name service
```

```bash ショートハンド
ag ws patch -e prd -i aws -n service
```

</CodeGroup>

<br />

<Note>

イメージを**のみ**再ビルドした場合、タスク定義を更新する必要はなく、サービスをパッチして新しいイメージをピックアップするだけで済みます。

</Note>
