## Workspace 设置

`workspace/settings.py` 文件中的 `WorkspaceSettings` 对象定义了您的工作区应用程序和资源常用的设置。

## 构建生产镜像

默认情况下，您的应用程序使用 `agno` 镜像。要使用您自己的镜像：

- 在 `ECR` 中创建一个存储库并进行身份验证，或使用 `Dockerhub`。
- 打开 `workspace/settings.py` 文件
- 更新 `image_repo` 为您的镜像存储库
- 设置 `build_images=True` 和 `push_images=True`
- 可选 - 设置 `build_images=False` 和 `push_images=False` 以使用存储库中现有的镜像

### 创建 ECR 存储库

要使用 ECR，请在推送镜像之前**创建镜像存储库并进行 ECR 身份验证**。

**1. 在 ECR 中创建镜像存储库**

存储库名称应与 `ws_name` 匹配。如果使用默认工作区名称，存储库名称将是 `ai`。

<img src="/images/create-ecr-image.png" alt="create-ecr-image" />

**2. 进行 ECR 身份验证**

```bash Authenticate with ECR
aws ecr get-login-password --region [region] | docker login --username AWS --password-stdin [account].dkr.ecr.[region].amazonaws.com
```

您也可以使用一个辅助脚本来避免运行完整的命令

<Note>

在运行脚本之前，请更新脚本以包含您的 ECR 存储库。

</Note>

<CodeGroup>

```bash Mac
./scripts/auth_ecr.sh
```

</CodeGroup>

### 更新 `WorkspaceSettings`

```python workspace/settings.py
ws_settings = WorkspaceSettings(
    ...
    # 在 aws_region 中的子网 ID
    subnet_ids=["subnet-xyz", "subnet-xyz"],
    # -*- Image Settings
    # 镜像存储库
    image_repo="your-image-repo",
    # 本地构建镜像
    build_images=True,
    # 构建后推送镜像
    push_images=True,
)
```

<Note>

`image_repo` 定义了您的镜像的存储库。

- 如果使用 dockerhub，它将是类似 `agno` 的内容。
- 如果使用 ECR，它将是类似 `[ACCOUNT_ID].dkr.ecr.us-east-1.amazonaws.com` 的内容

</Note>

### 构建新镜像

使用以下命令构建生产镜像：

<CodeGroup>

```bash terminal
ag ws up --env prd --infra docker --type image
```

```bash shorthand
ag ws up -e prd -i docker -t image
```

</CodeGroup>

要`强制`重新构建镜像，请使用 `--force` 或 `-f` 标志

<CodeGroup>

```bash terminal
ag ws up --env prd --infra docker --type image --force
```

```bash shorthand
ag ws up -e prd -i docker -t image -f
```

</CodeGroup>

由于生产环境中唯一的 docker 资源是 docker 镜像，您也可以使用：

<CodeGroup>

```bash Build Images
ag ws up prd:docker
```

```bash Force Build Images
ag ws up prd:docker -f
```

</CodeGroup>

## ECS 任务定义

如果您更新了镜像、CPU、内存或环境变量，请使用以下命令更新任务定义：

<CodeGroup>

```bash terminal
ag ws patch --env prd --infra aws --name td
```

```bash shorthand
ag ws patch -e prd -i aws -n td
```

</CodeGroup>

## ECS 服务

要重新部署生产应用程序，请使用以下命令更新 ECS 服务：

<CodeGroup>

```bash terminal
ag ws patch --env prd --infra aws --name service
```

```bash shorthand
ag ws patch -e prd -i aws -n service
```

</CodeGroup>

<br />

<Note>

如果您**仅**重新构建了镜像，则无需更新任务定义，只需更新服务即可拾取新镜像。

</Note>
