## ワークスペースの設定

`workspace/settings.py` ファイル内の `WorkspaceSettings` オブジェクトは、ワークスペースのアプリとリソースで共通して使用される設定を定義します。

## 開発イメージのビルド

デフォルトでは、アプリケーションは `agno` イメージを使用します。独自のイメージを使用するには、以下を行います。

- `workspace/settings.py` ファイルを開きます。
- `image_repo` を自分のイメージリポジトリに更新します。
- `build_images=True` を設定します。

```python workspace/settings.py
ws_settings = WorkspaceSettings(
    ...
    # -*- Image Settings
    # Repository for images
    image_repo="local",
    # Build images locally
    build_images=True,
)
```

### 新しいイメージのビルド

開発イメージをビルドするには、以下を使用します。

<CodeGroup>

```bash terminal
ag ws up --env dev --infra docker --type image
```

```bash short options
ag ws up -e dev -i docker -t image
```

</CodeGroup>

イメージの再ビルドを `force` するには、`--force` または `-f` フラグを使用します。

<CodeGroup>

```bash terminal
ag ws up --env dev --infra docker --type image --force
```

```bash short options
ag ws up -e dev -i docker -t image -f
```

</CodeGroup>

## すべてのコンテナの再起動

すべての Docker コンテナを再起動するには、以下を使用します。

<CodeGroup>

```bash terminal
ag ws restart --env dev --infra docker --type container
```

```bash short options
ag ws restart -e dev -c docker -t container
```

</CodeGroup>

---

## 開発リソースの再作成

すべての開発リソースを再作成するには、`--force` フラグを使用します。

<CodeGroup>

```bash terminal
ag ws up -f
```

```bash full options
ag ws up --env dev --infra docker --force
```

```bash shorthand
ag ws up dev:docker -f
```

```bash short options
ag ws up -e dev -i docker -f
```

</CodeGroup>
