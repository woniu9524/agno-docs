`WorkspaceSettings` 对象通常在 `workspace/settings.py` 文件中定义，用于定义您的工作空间应用程序和资源使用的通用设置。

它不是必需的，除了保存工作空间应用程序和资源使用的配置外，没有其他用途。`WorkspaceSettings` 对象中的值也可以通过环境变量或 `.env` 文件设置。

## 示例

一个由 `llm-app` 模板使用的示例 `WorkspaceSettings`。在 [github](https://github.com/agno-agi/llm-app/blob/main/workspace/settings.py) 上查看此文件

```python workspace/settings.py
from pathlib import Path

from agno.workspace.settings import WorkspaceSettings

#
# -*- 使用 WorkspaceSettings 对象定义工作空间设置
# 这些值也可以通过环境变量或 .env 文件设置
#
ws_settings = WorkspaceSettings(
    # 工作空间名称：用于命名资源
    ws_name="ai",
    # 工作空间根目录路径
    ws_root=Path(__file__).parent.parent.resolve(),
    # -*- 开发设置
    dev_env="dev",
    # -*- 开发应用
    dev_app_enabled=True,
    dev_api_enabled=True,
    dev_db_enabled=True,
    # dev_jupyter_enabled=True,
    # -*- 生产设置
    prd_env="prd",
    # -*- 生产应用
    prd_app_enabled=True,
    prd_api_enabled=True,
    prd_db_enabled=True,
    # -*- AWS 设置
    # AWS 资源的区域
    aws_region="us-east-1",
    # AWS 资源的可用区
    aws_az1="us-east-1a",
    aws_az2="us-east-1b",
    # 在 aws_region 中的子网 ID
    # subnet_ids=["subnet-xyz", "subnet-xyz"],
    #
    # -*- 镜像设置
    #
    # 镜像的默认仓库
    image_repo="agno"
    # 本地构建镜像
    build_images=False
    # 构建后推送镜像
    push_images=False
    # 构建镜像时跳过缓存
    skip_image_cache=False
    # 在 FROM 中强制拉取镜像
    force_pull_images=False
)
```

## 用法

使用工作空间设置来

- 命名资源
- 使用 `ws_settings.ws_root` 获取工作空间根目录路径

```python dev_resources.py
...
# -*- Streamlit 运行在端口 8501:8501
dev_streamlit = Streamlit(
    name=f"{ws_settings.dev_key}-app",
    enabled=ws_settings.dev_app_enabled,
    ...
    # 从 secrets/dev_app_secrets.yml 读取 secrets
    secrets_file=ws_settings.ws_root.joinpath("workspace/secrets/dev_app_secrets.yml")
)
```

- 保存 AWS 常量，如 `availability zone` 和 `subnets`

```python prd_resources.py
# -*- FastAPI 运行在 ECS 上
prd_fastapi = FastApi(
    name=f"{ws_settings.prd_key}-api",
    enabled=ws_settings.prd_api_enabled,
    ...
    subnets=ws_settings.subnet_ids,
    ...
)

# -*- RDS 数据库实例
prd_db = DbInstance(
    name=f"{ws_settings.prd_key}-db",
    enabled=ws_settings.prd_db_enabled,
    ...
    availability_zone=ws_settings.aws_az1,
    ...
)
