[Docker](https://docs.docker.com/get-started/overview/) は、アプリケーションをローカルで実行できるようにする画期的なテクノロジーです。アプリケーションを実行するために必要なすべてを含む**コンテナ**に[アプリ](/workspaces/apps/introduction)をパッケージ化します。

## Docker リソース

Agno を使用すると、Docker リソースを Pydantic オブジェクトとして定義できるため、Python のみでアプリケーションレイヤーを構築できます。ほとんどの場合、**Docker リソース**を直接作成することはありません。代わりに、[アプリ](/workspaces/apps/introduction)を使用してリソースを作成します。

<Accordion title="docker-compose との違い">

Docker Compose はより成熟しており、リソースを「YAML 設定」として定義しますが、Agno ではリソースを「Python オブジェクト」として定義できます。

私たちは Docker Compose を引き続き愛していますが、エクスペリエンスを向上させたいと考えています。Agno では、リソースを直接定義することはほとんどありません。代わりに、アプリを使用してリソースを作成します。

</Accordion>

### 利点

- 入力および型検証を備えた Pydantic オブジェクトとしてコンテナとイメージを定義します。
- リソースの再利用とテストを可能にします。
- 通常の Python オブジェクトのようにソフトウェアレイヤーにインポートします。
- 複数のリソースを[アプリ](/workspaces/apps/introduction)にパッケージ化して、**「コードとしてのアプリケーション」**を定義できるようにします。
- Python コードからリソースと対話する AI 機能を有効にします。

## コンテナ

`DockerContainer` クラスはコンテナを定義します。たとえば、次のコードを使用して [whoami](https://github.com/traefik/whoami) イメージを実行するコンテナを定義します。`ag start resources.py` を使用して起動します。

```python resources.py
from agno.docker.resource.container import DockerContainer

whoami = DockerContainer(
    name='whoami',
    image='traefik/whoami',
    ports={'80': 80},
)
```

[http://localhost:80](http://localhost:80) を開くか、次を使用してテストします。

```bash
curl -X POST http://localhost:80
```

同様のものを `App` として定義することもできます。

```python resources.py
from agno.docker.app.whoami import Whoami

whoami = Whoami()
```

`ag stop resources.py` を使用してリソースを停止します。

## イメージ

`DockerImage` クラスはイメージを定義します。たとえば、次のコードを使用して独自の Python イメージを作成し、コンテナで実行します。`ag start resources.py` を使用してビルドします。

<CodeGroup>

```python resources.py
from agno.docker.resource.container import DockerContainer
from agno.docker.resource.image import DockerImage

python_image = DockerImage(
    name="my/python",
    tag="3.12",
    path=".",
    # push_image=True,
)

python_container = DockerContainer(
    name='python',
    image=python_image.get_image_str(),
)
```

```docker Dockerfile
FROM agnohq/python:3.12

CMD ["chill"]
```

</CodeGroup>
