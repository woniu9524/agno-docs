リソースは、アプリケーションのインフラストラクチャコンポーネントです。 [Apps](/workspaces/apps/introduction) と同様に、Pythonクラスとして定義し、`ag start` または `ag ws up` を使用して作成します。

## 例

- **ローカルリソース**: Dockerコンテナとイメージ
- **クラウドリソース**: RDSデータベース、S3バケット、ECSサービス、タスク定義、セキュリティグループ
- **Kubernetesリソース**: サービス、デプロイメント

<CodeGroup>

```python Docker Container
from agno.docker.resource.container import DockerContainer

whoami = DockerContainer(
    name='whoami',
    image='traefik/whoami',
    ports={'80': 8080},
)
```

```python Docker Image
from agno.docker.resource.image import DockerImage

dev_image = DockerImage(
    name="repo/image",
    tag="latest",
    push_image=True,,
)
```

```python S3 Bucket
from agno.aws.resource.s3 import S3Bucket

# -*- S3バケット my-bucket
prd_bucket = S3Bucket(name="my-bucket")
```

```python Secret
from pathlib import Path
from agno.aws.resource.secret import SecretsManager

# -*- シークレット my-secret
my_secret = SecretsManager(
    name="my-secret",
    # シークレット変数を my_secrets.yml から読み込む
    secret_files=[Path('my_secrets.yml')],
)
```

```python RDS Database
from pathlib import Path
from agno.aws.resource.secret import SecretsManager

# -*- データベースシークレット
db_secret = SecretsManager(
    name="my-db-secret",
    # シークレット変数を db_secrets.yml から読み込む
    secret_files=[Path('db_secrets.yml')],
)

# -*- データベースサブネットグループ
db_subnet_group = DbSubnetGroup(name="my-db-sg")

# -*- データベースインスタンス
db = DbInstance(
    name="my-db",
    db_name="llm",
    port=5423,
    engine="postgres",
    engine_version="16.1",
    allocated_storage=64,
    db_instance_class="db.t4g.medium",
    db_subnet_group=db_subnet_group,
    availability_zone="us-east-1a",
    publicly_accessible=True,
    aws_secret=db_secret,
)
```

## モチベーション

リソースは、AI製品の**「インフラストラクチャレイヤー」**を提供します。私たちが書くソフトウェアは、アプリケーションによって提供される必要があり、そのアプリケーションはインフラストラクチャリソース上で実行される必要があります。

**アプリケーションをコードとして**、**インフラストラクチャをコードとして**定義することで、Pythonコードとしてアプリケーション全体を記述することが可能になり、再利用性、バージョン管理、単体テスト、フォーマットなどの多くのメリットが得られます。

Agnoは現在以下を提供しています：

- [Dockerリソース](/workspaces/resources/docker/introduction)
- [AWSリソース](/workspaces/resources/aws/introduction)
