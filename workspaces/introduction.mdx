在构建 Agentic 系统时，您需要一个 API 来服务您的 Agent，一个数据库来存储会话和向量数据，以及一个用于测试和评估的管理界面。您还需要 cron 作业、警报以及用于摄取和清理数据的数据管道。通常需要几个月的时间来构建这样的系统，我们将其开源，免费提供给社区。

# 什么是 Workspaces？

**Workspaces 是用于生产 Agentic 系统的标准化代码库。**它们包含：

- 一个用于服务 Agent、Team 和 Workflow 的 RestAPI (FastAPI)。
- 一个用于测试的 streamlit 应用程序——可以将其视为管理界面。
- 一个用于会话和向量存储的 postgres 数据库。

Workspaces 设置为使用 docker 本地运行，并易于部署到 AWS。它们是一个绝佳的起点，也是我们为客户提供的服务。您肯定需要根据您的具体需求进行定制，但它们将大大加快您的启动速度。

它们包含了多年的经验积累，免费提供给开源社区。

# 它们是如何工作的

- 使用以下命令创建您的代码库：`ag ws create`
- 使用 docker 本地运行：`ag ws up`
- 在 AWS 上运行：`ag ws up prd:aws`

我们建议从 `agent-app` 模板开始，然后在此基础上进行扩展。

# 我们如何构建 Agentic 系统

在构建 Agent 时，我们会在本地进行实验，直到达到 6/10 的质量。
这有助于我们快速看到结果，并大致了解我们的解决方案在生产环境中应该是什么样子。

然后，我们开始迁移到生产环境并从中进行迭代。以下是 __*我们*__ 构建生产系统的方式：
- 通过 REST API (FastAPI) 服务 Agent、Team 和 Workflow。
- 使用 streamlit 应用程序进行调试和测试。这个 streamlit 应用程序通常用作 Agentic 系统的管理界面，并显示各种数据。
- 监控、评估和改进实现，直到达到 9/10 的质量。
- 同时，我们开始将前端与上述 REST API 集成。

在构建了数百个此类系统后，我们有一套标准的、我们称之为 **Workspaces** 的代码库。它们帮助我们以代码形式管理我们的 Agentic 系统。
