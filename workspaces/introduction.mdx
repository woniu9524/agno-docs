在构建 Agentic System 时，您需要一个 API 来服务您的 Agent、一个数据库来存储会话和向量数据，以及一个用于测试和评估的管理界面。您还需要 cron 作业、警报和用于摄取和清理的数据管道。这个系统通常需要几个月才能构建完成，我们将其开源，免费提供给社区。

# 什么是 Workspaces？

**Workspaces 是生产 Agentic Systems 的标准化代码库。** 它们包含：

- 一个 RestAPI (FastAPI)，用于服务 Agents、Teams 和 Workflows。
- 一个 streamlit 应用程序用于测试——可以将其视为管理界面。
- 一个 postgres 数据库用于会话和向量存储。

Workspaces 设置为使用 docker 本地元运行，并易于部署到 AWS。它们是一个出色的起点，也是我们为客户使用的确切方案。您肯定需要根据您的具体需求进行定制，但它们将帮助您更快地启动。

它们包含了多年的经验总结，免费提供给开源社区。

# 它们是如何工作的

- 使用以下命令创建您的代码库：`ag ws create`
- 使用 docker 本地元运行：`ag ws up`
- 在 AWS 上运行：`ag ws up prd:aws`

我们建议从 `agent-app` 模板开始，并在此基础上进行扩展。

# 我们如何构建 Agentic Systems

在构建 Agent 时，我们会在本地进行实验，直到达到 6/10 的质量。
这有助于我们快速看到结果，并对我们的解决方案在生产环境中应有的样子有一个大致的了解。

然后，我们开始迁移到生产环境并从中进行迭代。以下是 __我们__ 构建生产系统的方式：
- 通过 REST API (FastAPI) 服务 Agents、Teams 和 Workflows。
- 使用 streamlit 应用程序进行调试和测试。这个 streamlit 应用程序通常用作 Agentic System 的管理界面，并显示各种数据。
- 监控、评估和改进实现，直到我们达到 9/10 的质量。
- 同时，我们开始将我们的前端与上述 REST API 集成。

在构建了数百个此类系统之后，我们拥有了一套标准的、可用的代码库，我们称之为 **Workspaces**。它们帮助我们以代码的形式管理我们的 Agentic System。
