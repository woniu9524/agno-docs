エージェントシステムを構築する際には、エージェントを提供するAPI、セッションおよびベクトルデータを格納するデータベース、テストおよび評価のための管理インターフェイスが必要になります。また、取り込みとクリーニングのためのcronジョブ、アラート、データパイプラインも必要になります。このシステムは通常、構築に数ヶ月かかりますが、コミュニティのために無料でオープンソース化しています。

# ワークスペースとは？

**ワークスペースは、本番環境のエージェントシステムのための標準化されたコードベースです。** これらには以下が含まれます：

- エージェント、チーム、ワークフローを提供するREST API (FastAPI)。
- テスト用のStreamlitアプリケーション -- これは管理インターフェイスと考えてください。
- セッションおよびベクトルストレージ用のPostgresデータベース。

ワークスペースはDockerを使用してローカルで実行できるようにセットアップされており、AWSに簡単にデプロイできます。これらは素晴らしい出発点であり、私たちが顧客のために使用しているものと全く同じです。特定のニーズに合わせてカスタマイズする必要があることは間違いありませんが、これにより開始がはるかに速くなります。

これらには長年の学習が含まれており、オープンソースコミュニティのために無料で利用できます。

# 仕組み

- コードベースを作成するには：`ag ws create`
- Dockerを使用してローカルで実行するには：`ag ws up`
- AWSで実行するには：`ag ws up prd:aws`

`agent-app`テンプレートから始めて、そこから進めることをお勧めします。

# エージェントシステムを構築する方法

エージェントを構築する際、ローカルで実験を行い、6/10の品質を達成します。
これにより、迅速な結果を確認し、ソリューションが本番環境でどのように見えるかの大まかなアイデアを得ることができます。

その後、本番環境に移行し、そこから反復処理を開始します。本番システムを構築する__*私たちの方法*__は以下の通りです：
- エージェント、チーム、ワークフローをREST API (FastAPI) を介して提供します。
- デバッグおよびテスト用にStreamlitアプリケーションを使用します。このStreamlitアプリは通常、エージェントシステムの管理インターフェイスとして使用され、あらゆる種類のデータが表示されます。
- 実装を監視、評価、改善し、9/10の品質に到達するまで続けます。
- 並行して、フロントエンドと上記のREST APIの統合を開始します。

このようなシステムを数百構築してきた経験から、私たちは標準的なコードベースセットを使用しており、それらを**ワークスペース**と呼んでいます。これらは、エージェントシステムをコードとして管理するのに役立ちます。
