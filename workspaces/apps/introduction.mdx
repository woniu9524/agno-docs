App គឺជាเครื่องมือต่างๆ เช่น `FastAPI`, `PgVector`, `Streamlit`, `Jupyter`, `Django` ที่เรากำหนดเป็นคลาส python และรันโดยใช้ `ag start` หรือ `ag ws up`

เมื่อรัน App โดยใช้ agno ให้คิดว่ามันเป็น infrastructure as code แต่มีระดับ abstraction ที่สูงกว่า แทนที่จะกำหนด containers, volumes และอื่นๆ เรามากำหนดแอปพลิเคชันที่เราต้องการรัน เรากำลังรัน **Applications as Code** แทน Infrastructure as Code

App เดียวกันสามารถรันบน docker, AWS (ECS) หรือ Kubernetes (EKS) App จะสร้างทรัพยากรพื้นฐาน เช่น LoadBalancers, Services, Deployments เมื่อทรัพยากรพื้นฐานมีความซับซ้อนมากขึ้น แนวคิดของ App จะน่าสนใจยิ่งขึ้น

## ตัวอย่าง

มาลองรัน Jupyter notebook และ PgVector บน docker กัน

คัดลอกเนื้อหาต่อไปนี้ไปยังไฟล์ `resources.py` และรัน `ag start resources.py`

```python resources.py
from agno.docker.app.jupyter import Jupyter
from agno.docker.app.postgres import PgVectorDb
from agno.docker.resources import DockerResources

# PgVector ทำงานบนพอร์ต 5432:5432
vector_db = PgVectorDb(pg_user="ai", pg_password="ai", pg_database="ai")

# Jupyter ทำงานบนพอร์ต 8888:8888
jupyter = Jupyter(mount_workspace=True)

# DockerResources
dev_docker_resources = DockerResources(apps=[vector_db, jupyter])
```

- แต่ละ App เป็น pydantic object ที่มีการตรวจสอบอินพุตและไทป์
- สังเกตว่า `mount_workspace` ทำการ mount directory โดยอัตโนมัติ
- สังเกตว่า `PgVectorDb` ตั้งค่าที่จำเป็นและสร้าง volume

แม้ว่านี่จะเป็นตัวอย่างที่เรียบง่าย แต่แนวคิดเหล่านี้จะมีประสิทธิภาพอย่างมากสำหรับแอปพลิเคชันที่ซับซ้อน

## แรงจูงใจ

App จะเป็น **"Application Layer"** สำหรับผลิตภัณฑ์ AI ของเรา

ซอฟต์แวร์ที่เราเขียนจำเป็นต้องถูกให้บริการโดย Application และ Application นี้จำเป็นต้อง **ทำงานเหมือนกัน** ในเครื่องสำหรับการพัฒนาและบนคลาวด์สำหรับการผลิต โดยการกำหนด **Applications as Code** เรานำประโยชน์ของ **Infrastructure as Code** มาสู่ชั้นซอฟต์แวร์

การกำหนด **Applications as Code** ยังช่วยให้เราสามารถแพ็ก "ระบบซอฟต์แวร์" เป็นเทมเพลตได้ ซึ่งหมายความว่าเทมเพลต agno ทุกตัวสามารถรันในเครื่องโดยใช้ docker หรือบน AWS ได้ด้วยคำสั่งเดียว

สุดท้าย การกำหนด **Applications เป็น python objects** หมายความว่าเราสามารถ import มันในโค้ดของเราเหมือนกับอ็อบเจกต์ทั่วไป ทำให้โค้ดต่อไปนี้เป็นไปได้:

```python
from resources import vector_db

db_url=vector_db.get_db_connection_local()
```
