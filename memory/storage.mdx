# メモリストレージ

セッションや実行サイクルをまたいでメモリを永続化するには、データベースのような永続ストレージにメモリを保存します。

本番環境でメモリを使用する場合、アプリケーションの再起動後もユーザーのメモリを保持したいでしょうから、永続ストレージは非常に重要です。

Agnoのメモリシステムは、複数の永続ストレージオプションをサポートしています。

## ストレージオプション

Memoryクラスは、プラグ可能なデータベースインターフェースを通じて、さまざまなバックエンドストレージオプションをサポートしています。現在、Agnoは以下を提供しています。

1. [SQLiteストレージ](/reference/memory/storage/sqlite)
2. [PostgreSQLストレージ](/reference/memory/storage/postgres)
3. [MongoDBストレージ](/reference/memory/storage/mongo)
4. [Redisストレージ](/reference/memory/storage/redis)

## ストレージの設定

メモリストレージを設定するには、データベースインスタンスを作成し、それをMemoryコンストラクタに渡す必要があります。

```python
from agno.memory.v2.memory import Memory
from agno.memory.v2.db.sqlite import SqliteMemoryDb

# メモリ用のSQLiteデータベースを作成
memory_db = SqliteMemoryDb(
    table_name="memories",  # 使用するテーブル名
    db_file="path/to/memory.db"  # SQLiteデータベースファイル
)

# ストレージバックエンドでMemoryを初期化
memory = Memory(db=memory_db)
```

## データモデル

永続ストレージを使用する場合、Memoryシステムは以下を保存します。

- **ユーザーメモリ** - ユーザーに関する事実と洞察
- **最終更新タイムスタンプ** - メモリが最後に変更された日時を追跡するため
- **メモリID** - 各メモリの一意の識別子

## ストレージの例

```python sqlite_memory.py
from agno.memory.v2.memory import Memory
from agno.memory.v2.db.sqlite import SqliteMemoryDb
from agno.memory.v2.schema import UserMemory

# SQLiteメモリデータベースを作成
memory_db = SqliteMemoryDb(
    table_name="user_memories",
    db_file="tmp/memory.db"
)

# ストレージバックエンドでMemoryを初期化
memory = Memory(db=memory_db)

# 再起動後も永続化されるユーザーメモリを追加
user_id = "user@example.com"
memory.add_user_memory(
    memory=UserMemory(
        memory="ユーザーはアプリケーションでダークモードを好みます",
        topics=["preferences", "ui"]
    ),
    user_id=user_id
)

# メモリを取得（これらはデータベースからロードされます）
user_memories = memory.get_user_memories(user_id=user_id)
for m in user_memories:
    print(f"Memory: {m.memory}")
    print(f"Topics: {m.topics}")
    print(f"Last Updated: {m.last_updated}")
```
```python postgres_memory.py
from agno.memory.v2.memory import Memory
from agno.memory.v2.db.postgres import PostgresMemoryDb
from agno.memory.v2.schema import UserMemory

# PostgreSQLメモリデータベースを作成
memory_db = PostgresMemoryDb(
    table_name="user_memories",
    connection_string="postgresql://user:password@localhost:5432/mydb"
)

# ストレージバックエンドでMemoryを初期化
memory = Memory(db=memory_db)

# ユーザーメモリを追加
user_id = "user@example.com"
memory.add_user_memory(
    memory=UserMemory(
        memory="ユーザーはプレミアムサブスクリプションを持っています",
        topics=["subscription", "account"]
    ),
    user_id=user_id
)

# メモリ操作はバックエンドに関係なく同じように機能します
print(f"User has {len(memory.get_user_memories(user_id=user_id))} memories stored")
```

## エージェントストレージとの統合

メモリを持つエージェントを構築する際には、エージェントセッションとメモリの両方を保存したい場合が多くあります。Agnoは、両方のストレージシステムを設定できるようにすることで、これを容易にします。

```python
from agno.agent import Agent
from agno.memory.v2.memory import Memory
from agno.memory.v2.db.sqlite import SqliteMemoryDb
from agno.models.openai import OpenAIChat
from agno.storage.sqlite import SqliteStorage

# メモリストレージを作成
memory_db = SqliteMemoryDb(
    table_name="memories",
    db_file="tmp/memory.db"
)
memory = Memory(db=memory_db)

# エージェントストレージを作成
agent_storage = SqliteStorage(
    table_name="agent_sessions",
    db_file="tmp/agent_storage.db"
)

# メモリとストレージの両方でエージェントを作成
agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    memory=memory,
    storage=agent_storage,
    enable_user_memories=True,
)
```

## メモリ管理

永続ストレージを使用する場合、Memoryシステムは保存されたメモリを管理するためのいくつかの機能を提供します。

```python
# 特定のメモリを削除
memory.delete_user_memory(user_id="user@example.com", memory_id="memory_123")

# メモリを置き換え/更新
memory.replace_user_memory(
    memory_id="memory_123",
    memory=UserMemory(memory="ユーザーに関する更新された情報"),
    user_id="user@example.com"
)

# すべてのメモリをクリア
memory.clear()
```
