**Session Storage** を使用して、エージェントのセッションと状態をデータベースまたはファイルに永続化します。

<Tip>
**セッションストレージが必要な理由**

エージェントは一時的なものであり、組み込みのメモリは現在の実行サイクルでのみ持続します。

本番環境では、API経由でエージェントを提供（またはトリガー）し、複数のリクエストにわたって同じセッションを継続する必要があります。ストレージはセッション履歴と状態をデータベースに永続化し、中断したところから再開できるようにします。

ストレージを使用すると、エージェントのセッションを検査および評価し、few-shotの例を抽出し、内部監視ツールを構築することもできます。これにより、**データを参照する**ことができ、より良いエージェントを構築するのに役立ちます。
</Tip>

エージェント、チーム、またはワークフローにストレージを追加することは、`Storage`ドライバーを提供するだけで、あとはAgnoが処理します。Sqlite、Postgres、Mongo、またはその他の任意のデータベースを使用できます。

実行サイクル間で永続化を示す簡単な例を以下に示します。

```python storage.py
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.storage.sqlite import SqliteStorage
from rich.pretty import pprint

agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    # 実行サイクル間で同じセッションを継続するためにセッションIDを固定します
    session_id="fixed_id_for_demo",
    storage=SqliteStorage(table_name="agent_sessions", db_file="tmp/data.db"),
    add_history_to_messages=True,
    num_history_runs=3,
)
agent.print_response("私の最後の質問は何でしたか？")
agent.print_response("フランスの首都は？")
agent.print_response("私の最後の質問は何でしたか？")
pprint(agent.get_messages_for_session())
```

最初にこれを実行したとき、「私の最後の質問は何でしたか？」という質問への回答は利用できません。しかし、もう一度実行すると、エージェントは正しく回答できるようになります。セッションIDを固定しているため、スクリプトを実行するたびにエージェントは同じセッションから再開します。

## ストレージのメリット

ストレージは通常、エージェントエンジニアリングにおいてあまり議論されない部分ですが、本番のエージェントアプリケーションの縁の下の力持ちであると考えています。

本番環境では、ストレージが必要です。

- セッションの継続：セッション履歴を取得し、中断したところから再開します。
- セッションリストの取得：以前のセッションを継続するには、そのエージェントで利用可能なセッションのリストを維持する必要があります。
- 実行間の状態の保存：エージェントの状態をデータベースまたはファイルに保存して、後で検査できるようにします。

しかし、それだけではありません。

- ストレージは、エージェントのセッションデータを検査および評価のために保存します。
- ストレージは、エージェントの改善に使用できるfew-shotの例を抽出するのに役立ちます。
- ストレージにより、内部監視ツールやダッシュボードを構築できます。

<Warning>
ストレージはエージェントインフラストラクチャの非常に重要な部分であるため、サードパーティにオフロードすることは決してありません。エージェントには、ほぼ常に独自のストレージレイヤーを使用する必要があります。
</Warning>

## エージェントストレージ

エージェントを操作する際、ストレージを使用すると、ユーザーは会話を中断したところから再開できます。すべてのメッセージとエージェントの応答は、選択したデータベースに保存されます。

エージェントにストレージを追加する簡単な例を以下に示します。

```python storage.py
"""依存関係をインストールするには `pip install duckduckgo-search sqlalchemy openai` を実行してください。"""

from agno.agent import Agent
from agno.storage.sqlite import SqliteStorage
from agno.tools.duckduckgo import DuckDuckGoTools

agent = Agent(
    storage=SqliteStorage(
        table_name="agent_sessions", db_file="tmp/data.db", auto_upgrade_schema=True
    ),
    tools=[DuckDuckGoTools()],
    add_history_to_messages=True,
    add_datetime_to_instructions=True,
)
agent.print_response("カナダには何人住んでいますか？")
agent.print_response("彼らの国歌は何ですか？")
agent.print_response("私のメッセージを一つずつリストしてください")
```

## チームストレージ

`Storage`ドライバーはチームとも連携し、マルチエージェント協調システムに永続的なメモリと状態管理を提供します。チームストレージを使用すると、複数のセッションにわたって会話履歴、共有コンテキスト、およびチームの状態を維持できます。

<Note>
強力なマルチエージェントシステムを永続的な状態で構築するために、[チーム](/teams/)とそのストレージ機能の詳細をご覧ください。
</Note>

## ワークフローストレージ

Agnoのストレージシステムはワークフローとも連携し、状態管理を備えたより複雑なマルチエージェントシステムを可能にします。これにより、ワークフローセッション全体での永続的な会話とキャッシュされた結果が可能になります。

<Note>
状態管理を備えた強力なマルチエージェントシステムを構築するために、[ワークフロー](/workflows/)でのストレージの使用方法の詳細をご覧ください。
</Note>
