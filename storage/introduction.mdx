使用**会话存储**将 Agent 会话和状态持久化到数据库或文件。

<Tip>
**为什么我们需要会话存储？**

Agent 是短暂的，内置的内存仅在当前执行周期内有效。

在生产环境中，我们通过 API 提供（或触发）Agent，并且需要在多次请求中继续相同的会话。存储会将会话历史记录和状态持久化到数据库中，使我们能够接续上次中断的地方。

存储还允许我们检查和评估 Agent 会话，提取少样本示例，并构建内部监控工具。它让我们能够**查看数据**，这有助于我们构建更好的 Agent。
</Tip>

向 Agent、Team 或 Workflow 添加存储非常简单，只需提供一个 `Storage` 驱动程序，Agno 即可处理其余部分。您可以使用 Sqlite、Postgres、Mongo 或任何您想要的数据库。

这是一个演示跨执行周期持久化的简单示例：

```python storage.py
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.storage.sqlite import SqliteStorage
from rich.pretty import pprint

agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    # 固定会话 ID 以在执行周期之间继续相同的会话
    session_id="fixed_id_for_demo",
    storage=SqliteStorage(table_name="agent_sessions", db_file="tmp/data.db"),
    add_history_to_messages=True,
    num_history_runs=3,
)
agent.print_response("我上次问的是什么？")
agent.print_response("法国的首都是哪里？")
agent.print_response("我上次问的是什么？")
pprint(agent.get_messages_for_session())
```

第一次运行此代码时，“我上次问的是什么？”的答案将不可用。但再次运行，Agent 将能够正确回答。因为我们固定了会话 ID，所以每次运行脚本时，Agent 都会从同一个会话继续。

## 存储的好处

存储通常是 Agent 工程中一个被低估的部分——但我们认为它是生产环境中 Agent 应用的无名英雄。

在生产环境中，您需要存储来：

- 继续会话：检索会话历史记录并接续上次中断的地方。
- 获取会话列表：要继续之前的会话，您需要维护该 Agent 可用的会话列表。
- 在运行之间保存状态：将 Agent 的状态保存到数据库或文件，以便稍后检查。

但还有更多：

- 存储保存我们的 Agent 的会话数据以供检查和评估。
- 存储帮助我们提取少样本示例，可用于改进 Agent。
- 存储使我们能够构建内部监控工具和仪表板。

<Warning>
存储是您 Agent 基础设施的关键部分，绝不应外包给第三方。您几乎应该始终为您的 Agent 使用自己的存储层。
</Warning>

## Agent 存储

在处理 Agent 时，存储允许用户接续之前的对话。每条消息以及 Agent 的响应都会保存到您选择的数据库中。

这是一个向 Agent 添加存储的简单示例：

```python storage.py
"""运行 `pip install duckduckgo-search sqlalchemy openai` 来安装依赖项。"""

from agno.agent import Agent
from agno.storage.sqlite import SqliteStorage
from agno.tools.duckduckgo import DuckDuckGoTools

agent = Agent(
    storage=SqliteStorage(
        table_name="agent_sessions", db_file="tmp/data.db", auto_upgrade_schema=True
    ),
    tools=[DuckDuckGoTools()],
    add_history_to_messages=True,
    add_datetime_to_instructions=True,
)
agent.print_response("加拿大有多少人？")
agent.print_response("他们的国歌是什么？")
agent.print_response("逐一列出我的消息")
```

## Team 存储

`Storage` 驱动程序也适用于 Team，为多 Agent 协作系统提供持久化内存和状态管理。通过 Team 存储，您可以跨多个会话维护对话历史记录、共享上下文和 Team 状态。

<Note>
了解更多关于 [Teams](/teams/) 及其存储功能的信息，以构建具有持久化状态的强大多 Agent 系统。
</Note>

## Workflow 存储

Agno 中的存储系统也适用于 Workflows，为具有状态管理能力提供更复杂的多 Agent 系统。这使得跨 Workflow 会话的持久化对话和缓存结果成为可能。
