title: Sessions
---

当我们调用 `Agent.run()` 时，它会创建一个无状态的、单一的 Agent 运行。

但如果我们想继续这次运行，也就是说进行多轮对话呢？这时就需要用到 `sessions`。Session 是连续运行的集合。

在实践中，一个 session 就是用户和 Agent 之间的一次多轮对话。通过 `session_id`，我们可以跨多次运行连接对话历史和状态。

让我们概述一些关键概念：

- **Session (会话):** Session 是连续运行的集合，就像用户和 Agent 之间的一次多轮对话。Session 由 `session_id` 标识，每一次交互都是一次 **run**。
- **Run (运行):** 与 Agent 的每一次交互（即聊天或回合）都称为一次 **run**。Run 由 `run_id` 标识，调用 `Agent.run()` 时会创建一个新的 `run_id`。
- **Messages (消息):** 是模型和 Agent 之间发送的单个消息。消息是 Agent 和模型之间的通信协议。

让我们从一个创建了单次运行的 Agent 的示例开始。系统会自动生成一个 `run_id`，以及一个 `session_id`（因为我们没有提供一个来继续对话）。这次运行尚未与用户关联。

```python
from typing import Iterator
from agno.agent import Agent, RunResponse
from agno.models.openai import OpenAIChat
from agno.utils.pprint import pprint_run_response

agent = Agent(model=OpenAIChat(id="gpt-4o-mini"))

# 运行 Agent 并将响应作为变量返回
agent.print_response("Tell me a 5 second short story about a robot")
```

## 多用户、多会话 Agent

与 Agent 交互的每个用户都会获得一组唯一的会话，并且您可以让多个用户同时与同一个 Agent 进行交互。

设置 `user_id` 以将用户与其在 Agent 上的会话关联起来。

在下面的示例中，我们设置了 `session_id` 来演示如何同时与多个用户进行多轮对话。在生产环境中，`session_id` 是自动生成的。

<Note>

注意：多用户、多会话目前仅支持 `Memory.v2`，它将在下一个版本中成为默认的内存实现。

</Note>

```python
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.memory.v2 import Memory

agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    # 多用户、多会话仅支持 Memory.v2
    memory=Memory(),
    add_history_to_messages=True,
    num_history_runs=3,
)

user_1_id = "user_101"
user_2_id = "user_102"

user_1_session_id = "session_101"
user_2_session_id = "session_102"

# 开始与用户 1 的会话
agent.print_response(
    "Tell me a 5 second short story about a robot.",
    user_id=user_1_id,
    session_id=user_1_session_id,
)
# 继续与用户 1 的会话
agent.print_response("Now tell me a joke.", user_id=user_1_id, session_id=user_1_session_id)

# 开始与用户 2 的会话
agent.print_response("Tell me about quantum physics.", user_id=user_2_id, session_id=user_2_session_id)
# 继续与用户 2 的会话
agent.print_response("What is the speed of light?", user_id=user_2_id, session_id=user_2_session_id)

# 要求 Agent 总结对话内容，这将使用之前消息的历史记录
agent.print_response(
    "Give me a summary of our conversation.",
    user_id=user_1_id,
    session_id=user_1_session_id,
)
```

## 获取最近 N 个会话的消息

在某些场景下，您可能希望获取最近 N 个会话的消息，以提供上下文或对话的连续性。

以下是如何实现此目的的示例：

```python
# 在运行脚本前删除临时数据库文件
import os

from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.storage.sqlite import SqliteStorage

os.remove("tmp/data.db")

agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    user_id="user_1",
    storage=SqliteStorage(table_name="agent_sessions_new", db_file="tmp/data.db"),
    search_previous_sessions_history=True,  # 允许搜索之前的会话
    num_history_sessions=2,  # 仅包含最近的 2 个会话进行搜索，以避免上下文长度问题
    show_tool_calls=True,
)

session_1_id = "session_1_id"
session_2_id = "session_2_id"
session_3_id = "session_3_id"
session_4_id = "session_4_id"
session_5_id = "session_5_id"

agent.print_response("What is the capital of South Africa?", session_id=session_1_id)
agent.print_response("What is the capital of China?", session_id=session_2_id)
agent.print_response("What is the capital of France?", session_id=session_3_id)
agent.print_response("What is the capital of Japan?", session_id=session_4_id)
agent.print_response(
    "What did I discuss in my previous conversations?", session_id=session_5_id
)  # 它应该只包含最近的 2 个会话
```

<Note>

要启用从最近 N 个会话中获取消息，您需要使用以下标志：

- `search_previous_sessions_history`: 设置为 `True` 以允许搜索之前的会话。
- `num_history_sessions`: 指定要包含在搜索中的过去会话数量。在此示例中，它设置为 `2`，仅包含最近的 2 个会话。
建议暂时将此数量保持在 2 或 3，因为数量过多可能会填满模型的上下文长度，从而可能导致性能问题。

这些标志有助于管理上下文长度，并确保只包含相关的会话历史记录。

</Note>
