---
title: セッション
---

`Agent.run()` を呼び出すと、ステートレスで単一の Agent 実行が作成されます。

しかし、この実行を継続したい場合、つまり複数ターンの会話を行いたい場合はどうすればよいでしょうか？そこで `sessions` が登場します。セッションとは、連続する実行の集まりです。

実際には、セッションはユーザーと Agent 間の複数ターンの会話です。`session_id` を使用することで、複数の実行にわたる会話履歴と状態を接続できます。

いくつかの重要な概念を概説しましょう。

- **セッション (Session):** セッションとは、ユーザーと Agent 間の複数ターンの会話のような、連続する実行の集まりです。セッションは `session_id` によって識別され、各ターンは **実行 (run)** です。
- **実行 (Run):** Agent とのすべての対話（チャットまたはターン）は **実行 (run)** と呼ばれます。実行は `run_id` によって識別され、`Agent.run()` が呼び出されると新しい `run_id` が作成されます。
- **メッセージ (Messages):** モデルと Agent 間で送受信される個々のメッセージです。メッセージは Agent とモデル間の通信プロトコルです。

Agent との単一の実行が作成される例から始めましょう。`run_id` は自動的に生成され、`session_id` も生成されます（会話を継続するために提供しなかったため）。この実行はまだユーザーに関連付けられていません。

```python
from typing import Iterator
from agno.agent import Agent, RunResponse
from agno.models.openai import OpenAIChat
from agno.utils.pprint import pprint_run_response

agent = Agent(model=OpenAIChat(id="gpt-4o-mini"))

# Agent を実行し、レスポンスを変数として返す
agent.print_response("Tell me a 5 second short story about a robot")
```

## マルチユーザー、マルチセッション Agent

Agent と対話する各ユーザーは一意のセッションセットを取得し、同じ Agent と同時に複数のユーザーが対話できます。

`user_id` を設定して、ユーザーを Agent とのセッションに接続します。

以下の例では、複数のユーザーが同時に複数ターンの会話を行う方法を示すために `session_id` を設定しています。本番環境では、`session_id` は自動生成されます。

<Note>

注: マルチユーザー、マルチセッションは現在 `Memory.v2` でのみ機能します。これは次回のリリースでデフォルトのメモリ実装になります。

</Note>

```python
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.memory.v2 import Memory

agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    # マルチユーザー、マルチセッションは Memory.v2 でのみ機能します
    memory=Memory(),
    add_history_to_messages=True,
    num_history_runs=3,
)

user_1_id = "user_101"
user_2_id = "user_102"

user_1_session_id = "session_101"
user_2_session_id = "session_102"

# ユーザー 1 とのセッションを開始
agent.print_response(
    "Tell me a 5 second short story about a robot.",
    user_id=user_1_id,
    session_id=user_1_session_id,
)
# ユーザー 1 とのセッションを継続
agent.print_response("Now tell me a joke.", user_id=user_1_id, session_id=user_1_session_id)

# ユーザー 2 とのセッションを開始
agent.print_response("Tell me about quantum physics.", user_id=user_2_id, session_id=user_2_id)
# ユーザー 2 とのセッションを継続
agent.print_response("What is the speed of light?", user_id=user_2_id, session_id=user_2_session_id)

# 会話の要約を Agent に依頼します。これは以前のメッセージの履歴を使用します
agent.print_response(
    "Give me a summary of our conversation.",
    user_id=user_1_id,
    session_id=user_1_session_id,
)
```

## 過去 N セッションからのメッセージの取得

シナリオによっては、コンテキストや会話の継続性を提供するために、過去 N セッションからのメッセージを取得したい場合があります。

以下にその方法を示す例を示します。

```python
# スクリプトを実行する前に tmp db ファイルを削除します
import os

from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.storage.sqlite import SqliteStorage

os.remove("tmp/data.db")

agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    user_id="user_1",
    storage=SqliteStorage(table_name="agent_sessions_new", db_file="tmp/data.db"),
    search_previous_sessions_history=True,  # 過去のセッションの検索を許可
    num_history_sessions=2,  # コンテキスト長の課題を回避するために、検索には最後の 2 セッションのみを含める
    show_tool_calls=True,
)

session_1_id = "session_1_id"
session_2_id = "session_2_id"
session_3_id = "session_3_id"
session_4_id = "session_4_id"
session_5_id = "session_5_id"

agent.print_response("What is the capital of South Africa?", session_id=session_1_id)
agent.print_response("What is the capital of China?", session_id=session_2_id)
agent.print_response("What is the capital of France?", session_id=session_3_id)
agent.print_response("What is the capital of Japan?", session_id=session_4_id)
agent.print_response(
    "What did I discuss in my previous conversations?", session_id=session_5_id
)  # 最後の 2 セッションのみを含める必要があります
```

<Note>

過去 N セッションからのメッセージの取得を有効にするには、次のフラグを使用する必要があります。

- `search_previous_sessions_history`: 前のセッションの検索を許可するには、これを `True` に設定します。
- `num_history_sessions`: 検索に含める過去のセッション数を指定します。この例では、最後の 2 セッションのみを含めるために `2` に設定されています。
コンテキスト長がモデルのコンテキスト長でいっぱいになり、パフォーマンスの問題が発生する可能性があるため、現時点ではこの数を 2 または 3 に保つことをお勧めします。

これらのフラグは、コンテキスト長を管理し、関連するセッション履歴のみが会話に含まれるようにするのに役立ちます。

</Note>
