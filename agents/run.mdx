`Agent.run()` 関数はエージェントを実行し、`RunResponse` オブジェクトまたは `RunResponse` オブジェクトのストリームとして応答を生成します。

多くの例では、ターミナルで応答を印刷するためのヘルパーユーティリティである `agent.print_response()` を使用しています。これは内部的に `agent.run()` を使用しています。

## エージェントの実行

エージェントを実行する方法は次のとおりです。応答は `response` にキャプチャされます。

```python
from typing import Iterator
from agno.agent import Agent, RunResponse
from agno.models.openai import OpenAIChat
from agno.utils.pprint import pprint_run_response

agent = Agent(model=OpenAIChat(id="gpt-4o-mini"))

# エージェントを実行し、応答を変数として返します
response: RunResponse = agent.run("ロボットについての5秒の短い物語を教えてください")

# 応答をマークダウン形式で印刷します
pprint_run_response(response, markdown=True)
```

## RunResponse

`Agent.run()` 関数は、ストリーミングしない場合は `RunResponse` オブジェクトを返します。このオブジェクトには次の属性があります。

## ストリーミング応答

ストリーミングを有効にするには、`run()` を呼び出す際に `stream=True` を設定します。これにより、単一の応答ではなく `RunResponseEvent` オブジェクトのイテレータが返されます。

<Note>
`agno` バージョン `1.6.0` 以降、`Agent.run()` 関数は `RunResponse` オブジェクトのイテレータではなく、`RunResponseEvent` のイテレータを返します。
</Note>

```python
from typing import Iterator
from agno.agent import Agent, RunResponseEvent
from agno.models.openai import OpenAIChat
from agno.utils.pprint import pprint_run_response

agent = Agent(model=OpenAIChat(id="gpt-4-mini"))

# エージェントを実行し、応答をストリームとして返します
response_stream: Iterator[RunResponseEvent] = agent.run(
    "ライオンについての5秒の短い物語を教えてください",
    stream=True
)

# 応答ストリームをマークダウン形式で印刷します
pprint_run_response(response_stream, markdown=True)
```

### 中間ステップのストリーミング

さらに詳細なストリーミングを行うには、`stream_intermediate_steps=True` を設定して中間ステップを有効にすることができます。これにより、エージェントの内部プロセスに関するリアルタイムの更新が提供されます。

```python
# 中間ステップでストリーミング
response_stream: Iterator[RunResponseEvent] = agent.run(
    "ライオンについての5秒の短い物語を教えてください",
    stream=True,
    stream_intermediate_steps=True
)
```

### イベントの処理

応答ストリームを反復処理することで、イベントが到着したときに処理できます。

```python
response_stream = agent.run("あなたのプロンプト", stream=True, stream_intermediate_steps=True)

for event in response_stream:
    if event.event == "RunResponseContent":
        print(f"Content: {event.content}")
    elif event.event == "ToolCallStarted":
        print(f"Tool call started: {event.tool}")
    elif event.event == "ReasoningStep":
        print(f"Reasoning step: {event.content}")
    ...
```

この動作は、[Playground](https://app.agno.com/playground/agents?endpoint=demo.agnoagents.com&agent=reasoning-agent) で実際に確認できます。

### イベントの保存

実行中に発生したすべてのイベントを `RunResponse` オブジェクトに保存できます。

```python
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.utils.pprint import pprint_run_response

agent = Agent(model=OpenAIChat(id="gpt-4o-mini"), store_events=True)

response = agent.run("ライオンについての5秒の短い物語を教えてください", stream=True, stream_intermediate_steps=True)
pprint_run_response(response)

for event in agent.run_response.events:
    print(event.event)
```

デフォルトでは、`RunResponseContentEvent` イベントは保存されません。`events_to_skip` パラメータを設定することで、スキップされるイベントを変更できます。

例：

```python
agent = Agent(model=OpenAIChat(id="gpt-4o-mini"), store_events=True, events_to_skip=[RunEvent.run_started.value])
```

### イベントの種類

エージェントの設定に応じて、`Agent.run()` および `Agent.arun()` 関数によって生成されるイベントは次のとおりです。

#### コアイベント
| イベントタイプ | 説明 |
|------------|-------------|
| `RunStarted` | 実行の開始を示します |
| `RunResponseContent` | モデルの応答テキストを個別のチャンクとして含みます |
| `RunCompleted` | 実行の正常な完了を示します |
| `RunError` | 実行中にエラーが発生したことを示します |
| `RunCancelled` | 実行がキャンセルされたことを示します |

#### コントロールフローイベント
| イベントタイプ | 説明 |
|------------|-------------|
| `RunPaused` | 実行が一時停止されたことを示します |
| `RunContinued` | 一時停止された実行が再開されたことを示します |

#### ツールイベント
| イベントタイプ | 説明 |
|------------|-------------|
| `ToolCallStarted` | ツール呼び出しの開始を示します |
| `ToolCallCompleted` | ツール呼び出しの結果を含む、ツール呼び出しの完了を示します |

#### 推論イベント
| イベントタイプ | 説明 |
|------------|-------------|
| `ReasoningStarted` | エージェントの推論プロセスの開始を示します |
| `ReasoningStep` | 推論プロセスの単一のステップを含みます |
| `ReasoningCompleted` | 推論プロセスの完了を示します |

#### メモリイベント
| イベントタイプ | 説明 |
|------------|-------------|
| `MemoryUpdateStarted` | エージェントがメモリを更新していることを示します |
| `MemoryUpdateCompleted` | メモリ更新の完了を示します |

[RunResponseEvent](/reference/agents/run-response) に関する詳細なドキュメントを参照してください。
