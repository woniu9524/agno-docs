**エージェントはツールを使用してアクションを実行し、外部システムと対話します**。

ツールは、エージェントがタスクを達成するために実行できる関数です。たとえば、Web の検索、SQL の実行、メールの送信、API の呼び出しなどです。Python 関数をツールとして使用することも、事前構築済みの**ツールキット**を使用することもできます。一般的な構文は次のとおりです。

```python
from agno.agent import Agent

agent = Agent(
    # 関数またはツールキットを追加
    tools=[...],
    # エージェントの応答でツール呼び出しを表示
    show_tool_calls=True
)
```

## ツールキットの使用

Agno は、エージェントに追加できる多くの事前構築済み**ツールキット**を提供します。たとえば、DuckDuckGo ツールキットを使用して Web を検索してみましょう。

<Tip>より多くのツールキットは、[ツールキット](/tools/toolkits) ガイドで見つけることができます。</Tip>

<Steps>
  <Step title="Web Search Agent の作成">
    `web_search.py` ファイルを作成します。

    ```python web_search.py
    from agno.agent import Agent
    from agno.tools.duckduckgo import DuckDuckGoTools

    agent = Agent(tools=[DuckDuckGoTools()], show_tool_calls=True, markdown=True)
    agent.print_response("Whats happening in France?", stream=True)
    ```

  </Step>
  <Step title="エージェントの実行">
    ライブラリをインストールします。

    ```shell
    pip install openai duckduckgo-search agno
    ```

    エージェントを実行します。

    ```shell
    python web_search.py
    ```

  </Step>
</Steps>

## 独自のツールの作成

より詳細な制御を行うには、独自の Python 関数を作成し、それをエージェントのツールとして追加します。たとえば、`get_top_hackernews_stories` ツールをエージェントに追加する方法を次に示します。

```python hn_agent.py
import json
import httpx

from agno.agent import Agent

def get_top_hackernews_stories(num_stories: int = 10) -> str:
    """Hacker News のトップストーリーを取得するためにこの関数を使用します。

    Args:
        num_stories (int): 返されるストーリーの数。デフォルトは 10 です。

    Returns:
        str: トップストーリーの JSON 文字列。
    """

    # トップストーリー ID を取得
    response = httpx.get('https://hacker-news.firebaseio.com/v0/topstories.json')
    story_ids = response.json()

    # ストーリーの詳細を取得
    stories = []
    for story_id in story_ids[:num_stories]:
        story_response = httpx.get(f'https://hacker-news.firebaseio.com/v0/item/{story_id}.json')
        story = story_response.json()
        if "text" in story:
            story.pop("text", None)
        stories.append(story)
    return json.dumps(stories)

agent = Agent(tools=[get_top_hackernews_stories], show_tool_calls=True, markdown=True)
agent.print_response("Summarize the top 5 stories on hackernews?", stream=True)
```

## 属性

次の属性により、`Agent` はツールを使用できます。

| パラメータ | タイプ | デフォルト | 説明 |
|-----------|------|---------|-------------|
| `tools` | `List[Union[Tool, Toolkit, Callable, Dict, Function]]` | - | モデルに提供されるツールのリスト。ツールは、モデルが JSON 入力を生成できる関数です。 |
| `show_tool_calls` | `bool` | `False` | モデルの応答でツール呼び出しのシグネチャを表示します。 |
| `tool_call_limit` | `int` | - | 単一の実行で許可されるツールの呼び出しの最大数。 |
| `tool_choice` | `Union[str, Dict[str, Any]]` | - | モデルによって呼び出されるツール (存在する場合) を制御します。"none" は、モデルがツールを呼び出さずにメッセージを生成することを意味します。"auto" は、モデルがメッセージの生成またはツールの呼び出しを選択できることを意味します。`{"type": "function", "function": {"name": "my_function"}}` を介して特定の関数を指定すると、モデルはそのツールを呼び出すようになります。"none" は、ツールが存在しない場合のデフォルトです。ツールが存在する場合のデフォルトは "auto" です。 |
| `read_chat_history` | `bool` | `False` | モデルがチャット履歴を読み取ることを許可するツールを追加します。 |
| `search_knowledge` | `bool` | `False` | モデルがナレッジベースを検索することを許可するツールを追加します (エージェンティック RAG とも呼ばれます)。 |
| `update_knowledge` | `bool` | `False` | モデルがナレッジベースを更新することを許可するツールを追加します。 |
| `read_tool_call_history` | `bool` | `False` | モデルがツール呼び出し履歴を取得することを許可するツールを追加します。 |
