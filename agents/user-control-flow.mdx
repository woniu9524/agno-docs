Agno のユーザー制御フローを使用すると、「Human in the Loop」パターンを実装できます。これは、エージェントの実行中に人間の監視と入力が必要となるパターンです。これは以下に不可欠です。

- 機密性の高い操作の検証
- 実行前のツール呼び出しのレビュー
- 意思決定のためのユーザー入力の収集
- 外部ツール実行の管理

## ユーザー制御フローの種類

Agno は、4 つの主要なユーザー制御フローの種類をサポートしています。

1. **ユーザー確認**: ツール呼び出しを実行する前に、明示的なユーザーの承認を要求します。
2. **ユーザー入力**: 実行中にユーザーから特定の情報を収集します。
3. **動的ユーザー入力**: エージェントが必要に応じてユーザー入力を収集できるようにします。
4. **外部ツール実行**: エージェントの制御外でツールを実行します。

## エージェントの実行の一時停止

ユーザー制御フローはエージェントの実行を中断し、人間の監視を必要とします。その後、`continue_run` メソッドを呼び出すことで実行を再開できます。

例：

```python
agent.run("機密性の高い操作を実行します")

if agent.is_paused:
    # エージェントはユーザー入力中に一時停止します
    # ... 他のタスクを実行します

    # ユーザーは実行を続行できます
    response = agent.continue_run()
    # または response = await agent.acontinue_run()
```

`continue_run` メソッドは、一時停止した時点のエージェントの状態を引き継いで実行を続行します。特定の実行の `run_response` または `run_id` を `continue_run` メソッドに渡すこともできます。

## ユーザー確認

ユーザー確認を使用すると、実行を一時停止し、ツール呼び出しに進む前に明示的なユーザーの承認を要求できます。これは以下に役立ちます。

- 機密性の高い操作
- データを変更する API 呼び出し
- 重大な結果を伴うアクション

次の例は、ユーザー確認の実装方法を示しています。

```python
from agno.tools import tool
from agno.agent import Agent
from agno.models.openai import OpenAIChat

@tool(requires_confirmation=True)
def sensitive_operation(data: str) -> str:
    """確認が必要な機密性の高い操作を実行します。"""
    # ここに実装
    return "操作が完了しました"

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    tools=[sensitive_operation],
)

# エージェントを実行します
agent.run("機密性の高い操作を実行します")

# 確認を処理します
if agent.is_paused:
    for tool in agent.run_response.tools_requiring_confirmation:
        # ユーザー確認を取得します
        print(f"ツール {tool.tool_name}({tool.tool_args}) は確認が必要です")
        confirmed = input(f"確認しますか？ (y/n): ").lower() == "y"
        tool.confirmed = confirmed

  # 実行を続行します
  response = agent.continue_run()
```

ツールキット内のどのツールが確認を必要とするかを指定することもできます。

```python
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.tools import tool
from agno.tools.yfinance import YFinanceTools
from agno.utils import pprint

agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[YFinanceTools(requires_confirmation_tools=["get_current_stock_price"])],
)

agent.run("Apple の現在の株価を教えてください。")
if agent.is_paused:
    for tool in agent.run_response.tools_requiring_confirmation:
        print(f"ツール {tool.tool_name}({tool.tool_args}) は確認が必要です")
        confirmed = input(f"確認しますか？ (y/n): ").lower() == "y"

        if message == "n":
            tool.confirmed = False
        else:
            # ツールをインプレースで更新します
            tool.confirmed = True

    run_response = agent.continue_run()
    pprint.pprint_run_response(run_response)
```

## ユーザー入力

ユーザー入力フローを使用すると、実行中にユーザーから特定の情報を収集できます。これは以下に役立ちます。

- 必須パラメータの収集
- ユーザー設定の取得
- 不足情報の収集

以下の例では、`send_email` ツールに必要なすべての入力をユーザーから取得します。

```python
from typing import List
from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.tools.function import UserInputField

# ツールのドキュメント文字列は引き続き提供します。これは `user_input_schema` を埋めるために使用されます。
@tool(requires_user_input=True)
def send_email(to: str, subject: str, body: str) -> dict:
    """ユーザーにメールを送信します。

    Args:
        to (str): メールを送信するアドレス。
        subject (str): メールの件名。
        body (str): メールの本文。
    """
    # ここに実装
    return f"メールを {to} に件名 {subject}、本文 {body} で送信しました"

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    tools=[send_email],
)

agent.run("メールを送信してください")
if agent.is_paused:
    for tool in agent.run_response.tools_requiring_user_input:
        input_schema: List[UserInputField] = tool.user_input_schema

        for field in input_schema:
            # ユーザーにフィールド情報を表示します
            print(f"\nフィールド: {field.name} ({field.field_type.__name__}) -> {field.description}")

            # ユーザー入力を取得します
            user_value = input(f"{field.name} の値を入力してください: ")

            # フィールド値を更新します
            field.value = user_value

    run_response = (
        agent.continue_run()
    )
```

`RunResponse` オブジェクトにはツールのリストがあり、`requires_user_input` の場合、入力が必要なツールには `user_input_schema` が設定されています。
これは `UserInputField` オブジェクトのリストです。

```python
class UserInputField:
    name: str  # フィールド名
    field_type: Type  # 必須フィールドの型
    description: Optional[str] = None  # フィールドの説明
    value: Optional[Any] = None  # フィールドの値。エージェントまたはユーザーによって入力されます。
```

ユーザーに入力してもらうフィールドを指定し、残りのフィールドはエージェントが提供するようにすることもできます。

```python

# ユーザーがすべてのフィールドを提供する必要がある場合は、user_input_fields を指定するか、空のままにします。
@tool(requires_user_input=True, user_input_fields=["to_address"])
def send_email(subject: str, body: str, to_address: str) -> str:
    """
    メールを送信します。

    Args:
        subject (str): メールの件名。
        body (str): メールの本文。
        to_address (str): メールを送信するアドレス。
    """
    return f"件名 '{subject}'、本文 '{body}' で {to_address} にメールを送信しました"

agent = Agent(
    model=OpenAIChat(id="gpt-4o"),
    tools=[send_email],
)

agent.run("件名を 'Hello'、本文を 'Hello, world!' としてメールを送信してください")
if agent.is_paused:
    for tool in agent.run_response.tools_requiring_user_input:
        input_schema: List[UserInputField] = tool.user_input_schema

        for field in input_schema:
            # ユーザーにフィールド情報を表示します
            print(f"\nフィールド: {field.name} ({field.field_type.__name__}) -> {field.description}")

            # ユーザー入力を取得します (値が設定されていない場合、ユーザーが値を提供する必要があることを意味します)
            if field.value is None:
                user_value = input(f"{field.name} の値を入力してください: ")
                field.value = user_value
            else:
                print(f"エージェントによって提供された値: {field.value}")

    run_response = (
        agent.continue_run()
    )
```

## 動的ユーザー入力

このパターンは、エージェントが必要なときにユーザー入力を示すためのツールを提供します。これは以下に最適です。

- ユーザーがエージェントとどのように対話するか不明な場合
- ユーザーとのフォームのような対話をしたい場合

次の例では、エージェントが必要に応じてユーザーフィードバックを収集できるように、特殊なツールを使用します。

```python
from typing import Any, Dict

from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.tools import tool
from agno.tools.toolkit import Toolkit
from agno.tools.user_control_flow import UserControlFlowTools
from agno.utils import pprint

# メール処理の例ツールキット
class EmailTools(Toolkit):
    def __init__(self, *args, **kwargs):
        super().__init__(
            name="EmailTools", tools=[self.send_email, self.get_emails], *args, **kwargs
        )

    def send_email(self, subject: str, body: str, to_address: str) -> str:
        """指定されたアドレスに、指定された件名と本文でメールを送信します。

        Args:
            subject (str): メールの件名。
            body (str): メールの本文。
            to_address (str): メールを送信するアドレス。
        """
        return f"メールを {to_address} に件名 {subject}、本文 {body} で送信しました"

    def get_emails(self, date_from: str, date_to: str) -> str:
        """指定された日付間のすべてのメールを取得します。

        Args:
            date_from (str): 開始日。
            date_to (str): 終了日。
        """
        return [
            {
                "subject": "Hello",
                "body": "Hello, world!",
                "to_address": "test@test.com",
                "date": date_from,
            },
            {
                "subject": "Random other email",
                "body": "This is a random other email",
                "to_address": "john@doe.com",
                "date": date_to,
            },
        ]


agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[EmailTools(), UserControlFlowTools()],
    markdown=True,
    debug_mode=True,
)

run_response = agent.run("本文を 'How is it going in Tokyo?' としてメールを送信してください")

# エージェントがユーザー入力に満足するまで実行を続行するために while ループを使用します
while run_response.is_paused:
    for tool in run_response.tools_requiring_user_input:
        input_schema: List[UserInputField] = tool.user_input_schema

        for field in input_schema:
            # ユーザーにフィールド情報を表示します
            print(f"\nフィールド: {field.name} ({field.field_type.__name__}) -> {field.description}")

            # ユーザー入力を取得します (値が設定されていない場合、ユーザーが値を提供する必要があることを意味します)
            if field.value is None:
                user_value = input(f"{field.name} の値を入力してください: ")
                field.value = user_value
            else:
                print(f"エージェントによって提供された値: {field.value}")

    run_response = agent.continue_run(run_response=run_response)

    # エージェントが入力のために一時停止されていない場合、完了です
    if not run_response.is_paused:
        pprint.pprint_run_response(run_response)
        break
```

## 外部ツール実行

外部ツール実行を使用すると、エージェントの制御外でツールを実行できます。これは以下に役立ちます。

- 外部サービス呼び出し
- データベース操作

```python
import subprocess

from agno.agent import Agent
from agno.models.openai import OpenAIChat
from agno.tools import tool
from agno.utils import pprint


# エージェントが呼び出しを知るためには、正しい名前、引数、ドキュメント文字列を持つツールを作成する必要があります。
@tool(external_execution=True)
def execute_shell_command(command: str) -> str:
    """シェルコマンドを実行します。

    Args:
        command (str): 実行するシェルコマンド

    Returns:
        str: シェルコマンドの出力
    """
    return subprocess.check_output(command, shell=True).decode("utf-8")


agent = Agent(
    model=OpenAIChat(id="gpt-4o-mini"),
    tools=[execute_shell_command],
    markdown=True,
)

run_response = agent.run("現在のディレクトリにあるファイルは何ですか？")
if run_response.is_paused:
    for tool in run_response.tools_awaiting_external_execution:
        if tool.tool_name == execute_shell_command.name:
            print(f"{tool.tool_name} を引数 {tool.tool_args} で外部実行します")

            # ツールを自分で実行します。任意の関数またはプロセスを実行し、tool_args を入力として使用できます。
            result = execute_shell_command.entrypoint(**tool.tool_args)
            # エージェントが続行できるように、結果を実行オブジェクトに設定する必要があります
            tool.result = result

    run_response = agent.continue_run()
    pprint.pprint_run_response(run_response)
```

## ベストプラクティス

1. **ユーザー入力をサニタイズする**: セキュリティ上の脆弱性を防ぐために、常にユーザー入力を検証およびサニタイズしてください。
2. **エラー処理**: ユーザー入力と外部呼び出しに対して、常に適切なエラー処理を実装してください。
3. **入力検証**: 処理する前にユーザー入力を検証してください。
