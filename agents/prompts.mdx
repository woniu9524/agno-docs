title: Prompts
---

エージェントには `description` と `instructions`、およびその他のいくつかの設定を使用してプロンプトを送信します。これらの設定は、言語モデルに送信される **システム** メッセージの構築に使用されます。

これらのプロンプトがどのように作成されるかを理解することで、より良いエージェントを構築できます。

主要な2つのパラメータは次のとおりです。

1. **Description**: エージェントの全体的な動作をガイドする説明。
2. **Instructions**: 目標を達成するための、正確でタスク固有の指示のリスト。

<Note>
  Description と instructions はフォーマット上の利点を提供するだけであり、情報の変更や抽象化は行いません。独自のシステムプロンプトを提供するために、常に `system_message` を設定できます。
</Note>

## システムメッセージ

システムメッセージは、`description`、`instructions`、およびその他のいくつかの設定を使用して作成されます。`description` はシステムメッセージの先頭に追加され、`instructions` は「Instructions」の後にリストとして追加されます。たとえば、次のようになります。

```python instructions.py
from agno.agent import Agent

agent = Agent(
    description="You are a famous short story writer asked to write for a magazine",
    instructions=["You are a pilot on a plane flying from Hawaii to Japan."],
    markdown=True,
    debug_mode=True,
)
agent.print_response("Tell me a 2 sentence horror story.", stream=True)
```

## システムメッセージを直接設定する

`system_message` パラメータを使用して、システムメッセージを手動で設定できます。

```python
from agno.agent import Agent

agent = Agent(system_message="Share a 2 sentence story about")
agent.print_response("Love in the year 12000.")
```

<Tip>
Groq の `llama-3.2-11b-vision-preview` のような一部のモデルプロバイダー経由のモデルでは、他のメッセージとともにシステムメッセージは必要ありません。システムメッセージを削除するには、`create_default_system_message=False` および `system_message=None` を設定します。また、`markdown=True` が設定されている場合、システムメッセージが追加されるため、削除するか、システムメッセージを明示的に無効にしてください。
</Tip>

## ユーザーメッセージ

`Agent.run()` または `Agent.print_response()` 関数に送信される入力 `message` は、ユーザーメッセージとして使用されます。

## デフォルトのシステムメッセージ

Agent はデフォルトのシステムメッセージを作成します。これは、次のパラメータを使用してカスタマイズできます。

| パラメータ | タイプ | デフォルト | 説明 |
|-----------|------|---------|-------------|
| `description` | `str` | `None` | エージェントの説明で、システムメッセージの先頭に追加されます。 |
| `goal` | `str` | `None` | エージェントが達成すべきタスクを記述します。 |
| `instructions` | `List[str]` | `None` | `<instructions>` タグにシステムプロンプトに追加される指示のリスト。`markdown`、`output_model` などの値に応じてデフォルトの指示も作成されます。 |
| `additional_context` | `str` | `None` | システムメッセージの末尾に追加される追加コンテキスト。 |
| `expected_output` | `str` | `None` | エージェントからの期待される出力を提供します。これはシステムメッセージの末尾に追加されます。 |
| `markdown` | `bool` | `False` | 出力をマークダウンを使用してフォーマットする指示を追加します。 |
| `add_datetime_to_instructions` | `bool` | `False` | True の場合、現在の日時をプロンプトに追加して、エージェントに時間の感覚を与えます。これにより、「明日」のような相対的な時間を使用できるようになります。 |
| `system_message` | `str` | `None` | システムプロンプト: システムプロンプトを文字列として提供します。 |
| `system_message_role` | `str` | `system` | システムメッセージのロール。 |
| `create_default_system_message` | `bool` | `True` | True の場合、エージェント設定を使用してデフォルトのシステムプロンプトを構築して使用します。 |

<Tip>
`create_default_system_message=False` を設定して、デフォルトのシステムメッセージを無効にします。
</Tip>

## デフォルトのユーザーメッセージ

Agent はデフォルトのユーザーメッセージを作成します。これは、入力メッセージまたは `enable_rag=True` の場合の `context` を持つメッセージです。デフォルトのユーザーメッセージは、次を使用してカスタマイズできます。

| パラメータ | タイプ | デフォルト | 説明 |
|-----------|------|---------|-------------|
| `context` | `str` | `None` | ユーザーメッセージの末尾に追加される追加コンテキスト。 |
| `add_context` | `bool` | `False` | True の場合、コンテキストをユーザープロンプトに追加します。 |
| `resolve_context` | `bool` | `True` | True の場合、コンテキストを解決します（つまり、コンテキスト内の関数を呼び出します）その後、ユーザープロンプトに追加します。 |
| `add_references` | `bool` | `False` | ナレッジベースからの参照をプロンプトに追加することにより、RAG を有効にします。 |
| `retriever` | `Callable` | `None` | ユーザーメッセージに追加する参照を取得するための関数。この関数は、`add_references` が True の場合に呼び出されます。 |
| `references_format` | `Literal["json", "yaml"]` | `"json"` | 参照のフォーマット。 |
| `add_history_to_messages` | `bool` | `False` | True の場合、チャット履歴をモデルに送信されるメッセージに追加します。 |
| `num_history_responses` | `int` | `3` | メッセージに追加される履歴応答の数。 |
| `user_message` | `Union[List, Dict, str]` | `None` | ユーザープロンプトを文字列として提供します。注意: これは、実行関数に送信されたメッセージを無視します。 |
| `user_message_role` | `str` | `user` | ユーザーメッセージのロール。 |
| `create_default_user_message` | `bool` | `True` | True の場合、参照とチャット履歴を使用してデフォルトのユーザープロンプトを構築します。 |

<Tip>
`create_default_user_message=False` を設定して、デフォルトのユーザーメッセージを無効にします。
</Tip>
