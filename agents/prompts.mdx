title: Prompts
---

我们通过 `description` 和 `instructions` 以及其他一些设置来提示 Agents。这些设置用于构建发送给语言模型的**系统**消息。

了解这些提示的创建方式将有助于您构建更好的 Agents。

关键的 2 个参数是：

1. **Description**: 描述，用于指导 Agent 的整体行为。
2. **Instructions**: 一系列精确的、特定于任务的指令，说明如何实现其目标。

<Note>
  Description 和 instructions 仅提供格式化上的便利，我们不会更改或抽象任何信息，您可以随时设置 `system_message` 来提供您自己的系统提示。
</Note>

## 系统消息

系统消息是使用 `description`、`instructions` 和其他一些设置创建的。`description` 添加到系统消息的开头，`instructions` 作为列表添加到“Instructions”之后。例如：

```python instructions.py
from agno.agent import Agent

agent = Agent(
    description="You are a famous short story writer asked to write for a magazine",
    instructions=["You are a pilot on a plane flying from Hawaii to Japan."],
    markdown=True,
    debug_mode=True,
)
agent.print_response("Tell me a 2 sentence horror story.", stream=True)
```

## 直接设置系统消息

您可以使用 `system_message` 参数手动设置系统消息。

```python
from agno.agent import Agent

agent = Agent(system_message="Share a 2 sentence story about")
agent.print_response("Love in the year 12000.")
```

<Tip>
某些模型通过某些模型提供商（例如 Groq 上的 `llama-3.2-11b-vision-preview`）需要与其他消息一起使用，无需系统消息。要删除系统消息，请设置 `create_default_system_message=False` 和 `system_message=None`。此外，如果设置了 `markdown=True`，它将添加一个系统消息，因此请将其删除或显式禁用系统消息。
</Tip>

## 用户消息

发送给 `Agent.run()` 或 `Agent.print_response()` 函数的输入 `message` 用作用户消息。

## 默认系统消息

Agent 会创建一个默认的系统消息，可以使用以下参数进行自定义：

| 参数 | 类型 | 默认值 | 描述 |
|---|---|---|---|
| `description` | `str` | `None` | Agent 的描述，添加到系统消息的开头。 |
| `goal` | `str` | `None` | 描述 Agent 应完成的任务。 |
| `instructions` | `List[str]` | `None` | 添加到系统提示中的指令列表，包含在 `<instructions>` 标签中。根据 `markdown`、`output_model` 等值，还会创建默认指令。 |
| `additional_context` | `str` | `None` | 添加到系统消息末尾的附加上下文。 |
| `expected_output` | `str` | `None` | 提供 Agent 的预期输出。这会添加到系统消息的末尾。 |
| `markdown` | `bool` | `False` | 添加一条指令，使用 markdown 格式化输出。 |
| `add_datetime_to_instructions` | `bool` | `False` | 如果为 True，则将当前日期时间添加到提示中，让 Agent 了解时间。这允许在提示中使用相对时间，如“明天”。 |
| `system_message` | `str` | `None` | 系统提示：将系统提示作为字符串提供。 |
| `system_message_role` | `str` | `system` | 系统消息的角色。 |
| `create_default_system_message` | `bool` | `True` | 如果为 True，则使用 Agent 设置构建默认系统提示。 |

<Tip>
通过设置 `create_default_system_message=False` 来禁用默认系统消息。
</Tip>

## 默认用户消息

Agent 会创建一个默认的用户消息，该消息是输入消息，或者是在 `enable_rag=True` 时带有 `context` 的消息。默认用户消息可以通过以下方式自定义：

| 参数 | 类型 | 默认值 | 描述 |
|---|---|---|---|
| `context` | `str` | `None` | 添加到用户消息末尾的附加上下文。 |
| `add_context` | `bool` | `False` | 如果为 True，则将上下文添加到用户提示中。 |
| `resolve_context` | `bool` | `True` | 如果为 True，则在将上下文添加到用户提示之前解析上下文（即调用上下文中的任何函数）。 |
| `add_references` | `bool` | `False` | 通过添加知识库中的引用来启用 RAG。 |
| `retriever` | `Callable` | `None` | 用于获取要添加到 user_message 的引用的函数。如果提供了此函数，则在 `add_references` 为 True 时调用它。 |
| `references_format` | `Literal["json", "yaml"]` | `"json"` | 引用的格式。 |
| `add_history_to_messages` | `bool` | `False` | 如果为 True，则将聊天历史记录添加到发送给模型的消息中。 |
| `num_history_responses` | `int` | `3` | 要添加到消息中的历史响应数量。 |
| `user_message` | `Union[List, Dict, str]` | `None` | 将用户提示作为字符串提供。注意：这将忽略发送到 run 函数的消息。 |
| `user_message_role` | `str` | `user` | 用户消息的角色。 |
| `create_default_user_message` | `bool` | `True` | 如果为 True，则使用引用和聊天历史记录构建默认用户提示。 |

<Tip>
通过设置 `create_default_user_message=False` 来禁用默认用户消息。
</Tip>
