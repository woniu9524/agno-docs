**Knowledge** とは、エージェントが実行時に**検索**できるドメイン固有の情報であり、より良い意思決定（動的な少数ショット学習）と正確な応答（エージェント型RAG）を提供します。KnowledgeはベクトルDBに保存され、この**オンデマンド検索**パターンはエージェント型RAGと呼ばれます。

<Accordion title="動的な少数ショット学習: Text2Sql Agent" icon="database">
例: Text2Sql Agentを構築する場合、テーブルスキーマ、カラム名、データ型、クエリ例、一般的な「落とし穴」を提供して、可能な限り最良のSQLクエリを生成できるようにする必要があります。

これらすべてをシステムプロンプトに入れることはできません。代わりに、この情報をベクトルデータベースに保存し、エージェントが実行時にクエリできるようにします。

この情報を使用して、エージェントは可能な限り最良のSQLクエリを生成できます。これは動的な少数ショット学習と呼ばれます。
</Accordion>

**Agno AgentsはデフォルトでAgentic RAGを使用します**。これは、エージェントに`knowledge`を提供すると、タスクを達成するために必要な特定の情報を実行時にこのナレッジベースで検索することを意味します。

エージェントにナレッジを追加するための擬似ステップは次のとおりです。

```python
from agno.agent import Agent, AgentKnowledge

# エージェントのナレッジベースを作成します
knowledge_base = AgentKnowledge(vector_db=...)

# ナレッジベースに情報を追加します
knowledge_base.load_text("The sky is blue")

# ナレッジベースをエージェントに追加し、
# 必要に応じてナレッジベースを検索するためのツールを提供します
agent = Agent(knowledge=knowledge_base, search_knowledge=True)
```

以下の方法でエージェントにナレッジベースへのアクセスを提供できます。

- `search_knowledge=True` を設定すると、エージェントに `search_knowledge_base()` ツールが追加されます。`knowledge` をエージェントに追加すると、`search_knowledge` は **デフォルトで `True`** になります。
- `add_references=True` を設定すると、ナレッジベースからの参照がエージェントのプロンプトに自動的に追加されます。これは従来の2023 RAGアプローチです。

<Tip>

ナレッジベースの検索を完全に制御したい場合は、次のシグネチャを持つ独自の `retriever` 関数を渡すことができます。

```python
def retriever(agent: Agent, query: str, num_documents: Optional[int], **kwargs) -> Optional[list[dict]]:
  ...
```

この関数は `search_knowledge_base()` の呼び出し中に実行され、エージェントがナレッジベースから参照を取得するために使用されます。

</Tip>

## Vector Databases

あらゆる種類のストレージがナレッジベースとして機能できますが、ベクトルデータベースは、密な情報から関連性の高い結果を迅速に取得するための最良のソリューションを提供します。ベクトルデータベースはエージェントと共に次のように使用されます。

<Steps>
  <Step title="情報をチャンク化する">
    ナレッジをより小さなチャンクに分割して、検索クエリが
    関連性の高い結果のみを返すようにします。
  </Step>
  <Step title="ナレッジベースをロードする">
    チャンクを埋め込みベクトルに変換し、ベクトルデータベースに保存します。
  </Step>
  <Step title="ナレッジベースを検索する">
    ユーザーがメッセージを送信すると、入力メッセージを
    埋め込みに変換し、ベクトルデータベースで最近傍を「検索」します。
  </Step>
</Steps>

<Note>
ナレッジフィルターは現在、以下のナレッジベースタイプでサポートされています: <b>PDF</b>、<b>PDF_URL</b>、<b>Text</b>、<b>JSON</b>、および <b>DOCX</b>。
詳細については、[Knowledge Filters ドキュメント](/filters/introduction)を参照してください。
</Note>

## 例: PDFナレッジベースを持つRAG Agent

PDFから質問に答える**RAG Agent**を構築しましょう。

### ステップ1: PgVectorを実行する

`PgVector`をベクトルDBとして使用します。これはエージェントのストレージも提供できます。

[docker desktop](https://docs.docker.com/desktop/install/mac-install/)をインストールし、以下を使用してポート **5532** で **PgVector** を実行します。

```bash
docker run -d \
  -e POSTGRES_DB=ai \
  -e POSTGRES_USER=ai \
  -e POSTGRES_PASSWORD=ai \
  -e PGDATA=/var/lib/postgresql/data/pgdata \
  -v pgvolume:/var/lib/postgresql/data \
  -p 5532:5432 \
  --name pgvector \
  agnohq/pgvector:16
```

### ステップ2: 従来のRAG

Retrieval Augmented Generation (RAG)とは、モデルの応答を改善するために**「プロンプトに関連情報を含める」**ことを意味します。これは2段階のプロセスです。

1. ナレッジベースから関連情報を取得します。
2. モデルにコンテキストを提供するためにプロンプトを拡張します。

レシピのPDFから質問に答える**従来のRAG** Agentを構築しましょう。

<Steps>
  <Step title="ライブラリをインストールする">
    pipを使用して必要なライブラリをインストールします

    <CodeGroup>

    ```bash Mac
    pip install -U pgvector pypdf "psycopg[binary]" sqlalchemy
    ```

    ```bash Windows
    pip install -U pgvector pypdf "psycopg[binary]" sqlalchemy
    ```

    </CodeGroup>

  </Step>
  <Step title="従来のRAG Agentを作成する">
    次の内容で `traditional_rag.py` ファイルを作成します

    ```python traditional_rag.py
    from agno.agent import Agent
    from agno.models.openai import OpenAIChat
    from agno.knowledge.pdf_url import PDFUrlKnowledgeBase
    from agno.vectordb.pgvector import PgVector, SearchType

    db_url = "postgresql+psycopg://ai:ai@localhost:5532/ai"
    knowledge_base = PDFUrlKnowledgeBase(
        # このURLからPDFを読み込みます
        urls=["https://agno-public.s3.amazonaws.com/recipes/ThaiRecipes.pdf"],
        # `ai.recipes` テーブルに埋め込みを保存します
        vector_db=PgVector(table_name="recipes", db_url=db_url, search_type=SearchType.hybrid),
    )
    # ナレッジベースをロードします: 初回実行後にコメントアウトしてください
    knowledge_base.load(upsert=True)

    agent = Agent(
        model=OpenAIChat(id="gpt-4o"),
        knowledge=knowledge_base,
        # AgentKnowledgeからの参照をユーザープロンプトに追加することでRAGを有効にします。
        add_references=True,
        # エージェントはデフォルトで `search_knowledge=True` になるためFalseに設定します
        search_knowledge=False,
        markdown=True,
        # debug_mode=True,
    )
    agent.print_response("How do I make chicken and galangal in coconut milk soup")
    ```

  </Step>
  <Step title="エージェントを実行する">
    エージェントを実行します（ナレッジベースのロードに数秒かかります）。

    <CodeGroup>

    ```bash Mac
    python traditional_rag.py
    ```

    ```bash Windows
    python traditional_rag.py
    ```

    </CodeGroup>

    <br />

  </Step>
</Steps>

<Accordion title="ローカルPDFの使用方法" icon="file-pdf" iconType="duotone">
ローカルPDFを使用したい場合は、代わりに `PDFKnowledgeBase` を使用します

```python agent.py
from agno.knowledge.pdf import PDFKnowledgeBase

...
knowledge_base = PDFKnowledgeBase(
    path="data/pdfs",
    vector_db=PgVector(
        table_name="pdf_documents",
        db_url=db_url,
    ),
)
...
```

</Accordion>

### ステップ3: エージェント型RAG

上記の従来のRAGでは、`add_references=True` は、質問に関連しているか役立つかに関係なく、常にナレッジベースからの情報をプロンプトに追加します。

エージェント型RAGでは、エージェントがナレッジベースにアクセスする必要があるかどうか、およびナレッジベースをクエリするために必要な検索パラメータを決定できるようにします。

`search_knowledge=True` と `read_chat_history=True` を設定し、エージェントにオンデマンドでナレッジとチャット履歴を検索するツールを提供します。

<Steps>
  <Step title="エージェント型RAG Agentを作成する">
    次の内容で `agentic_rag.py` ファイルを作成します

    ```python agentic_rag.py
    from agno.agent import Agent
    from agno.models.openai import OpenAIChat
    from agno.knowledge.pdf_url import PDFUrlKnowledgeBase
    from agno.vectordb.pgvector import PgVector, SearchType

    db_url = "postgresql+psycopg://ai:ai@localhost:5532/ai"
    knowledge_base = PDFUrlKnowledgeBase(
        urls=["https://agno-public.s3.amazonaws.com/recipes/ThaiRecipes.pdf"],
        vector_db=PgVector(table_name="recipes", db_url=db_url, search_type=SearchType.hybrid),
    )
    # ナレッジベースをロードします: 初回実行後にコメントアウトしてください
    knowledge_base.load(upsert=True)

    agent = Agent(
        model=OpenAIChat(id="gpt-4o"),
        knowledge=knowledge_base,
        # ナレッジベースを検索できるツールを追加します。これによりエージェント型RAGが有効になります。
        search_knowledge=True,
        # チャット履歴を読み取るツールを追加します。
        read_chat_history=True,
        show_tool_calls=True,
        markdown=True,
        # debug_mode=True,
    )
    agent.print_response("How do I make chicken and galangal in coconut milk soup", stream=True)
    agent.print_response("What was my last question?", markdown=True)
    ```

  </Step>
  <Step title="エージェントを実行する">
    エージェントを実行します

    <CodeGroup>

    ```bash Mac
    python agentic_rag.py
    ```

    ```bash Windows
    python agentic_rag.py
    ```

    </CodeGroup>

    <Note>
    必要に応じてナレッジベースとチャット履歴を検索する方法に注意してください
    </Note>

  </Step>
</Steps>

## Attributes

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `knowledge` | `AgentKnowledge` | `None` | エージェントによって使用されるナレッジベースを提供します。 |
| `search_knowledge` | `bool` | `True` | モデルがナレッジベースを検索できるようにするツールを追加します（エージェント型RAGとも呼ばれます）。`knowledge` が提供されるとデフォルトで有効になります。 |
| `add_references` | `bool` | `False` | AgentKnowledgeからの参照をユーザープロンプトに追加することでRAGを有効にします。 |
| `retriever` | `Callable[..., Optional[list[dict]]]` | `None` | ユーザーメッセージに追加するコンテキストを取得するための関数。この関数は `add_references` が True の場合に呼び出されます。 |
| `context_format` | `Literal['json', 'yaml']` | `json` | RAGのフォーマットを指定します。"json" または "yaml" のいずれかです。 |
| `add_context_instructions` | `bool` | `False` | Trueの場合、システムプロンプトにコンテキストを使用するための指示を追加します（ナレッジも提供される場合）。たとえば、トレーニングデータよりもナレッジベースからの情報を優先する指示を追加します。 |
